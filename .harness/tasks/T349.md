## Overview
Per the idea's explicit chooser/needs-human/finalize requirement (mirroring this repo's own T331/T332/T333 PR-badge-pill precedent), before committing to a real placement for the Top of Mind note on the homepage, build 5 genuinely different dummy-content visual treatments directly on the real src/pages/index.js layout (not a standalone dev comparison page, since judging placement means judging it in context among the real hero/stat-tiles/collection-wall/rail sections), switchable via a `?tom=1..5` query param (default 1), with `?tom=0` rendering no section at all as the baseline. This unit does NOT depend on the storage/API unit — it uses hardcoded dummy markdown (a heading + short paragraph + a bullet list, to exercise wrapping/overflow) and is purely a visual/placement comparison; no fetch, no real data.

## Do
In src/pages/index.js, read the `tom` query param via `useRouter` (`router.query.tom`, default to '1' if absent/invalid) and gate which variant renders behind it (client-side only is fine, matching this page's existing client-fetch pattern — no need for getServerSideProps). Define a single hardcoded dummy markdown string (module-level constant, e.g. `DUMMY_TOM_TEXT` with a `## heading`, a short paragraph, and a 3-4 item bullet list) and render it through the existing `Markdown` component (src/components/Markdown.js) in every variant, styled per-variant via new globals.css rules (prefix new classes `.tom-variant-*` or `.top-of-mind-*` to avoid clashing with existing `.home-*` classes). Build these 5 variants: (1) 'Rail panel' — a new panel appended into the existing `.home-rail` column (after home-hot-takes-panel, ~line 303), styled as a sibling to home-hot-takes-panel/home-shelf-panel (same panel chrome: title row + body); (2) 'Hero band' — a full-width band inserted directly under the `home-hero` section (~after line 163), before `home-stats`, spanning the container width with its own distinct background tint; (3) 'Collapsible section' — a `<details>`/`<summary>`-based collapsible block inserted between `home-wall` (~187) and `home-lower` (~189), collapsed by default with a short teaser visible when closed; (4) 'Latest-takes card' — a card woven into the top of the 'Latest takes' left column (inside `.home-latest`, ~190-216), styled consistently with the `.home-latest-card` items already there but visually distinguished (e.g. an accent border) as a non-review item; (5) 'Hero inline strip' — a compact single-line-teaser strip placed inside `home-hero` itself, directly under the tagline/meta row (~after line 159/163, before or replacing nothing existing), not full width, minimal height. Each variant must be reachable independently via its `?tom=N` value with no bleed from the others (only one renders at a time), and `?tom=0` (or any other value) renders none of them — the rest of the homepage must look and behave exactly as it does today in every case.
