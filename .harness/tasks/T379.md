## Overview
On every paginated view, the 'next →' pill in the shared Pagination component stays clickable past the last page — it calls onPageChange(currentPage + 1) with no disabled guard, so you can page past the end. The owner has decided to remove the 'next →' button entirely rather than disable it on the last page: users advance by clicking the next page NUMBER instead, and removing the button eliminates the past-the-last-page bug as a side effect. src/components/Pagination.js is the single shared pagination component — all paginated surfaces route through it (the 5 direct consumers: src/pages/workouts/index.js and src/pages/reviews/{books,movies,tv,albums}/index.js; plus the 4 backfill pages via src/components/BulkBackfillList.js), so this one edit fixes all of them. This is a distinct concern from the album-backfill logic ideas that touch the same backfill pages — it's pager UI only.
