# T018 — TMDB search proxy route + normaliser

## Do

Add a server-side TMDB search proxy so the API key stays off the client.

- `src/pages/api/tmdb/search.js` (GET) — params `query` (title) and `type`
  (`movie` | `tv`); calls the TMDB search endpoint server-side using
  `process.env.TMDB_API_KEY`; returns a normalised result list. Wrap the read in
  `withApiCache` (`generateCacheKey('tmdb-search', { query, type })`).
- Put the pure normalisation in `src/lib/tmdb.js`: `mapTmdbResult(raw, type)` →
  `{ tmdbId, mediaType, title, overview, posterPath, date }` (date = `release_date` for
  movies / `first_air_date` for tv; title = `title`/`name`), plus a `tmdbPosterUrl(path)`
  helper for building an image URL from a poster path. Unit-test these.
- Document the new `TMDB_API_KEY` env var in `README.md` (and the CLAUDE.md env list).

## Done when

- `GET /api/tmdb/search?query=...&type=movie|tv` returns a normalised array
  `[{ tmdbId, mediaType, title, overview, posterPath, date }]`, cached via `withApiCache`,
  reading the key from `process.env.TMDB_API_KEY` (never exposing it to the client).
- `src/lib/tmdb.js` is covered by `src/lib/tmdb.test.js` against a synthetic TMDB response
  (hermetic — no live call): movie maps `release_date`+`title`, tv maps `first_air_date`+
  `name`, missing poster yields a null/blank `posterPath`. `npm test` passes.
- `TMDB_API_KEY` is documented in README + CLAUDE.md.
- `npm run lint`, `npm test`, `npm run build` pass.
