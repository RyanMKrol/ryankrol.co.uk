# T028 — Last.fm album-search proxy routes + normaliser

## Do

Add server-side Last.fm proxy routes so the album/vinyl add flows can search for an album and
fetch rich metadata, mirroring the TMDB (`src/pages/api/tmdb/search.js` + `src/lib/tmdb.js`)
and Open Library (`src/pages/api/books/search.js` + `src/lib/openlibrary.js`) patterns. The
Last.fm key already exists in env as `LAST_FM_API_KEY` — reuse it; do NOT add a new key.

Create:
- `src/lib/lastfm.js` — **pure** normalisers (no network), unit-tested:
  - `mapAlbumSearchResult(raw)` — from a Last.fm `album.search` result item → a clean object
    `{ title, artist, mbid, url, image }` where `image` is the largest non-empty image URL (the
    Last.fm `image` array has sizes small/medium/large/extralarge — pick the biggest with a
    non-empty `#text`, else null).
  - `mapAlbumInfo(raw)` — from a Last.fm `album.getInfo` result → a generous metadata object
    (we deliberately OVER-store): `{ title, artist, mbid, url, image, listeners, playcount,
    tags (array of tag names), trackCount, summary (the wiki summary text, HTML stripped or
    raw — your call, but documented), releaseDate (if present, else null), images (all sizes
    keyed by size) }`. Be defensive: Last.fm fields are frequently missing — default to
    null/[]/0, never throw on a sparse response.
- `src/pages/api/lastfm/album-search.js` (GET `?query=`) — calls Last.fm `album.search` with the
  query (album name; Last.fm's album.search takes the album term only — the artist is shown in
  results for disambiguation), returns `{ results: [mapAlbumSearchResult, ...] }`. Wrap in
  `withApiCache(generateCacheKey('lastfm-album-search', { query }), ...)`. Return `[]` on a blank
  query. 405 non-GET. Log cache hit/miss + timing with emoji prefixes like the other routes.
- `src/pages/api/lastfm/album-info.js` (GET `?artist=&album=` and/or `?mbid=`) — calls Last.fm
  `album.getInfo` (prefer mbid when supplied, else artist+album), returns `{ info: mapAlbumInfo }`.
  Wrap in `withApiCache(generateCacheKey('lastfm-album-info', { artist, album, mbid }), ...)`.
  405 non-GET.

Update `CLAUDE.md` (file map + API table) and `README.md` if it lists API routes, in the same
commit, documenting the two new routes + `src/lib/lastfm.js`.

## Done when

- `src/lib/lastfm.js` exports `mapAlbumSearchResult` + `mapAlbumInfo`, covered by
  `src/lib/lastfm.test.js`: a representative `album.search` item maps to `{title,artist,mbid,url,
  image}` with the LARGEST image chosen; a sparse `album.getInfo` (missing tags/wiki/tracks) maps
  without throwing, defaulting missing fields to null/[]/0; a full `getInfo` captures listeners,
  playcount, tag names, track count, and summary. `npm test` passes.
- `GET /api/lastfm/album-search?query=rumours` returns a JSON `{ results: [...] }` array of
  normalised matches (each with title, artist, and a cover image url when available); a blank
  `query` returns `{ results: [] }`; non-GET → 405.
- `GET /api/lastfm/album-info?artist=Fleetwood+Mac&album=Rumours` returns `{ info: {...} }` with
  the generous metadata object; non-GET → 405.
- Both routes are wrapped in `withApiCache` with distinct, parameterised cache keys and use the
  existing `LAST_FM_API_KEY` env var (a missing key returns a clear 500, like `top-albums.js`).
- `CLAUDE.md` (and `README.md` if applicable) document the new routes + lib in the same commit.
- `npm run lint`, `npm test`, `npm run build` all pass.
