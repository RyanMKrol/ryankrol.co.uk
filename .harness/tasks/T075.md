# T075 — Perfume "Application notes" field — form UI

## Do

T074 added an `applicationSpots` field (array subset of `"Wrists"`, `"Elbows"`, `"Clavicles"`,
`"Beard"`, `"Back of neck"`, `"Behind ears"`) to the perfume review backend. Update
`src/pages/reviews/perfumes/add.js` and `src/pages/reviews/perfumes/edit/[id].js`:

1. Add six checkboxes (one per application spot listed above) letting the user tick any
   combination for a given wear; submit as an array of the ticked labels (exact strings as above)
   into `applicationSpots`.
2. Relabel the existing "Notes" field/label to **"Application notes"**, and change its
   placeholder text from "Top/heart/base notes..." to something reflecting general application
   notes (e.g. "Any other notes about this wear..."). The field itself (`formData.notes`) is
   otherwise unchanged — it remains a free-text textarea; only the label/placeholder change,
   reflecting that this field's purpose has fully shifted from fragrance-note commentary to
   general application-notes commentary (the fragrance-notes framing is not kept anywhere else).
3. Wire `applicationSpots` into each form's existing `formData` state and submit payload
   (`add.js` → `POST /api/reviews/perfumes/add`, `edit/[id].js` → `POST
   /api/reviews/perfumes/update`). On the edit form, pre-populate the checkboxes from the loaded
   review's existing `applicationSpots` (falling back to none ticked if the review predates this
   field).

## Done when

- `/reviews/perfumes/add` renders six checkboxes for Wrists/Elbows/Clavicles/Beard/Back of
  neck/Behind ears, and a field labeled "Application notes" (not "Notes") with an updated
  placeholder. Submitting sends `applicationSpots` (array of ticked labels) and the free-text
  value to `POST /api/reviews/perfumes/add`.
- `/reviews/perfumes/edit/[id]` renders the same six checkboxes pre-filled from the loaded
  review's `applicationSpots` (or none ticked if absent), and the same relabeled "Application
  notes" field. Submitting sends the updated values to `POST /api/reviews/perfumes/update`.
- Manually loading `/reviews/perfumes/add` in a running dev server (`npm run dev`), ticking two
  checkboxes (e.g. Wrists and Behind ears), and submitting persists both in the saved review's
  `applicationSpots` array (verify by reloading the edit page for that review and seeing both
  still ticked).
