# T015 — Appearance picker modal

## Do

Build the **Appearance** picker modal, modelled on the owner's screenshot, wired to the
T013 theme system and the T014 themes.

- `src/components/AppearancePicker.js`: a centered card titled "🎨 Appearance" with a
  close (×) button and labelled sections:
  - **THEME** — a grid of selectable tiles (emoji + name per theme), active one highlighted.
  - **FONT** — tiles for the available fonts (System default + the theme fonts), active highlighted.
  - **MODE** — tiles Dark / Light / System, with helper text "System follows your OS
    dark/light preference."
  - A checkbox **"Reduce motion & minimise emoji"** at the bottom.
- Wire every control to `useTheme` so selections apply live and persist to `localStorage`.
- Add a trigger to open the modal — a 🎨 control in `src/components/Header.js`.
- Style the modal (rounded tiles, soft borders, the warm card look) in `globals.css`,
  themed via the CSS vars so it adapts to the active theme.

## Done when

- A 🎨 control in the header opens the Appearance modal; it shows THEME / FONT / MODE
  tiles + the reduce-motion checkbox and highlights the active selections.
- Changing any control applies live AND persists (reload keeps the choice). MODE `System`
  follows the OS; Dark/Light force it. The reduce-motion toggle sets `data-motion` and
  dampens animations (including the Matrix rain / CSS flicker, per the existing
  `prefers-reduced-motion` handling).
- The modal is keyboard-closable (× / Esc) and does not trap focus oddly.
- `npm run lint`, `npm test`, `npm run build` pass; manually open the picker, switch
  theme/font/mode, toggle reduce-motion, reload, and confirm everything persists.
