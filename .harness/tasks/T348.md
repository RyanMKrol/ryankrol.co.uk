## Overview
The owner wants a free-form 'Top of Mind' Markdown note editable from a dedicated page, persisted in DynamoDB, with a 90-day TTL measured from the last edit: past 90 days since the last save, the note is treated as expired everywhere it's read for display (the homepage, in a later task) but the edit page must still show the owner their old/expired text so they can revive it (a fresh save always resets the clock). No generic key-value/settings table exists in this codebase today (checked src/lib/constants.js's DYNAMO_TABLES — every existing table is a UUID-keyed collection like HotTakes/PerfumeRatings), so this task introduces one dedicated single-row table for this note specifically, following this repo's other conventions (DYNAMO_TABLES as the single source of table names, docClient/PutCommand/GetCommand from src/lib/dynamo.js, RYANKROL_SITE_KEY write-gating, clearApiCache on write).

Create the actual DynamoDB table out of band (partition key `id`, String) the same way other tables in this repo are provisioned (mirror the style of src/scripts/createWorkoutTables.js if a table-creation script convention needs following — check src/scripts/ for the closest existing precedent and follow it; if table creation here is understood to be a manual/out-of-band AWS console step rather than a checked-in script for other tables, follow that same precedent instead of inventing a new script). The table stores exactly one item, always at the fixed key `{ id: 'top-of-mind' }`, shaped `{ id: 'top-of-mind', text: '<markdown string>', updatedAt: '<ISO 8601 UTC timestamp, e.g. new Date().toISOString()>' }`.
