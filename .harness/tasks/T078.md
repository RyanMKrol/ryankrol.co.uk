# T078 — Delete src/lib/openlibrary.js + update docs

## Do

**Update (found by a prior stuck attempt, corrected here):** T077 removed Open Library from the
*search* path, but two more call sites still import `bookCoverUrl` from `../lib/openlibrary` as a
fallback for legacy Open-Library-sourced book reviews (`coverId` present, no `coverUrl`):
`src/components/BookSearch.js` (`if (r.coverId) return bookCoverUrl(r.coverId, 'S')`) and
`src/pages/index.js` (`b.coverUrl || (b.coverId ? bookCoverUrl(b.coverId) : null)`). T080 already
ran the production migration (`src/scripts/migrateOpenLibraryBooksToGoogleBooks.js`) and confirmed
**zero** `source === 'openlibrary'` records existed in `BookRatingsV3` at that time (53 total
records scanned, 0 openlibrary-sourced) — and since T077 already removed the `openlibrary` search
provider, no new review can be created with `coverId`/no `coverUrl` since. So the `coverId`
fallback in both files is dead code: remove the `bookCoverUrl` import from each and use `coverUrl`
directly (`r.coverUrl` in `BookSearch.js`, `b.coverUrl` in `index.js`'s wall-tile mapping — drop
the `(b.coverId ? bookCoverUrl(b.coverId) : null)` fallback entirely).

Confirm nothing else in `src/` still imports from `../lib/openlibrary` or `./openlibrary` (grep for
`openlibrary` across `src/` to be sure — **except** `src/scripts/migrateOpenLibraryBooksToGoogleBooks.js`
and its test, which legitimately keep textual `'openlibrary'` references as data-migration logic
per T079/T080 and are NOT part of this deletion), then delete `src/lib/openlibrary.js` and its test
file `src/lib/openlibrary.test.js` entirely.

Update documentation to remove Open Library references (per this repo's Definition-of-Done rule
that docs are updated in the same change as the code they describe):
- **`CLAUDE.md`**: remove the `src/lib/openlibrary.js` row from the file-map table; update the
  `src/pages/api/books/search.js` file-map row to no longer mention the `provider=openlibrary|
  googlebooks` switch (it's Google-Books-only now, per T077); remove `source:'openlibrary'`/`olid`/
  `coverId` from the `BookRatingsV3` data-model field list (these fields will still exist on
  historical rows — see T079/T080 for the migration — but new reviews never write them, so drop
  them from the documented "current" schema; leave `source`/`coverUrl`/`volumeId`/`bookAuthors`/
  `firstPublishedYear`/`isbn`/`subjects`/`pageCount`/`publisher` as-is, those are all
  Google-Books-compatible fields that remain).
- **`README.md`**: remove any Open Library mention in the `BookRatingsV3` schema section, matching
  the `CLAUDE.md` change above.
