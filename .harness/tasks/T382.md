## Overview
Blank lines typed in the Markdown editor currently vanish. A user typing a completely empty line between two paragraphs (in the /tom 'Top of mind' note) sees no extra gap — neither in the live PREVIEW pane nor on the rendered homepage — because the shared renderer src/components/Markdown.js uses react-markdown/CommonMark, under which any run of blank lines collapses to a single paragraph break. Fix the shared component so intentional blank lines survive as visible vertical gaps, without changing normal spacing, without touching inline mode, and without breaking any existing formatting.

## Do
Edit src/components/Markdown.js. Add a pure helper (e.g. `preserveBlankLines(src)`) that, given the markdown source string, converts each run of N>=2 consecutive blank lines into a paragraph break plus (N-1) 'gap paragraphs', where each gap paragraph is a line whose sole content is a non-breaking space (\u00A0) delimited by blank lines so react-markdown renders it as a standalone <p>. Call this helper on `children` ONLY in the block-mode branch (the `return <ReactMarkdown remarkPlugins={[remarkGfm]}>...` path), never in the `inline` branch. Guard for non-string children (pass through untouched). Keep remark-gfm and the existing inline-mode behaviour exactly as-is. Add tests to src/components/Markdown.test.js. Update CLAUDE.md's Markdown/component notes only if a convention changes (the component's behaviour is now blank-line-preserving in block mode).

## Done when
A completely blank line typed between two paragraphs renders as a visible extra vertical gap in BOTH the /tom preview (src/pages/tom.js) and the homepage render (src/pages/index.js); two consecutive blank lines produce roughly twice the gap; normal single-paragraph-break spacing is visually unchanged; bold/italic/list/link still render; inline mode (album track highlights via ReviewCard) is byte-for-byte unaffected. Tests in src/components/Markdown.test.js MUST assert: (1) a single blank line between two paragraphs produces an extra gap paragraph in block mode (e.g. an additional <p> beyond the two content paragraphs, and the existing content paragraphs still present); (2) inline mode does NOT emit any gap paragraph / extra empty element for the same blank-line input; (3) the existing bold/italic/list/link assertions still pass. `npm run lint`, `npm test`, `npm run build` all green. Because this is a visual change, verify with `node scripts/visual-check.mjs` and LOOK at the /tom and homepage shots (per the project's visual-verification convention) — a passing build alone is not sufficient for a rendering fix (cf.
