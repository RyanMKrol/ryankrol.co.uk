# T030 — Last.fm search + confirm step in the album & vinyl add flows

## Do

Add a Last.fm search/confirm step to the **album** review add page and the **vinyl** add page,
using ONE shared component, mirroring `src/components/TmdbSearch.js` (movies/TV) and
`src/components/BookSearch.js` (books). The search must be driven by the form fields the user is
ALREADY entering — not a separate standalone search box.

- Create `src/components/LastfmAlbumSearch.js` — a reusable search/confirm component:
  - A **"Search"** button that reads the form's existing **title** field (Last.fm `album.search`
    takes the album term only) and calls `GET /api/lastfm/album-search?query=<title>`.
  - Renders the result list with **cover thumbnail + album title + artist** so the user can
    disambiguate (artist isn't a search param, so the picker is how the right artist is chosen).
  - On **selecting** a result, fetch `GET /api/lastfm/album-info?mbid=…` (or `?artist=&album=`)
    to pull the rich metadata, then call an `onSelect(metadata)` prop with the normalised object
    (cover URL, mbid, url, listeners, playcount, tags, trackCount, summary, releaseDate, images)
    so the parent form can populate `title`/`artist` and stash the enrichment for submit.
  - Use theme CSS custom properties (no hardcoded colours); look correct under default +
    `matrix-active` themes. Show a loading state and a graceful "no results"/error message.
- Wire it into `src/pages/reviews/albums/add.js`: the component reads the title field; selecting a
  match fills in title + artist and attaches the Last.fm metadata to the POST body sent to
  `/api/reviews/albums/add` (consumed by T029). The user can still type everything by hand and
  submit without searching.
- Wire the SAME component into `src/pages/vinyl/add.js`: selecting a match fills title + artist
  and attaches the metadata to the POST body sent to `/api/vinyl/add` (consumed by T029).

## Done when

- `src/components/LastfmAlbumSearch.js` exists as a single shared component and is used on BOTH
  `src/pages/reviews/albums/add.js` and `src/pages/vinyl/add.js`.
- The Search button reads the form's title field (no duplicate search-term input); results show
  cover + title + artist; picking one fills title + artist and captures the Last.fm metadata for
  submission.
- Submitting after a pick sends the enrichment so the stored album/vinyl record gets `thumbnail` +
  `lastfm` (verifiable end-to-end with T029). Submitting WITHOUT searching still works (no field
  becomes required).
- Component styling is theme-driven (CSS custom properties), correct under default + matrix themes.
- Manually verify against the running app: on `/reviews/albums/add` and `/vinyl/add`, type a title,
  click Search, pick a result, and confirm title/artist auto-fill and a cover preview appears.
- `npm run lint`, `npm test`, `npm run build` all pass; docs updated if any convention changed.
