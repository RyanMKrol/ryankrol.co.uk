# T011 — Date-range filter on the per-exercise page

## Do

Add the date-range filter (T010) to the per-exercise chart page
(`src/pages/exercises/[exerciseName].js`), defaulting to **Last 1 year**.

- Render `DateRangeFilter` (from T010) on the page and keep the selected range in state
  (default `1y`).
- Make the range accurate over ALL history: raise/remove the 50-session cap so the page
  has the full exercise history to filter. Update `getExerciseHistory` in
  `src/lib/workoutQueries.js` (its `Limit: 50`) and the API route
  `src/pages/api/exercises/history/[exerciseName].js` (which passes `limit`) so the page
  can request the full history (e.g. a high/no limit, or an explicit "all" mode). Keep the
  existing `withApiCache` wrapper.
- Filter the charted sessions to the selected range with `filterByDateRange` (keying on
  each record's `workout_date`) BEFORE passing data to `ExerciseProgressCharts` /
  `CardioProgressCharts` and before computing the on-page stats, so every chart + stat
  reflects the chosen window.

## Done when

- The per-exercise page shows a 3-months / 1-year / all-time control, defaulting to 1 year.
- Selecting a range re-scopes every chart AND the derived stats on the page to that window.
- The page loads the full exercise history (not just the latest 50) so 1-year / all-time
  are accurate; the read stays wrapped in `withApiCache`.
- `npm run lint`, `npm test`, `npm run build` pass; manually load an exercise page, switch
  ranges, and confirm the charts + stats update.
