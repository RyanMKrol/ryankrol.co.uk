## Overview
Hardcover's own docs describe their API as 'only for offline use...only from localhost or APIs...never from a browser' — ambiguous as to whether this is just 'server-side only' (which this repo's existing /api/* proxy architecture already satisfies for every provider it uses) or a genuine restriction against calling it from cloud/production infrastructure like Vercel's serverless functions. The prior trial only ever ran the Hardcover call via `npm run dev` locally and never tested a real deployed instance. Nothing that assumes Hardcover works reliably in production — the final removal of Google Books (if that's the decision), and definitely the destructive strip-old-metadata-from-BookRatingsV4 migration — should proceed until this is empirically resolved, since reverting a broken production book-search route or a botched destructive DynamoDB migration are both far more costly than the small delay of checking first.

## Do
This is a needs-human gated investigation, not an autonomous-build unit — do not attempt code changes here. The owner (or a session working with the owner) needs to: (1) add HARDCOVER_API_TOKEN to Vercel's project env vars (currently only in local .env.local, gitignored); (2) deploy the branch/commit containing hardcover-migration-a's rewritten /api/books/search.js to a real Vercel deployment (preview deployment is sufficient to test the same serverless runtime prod uses); (3) hit the deployed /api/books/search?title=... endpoint directly (curl or browser) and confirm it returns real Hardcover results rather than an error, a block, or a silent empty response; (4) if Hardcover's request is blocked/rejected specifically because of Vercel's origin/infrastructure (as opposed to an unrelated bug), treat that as a hard blocker and escalate back to the owner for a fallback decision (e.g. keep Google Books permanently, proxy Hardcover through a different runtime, or abandon the migration) rather than silently reverting or shipping around it.

## Done when
A deployed (non-localhost) Vercel instance has made at least one real, successful Hardcover GraphQL search request and returned normal results, OR the investigation has conclusively determined Hardcover blocks/rejects Vercel-origin requests and that finding has been reported back to the owner for a decision — either outcome closes this unit; it is not 'done' while the question remains untested.
