# T005 — Perfumes review pages + nav link

## Do

Add the perfumes pages under `src/pages/reviews/perfumes/`, mirroring
`src/pages/reviews/movies/`, wired to the T004 API routes:

- `index.js` — fetch `/api/reviews/perfumes`, sort, render the list (reuse
  `ReviewCard`, adapting it if needed to show a perfume sensibly).
- `add.js` — a password-gated form with fields: `title`, `designer`,
  `type` (EDP / EDT / Parfum), `rating` (integer 0–10), `description`, `notes`,
  `considerTravelSize` (checkbox), `considerFullBottle` (checkbox); POSTs to the add API.
- `edit.js` — list of perfumes with edit links.
- `edit/[id].js` — detail form → update/delete API; `[id]` is the URL-encoded `id`.

Add a link to `/reviews/perfumes` in the home page link grid (`src/pages/index.js`).

## Done when

- The four pages render and round-trip against the perfumes API (add → appears in list →
  edit → delete).
- The rating input accepts only integers 0–10; the two booleans persist.
- A link to the perfumes section appears in the home grid.
- A perfume entry displays sensibly (title, designer, type, rating, description) via
  `ReviewCard` or a small adaptation.
- `npm run lint`, `npm test`, `npm run build` pass; manually load `/reviews/perfumes`
  and confirm the list renders and an add round-trips.
