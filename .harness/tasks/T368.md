## Overview
Album and vinyl review saves never persist a Last.fm-derived thumbnail. src/lib/lastfm.js's mapAlbumInfo() (lines 107-145) builds the `images` field as an object keyed by size — { small, medium, large, extralarge, mega } (built at lines 123-129) — but three separate call sites still derive the stored `thumbnail` with an expression written for the OLD shape (an array of {size, '#text'} entries): `lastfm?.images?.[lastfm.images.length - 1]?.['#text']`. Against an object, `.length` is undefined, so `images[undefined - 1]` is `images[NaN]`, which is always undefined — thumbnail can never be set via this path. Confirmed live against production data: 0 of 48 AlbumRatingsV3 records and 0 of 190 VinylCollectionV2 records have a thumbnail or lastfm.mbid from this feature (the 188 vinyl records that do have a thumbnail got it from an unrelated pre-Last.fm-integration seed/migration).
