# T008 — Programme (Push/Pull/Legs) stats aggregation helper

## Do

Add `src/lib/programmeStats.js` with pure, unit-tested helpers that aggregate a list
of workouts into per-programme (Push / Pull / Legs) statistics. These power the
programme overview page (T009); keep them free of React/DOM/DynamoDB so they're testable.

- `classifyProgramme(workout)` → `'push' | 'pull' | 'legs' | null` — case-insensitive
  substring match of those words against `workout.title` (there is no structured
  programme field; `workoutType` is `strength`/`cardio`/`mixed` and is unrelated).
- `aggregateProgramme(workouts, programme)` → an object with:
  - `sessions`: `[{ date, volume, workingSets }]` for workouts of that programme, sorted
    by date ascending (use `workout.start_time`/`workoutDate`, `totalVolume`,
    `totalWorkingSets`).
  - `frequencyByMonth`: `[{ month: 'YYYY-MM', count }]`.
  - `totals`: `{ workouts, totalVolume, avgVolumePerWorkout }` (round sensibly).

## Done when

- `classifyProgramme` matches title case-insensitively and returns `null` for
  non-Push/Pull/Legs titles.
- `aggregateProgramme` returns the date-ascending per-session volume + working-set series,
  the per-month frequency series, and the headline totals, computed only from the passed
  workouts (no I/O).
- Covered by `src/lib/programmeStats.test.js` using synthetic workouts (hermetic): asserts
  classification, that only the selected programme's workouts are included, the series are
  date-sorted, and the totals math (e.g. avg = totalVolume / workouts). `npm test` passes.
- `npm run lint`, `npm test`, `npm run build` pass.
