## Overview
Swap the Hot Takes add form's plain <textarea> for the shared MarkdownEditor component, matching every review-type add form (movies/tv/books/albums/perfumes). This also fixes the two visual complaints (pill-shaped corners, cramped placeholder) as a side effect of using the correct CSS class.

## Do
In src/pages/hot-takes/add.js, import MarkdownEditor from '../../components/MarkdownEditor' and replace the bare `<textarea id="text" name="text" ... className="collection-form-input" placeholder="What's your hot take..." rows={4} required />` (currently around line 82-91) with `<MarkdownEditor id="text" name="text" value={formData.text} onChange={handleInputChange} className="collection-form-textarea" placeholder="What's your hot take..." required />`, following the exact prop pattern used in src/pages/reviews/tv/add.js (around lines 132-138: id="gist", className="collection-form-textarea", placeholder text, required). Do not add any new CSS — root-cause investigation confirmed the visual complaints (pill-shaped rounded corners, placeholder sitting too close to the border) are caused entirely by the current markup using className="collection-form-input" instead of "collection-form-textarea": globals.css's .collection-form-textarea rule (~line 2847-2851) overrides border-radius from the pill shape (var(--radius-pill), 999px) down to a modest rounded-rect (var(--radius-stat), 16px) and sets min-height: 120px — none of which currently applies to the hot-takes field because it's on the wrong class. Using className="collection-form-textarea" (as MarkdownEditor defaults to, and as every review form explicitly passes) picks up that override automatically, with no extra CSS change needed.
