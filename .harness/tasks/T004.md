# T004 — Perfumes review API routes

## Do

Add the four API routes for the `perfumes` review type under
`src/pages/api/reviews/perfumes/`, mirroring `src/pages/api/reviews/movies/`:

- `index.js` (GET) —
  `withApiCache(generateCacheKey('perfumes'), () => scanTable(DYNAMO_TABLES.PERFUME_RATINGS_TABLE))`.
- `add.js` (POST) — password-gate on `RYANKROL_SITE_KEY`; build the record
  `{ id, title, designer, type, description, rating, notes, considerTravelSize,
  considerFullBottle, date }`; `PutCommand`; then `clearApiCache('api-perfumes')`.
- `update.js` (POST) — password-gate; if `id` changed, delete the old item then put
  the new one; `clearApiCache('api-perfumes')`.
- `delete.js` (DELETE) — password-gate; `DeleteCommand` by `{ id }`;
  `clearApiCache('api-perfumes')`.

Put the genuinely-new pure logic in a new `src/lib/perfumes.js` and unit-test it:
- `validatePerfumeRating(value)` — accepts an **integer 0–10 only** (reject decimals,
  negatives, >10, non-numbers).
- `perfumeId({ title, designer, type })` — a deterministic, unique key string from
  title+designer+type (so the duplicate "Ombré Leather" EDP vs Parfum both persist).
- `type` is one of `EDP` / `EDT` / `Parfum`.

`date` uses the existing review format `new Date().toLocaleDateString('en-GB').replace(/\//g, '-')`.

## Done when

- All four routes exist, follow the movies pattern, and gate writes on `RYANKROL_SITE_KEY`
  (a missing/incorrect password is rejected; non-GET-only methods 405 as in the existing routes).
- `add`/`update` reject a `rating` that isn't an integer 0–10 with HTTP 400.
- The read route is wrapped in `withApiCache` under key `api-perfumes`, and EVERY write
  calls `clearApiCache('api-perfumes')` (exact key — do NOT repeat the vinyl-cache key
  mismatch noted in CLAUDE.md).
- `src/lib/perfumes.js` exports `validatePerfumeRating` + `perfumeId`, covered by
  `src/lib/perfumes.test.js`: rating 0/5/10 valid; 6.5, -1, 11, "x" invalid; `perfumeId`
  is stable and differs for the same title with a different `type`. `npm test` passes.
- `npm run lint`, `npm test`, `npm run build` all pass.
