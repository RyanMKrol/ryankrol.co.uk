# T087 — Require a confirmed Google Books match to add a book review

## Do

Today, on `src/pages/reviews/books/add.js`, the Google Books lookup match (`bookMatch` state) is OPTIONAL
— the form can be submitted whether or not the user ever ran/confirmed a search, since the match
fields are only conditionally spread into the payload (`...(bookMatch && { volumeId: bookMatch.volumeId , ... })`).
Make confirming a Google Books match REQUIRED to submit this form:

1. **Client-side**: disable the submit button (or block `handleSubmit`) until `bookMatch`
   is non-null (i.e. the user has run a search and picked a candidate). Show a clear inline message
   near the search/match UI when no match is selected yet (e.g. "Search and select a Google Books
   match before saving").
2. **Server-side**: in `src/pages/api/reviews/books/add.js`, add `volumeId` to the required-fields check — reject
   the request with HTTP 400 (matching this route's existing error-response style) if
   `volumeId` is missing from the request body, alongside the existing required-field
   checks (title/rating/etc.) already enforced there.

Do NOT touch the edit/update flow (`src/pages/reviews/books/edit/[id].js` or the
corresponding `update.js` API route) — this idea is scoped to the ADD flow only, per the
original request ("when adding a new [type]").

## Done when

- Submitting `src/pages/reviews/books/add.js`'s form without ever having selected a Google Books match is blocked
  client-side (the submit action does not fire / is disabled), with a visible message explaining
  why.
- `POST /api/reviews/books/add` (or the route this file
  serves) rejects a request missing `volumeId` with HTTP 400, even if all other required
  fields (title, rating, etc.) are present — a test (new or added to an existing test file for
  this route, if one exists) asserts this.
- Submitting with a confirmed match (i.e. `volumeId` present) still succeeds exactly as
  before — no regression to the existing happy path.
- The edit/update flow for this review type is unchanged — this task does not touch it.
