## Overview
Idea #2 explicitly wants existing Google-Books/Open-Library-sourced metadata wiped from BookRatingsV4 once Hardcover is the sole provider, so old-provider data doesn't linger and so those books become newly eligible for a clean Hardcover-sourced re-backfill (clearing volumeId and olid flips needsBookBackfill(item) back to true per src/lib/backfillEligibility.js). This must mirror the dry-run/LIVE=1 discipline of src/scripts/stripBackfillEditedDates.js exactly, since it's a real, destructive write across production DynamoDB data with no undo. Gated on hardcover-migration-d (the production-access risk) because this is destructive and should not run until the new Hardcover-only path is confirmed viable end-to-end — running this before Hardcover access from production is confirmed would leave book reviews with NO rich metadata at all if the new provider then turns out to be blocked.

## Do
Create src/scripts/stripOldBookProviderMetadata.js following src/scripts/stripBackfillEditedDates.js's exact structure and conventions: dotenv-loads .env.local, builds a DynamoDBDocumentClient the same way, defaults to a dry run that only logs, and only performs real writes when `LIVE=1` is set in the environment. Scan DYNAMO_TABLES.BOOK_RATINGS_TABLE (from src/lib/constants.js) for every row that has any of these fields present: `source`, `coverUrl`, `volumeId`, `bookAuthors`, `firstPublishedYear`, `isbn`, `subjects`, `pageCount`, `publisher`, plus the legacy Open-Library-era `olid`/`coverId` fields (per root CLAUDE.md's Data model notes on historical rows). For each such row, build an UpdateCommand with a `REMOVE` UpdateExpression naming exactly the fields present on that row (don't blindly REMOVE fields that were never set — mirror stripBackfillEditedDates.js's targeted approach). Log each row's title and which fields it's stripping, both in dry-run (no write) and live (write + confirmation) modes, exactly matching the existing script's console.log style/emoji conventions. Add src/scripts/stripOldBookProviderMetadata.test.js unit-testing the pure decision logic (which fields a given row shape would have stripped, and that a row with none of the target fields is left untouched) the same way stripBackfillEditedDates.js exports `shouldStrip` for testability — export an equivalent pure helper (e.g. `fieldsToStrip(item)`) from this new script. Update README.md's BookRatingsV4 schema section to note that `source`/`coverUrl`/`volumeId`/etc. are Hardcover-only now (once hardcover-migration-a/b/c have shipped) and that legacy Google-Books/Open-Library values no longer exist on any row post-migration.
