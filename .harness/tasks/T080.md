# T080 — Run the Open-Library-to-Google-Books book migration against production

## Do

T079 built `src/scripts/migrateOpenLibraryBooksToGoogleBooks.js` (dry-run by default, real writes
behind a `LIVE=1` flag) and its npm script. This step requires the owner's AWS credentials and a
judgement call on the script's dry-run output, so it cannot be done by the autonomous loop.

1. Run `npm run books:migrate-openlibrary` (dry-run) and review the printed output: how many
   records would be updated, how many left as "no match — left as openlibrary".
2. If the dry-run output looks reasonable, re-run with the live flag (e.g. `LIVE=1 npm run
   books:migrate-openlibrary`) to actually apply the updates.
3. For any records the script logged as "no match", handle them manually if desired (e.g. edit
   that book review by hand via the site's edit page) — optional, at the owner's discretion.

## Done when

- The dry run has been reviewed by the owner.
- The live migration has been run against the production `BookRatingsV3` table (or the owner has
  explicitly decided not to, e.g. because there were zero Open-Library-sourced records to
  migrate) — either way, this task is marked done once the owner has made and acted on that call.
