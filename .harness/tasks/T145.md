Build `src/components/CoverTile.js`: a square (or near-square, per the vinyl vs. album/book use case
— accept a `variant`/`aspectRatio` prop if the aspect differs between contexts) tile with:
- Rounded corners (`var(--radius-cover)`), drop shadow `var(--shadow-cover)`.
- A gradient placeholder background when no real image is available — the design uses varied
  2-color diagonal gradients per item (see the many different gradient tiles in `renders/2b.png` and
  `renders/1a.png`, e.g. teal→lighter-teal, purple→pink, orange→yellow). Since there's no real
  design-system-specified gradient palette (the design brief says these are arbitrary placeholder
  colors, real cover art replaces them), derive a stable pseudo-random gradient per item from a hash
  of its title/id (so the same item always gets the same placeholder gradient across renders/reloads,
  not a new random one every render) — pick from a small fixed set of 2-color pairs spanning the
  section accent palette (coral/indigo/teal/marigold/pink/grape/green + a couple of neutrals) rather
  than fully random colors.
- A real-image slot: accept an `imageUrl` prop; when present, render an `<img>` filling the tile
  (with the same rounded corners + shadow) instead of the gradient placeholder.
- A caption below the tile: `title` (bold) + optional `subtitle` (e.g. artist name), smaller and
  muted, per the design's cover-tile captions.

This component will be reused by: the vinyl grid, the home page's "collection wall", and (indirectly,
via the album/book ReviewCard variants) cover art in review cards — build it generically enough for
all three (e.g. don't hardcode "artist" as a prop name if a more generic `subtitle` reads fine).
