# T037 — Dashboard: don't let already-reviewed Done tasks be re-selected for "Mark reviewed"

## Do

In the local backlog dashboard (`.harness/dashboard/server.js`), an already-**reviewed** task in the
Done section can still be manually ticked and re-submitted via "Mark N reviewed" — a meaningless
no-op write / extra commit (reviewing twice does nothing, and there is intentionally no "un-review"
action). Close that gap.

- In the client `task(t)` template, the Done-row selection checkbox (`class="sel-rev"`, currently
  rendered for every `bucket==='done'` row at the `else if (t.bucket==='done')` branch) should only
  be rendered when the task is **NOT** already reviewed. For a reviewed Done task, render **no
  selection checkbox at all** (the existing "reviewed" pill already communicates its state).
- Net effect: only not-reviewed Done rows have a `sel-rev` checkbox, so a reviewed task cannot be
  individually selected and cannot be included in a "Mark N reviewed" submit. The existing
  `toggleAllRev` "Select all unreviewed" already skips reviewed rows; this makes the per-row control
  consistent with it.
- Do NOT add an un-review action and do NOT otherwise change the review flow (reviews.json /
  mark-reviewed.sh / the bulk endpoint are untouched). Reviewed rows must still render normally
  (expandable, with the "reviewed" pill) — they just lack the selection checkbox.

## Done when

- A Done task that is already reviewed renders with its "reviewed" pill but **no** `sel-rev`
  selection checkbox; it cannot be ticked or included in a "Mark N reviewed" submit.
- A not-reviewed Done task still renders its `sel-rev` checkbox and can be selected + bulk-marked
  reviewed exactly as before; "Select all unreviewed" and the Show All/Reviewed/Not-reviewed filter
  still work.
- No un-review action is introduced; reviews.json / mark-reviewed.sh / the mutation endpoints are
  unchanged.
- Manually verify: run `npm run harness:dashboard`; in the Done section a reviewed task has no
  checkbox while an unreviewed one does; bulk "Mark N reviewed" only counts unreviewed selections.
- The dashboard remains dependency-free; `npm run lint` and `npm test` pass; the existing
  `.harness/dashboard/lib.test.js` still passes.
