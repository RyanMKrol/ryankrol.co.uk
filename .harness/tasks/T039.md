# T039 — Perfume longevity/projection/seasons fields — form UI

## Do

T038 added `longevity` (integer 0-8), `projection` (integer 1-4), and `seasons` (array subset of
`"Winter"`, `"Spring"`, `"Summer"`, `"Autumn"`, `"Day"`, `"Night"`) to the perfume review backend.
Add matching UI controls to both `src/pages/reviews/perfumes/add.js` and
`src/pages/reviews/perfumes/edit/[id].js`:

- A **longevity slider** ranging 0-8, displaying the current value as plain text EXCEPT at the
  maximum (8), where it must display "8+" instead of "8".
- A **projection slider** ranging 1-4, displaying the Fragrantica-style label for the current
  value live as it's dragged: 1="Skin scent", 2="Moderate", 3="Strong", 4="Beast mode".
- A **seasons multi-select** (checkboxes or an equivalent tickable control) with six independent
  options — Winter, Spring, Summer, Autumn, Day, Night — any combination of which can be ticked,
  submitted as an array of the ticked labels (using the exact strings `"Winter"`, `"Spring"`,
  `"Summer"`, `"Autumn"`, `"Day"`, `"Night"`).

No slider or multi-select-checkbox component exists in this codebase yet — build minimal
presentational components under `src/components/` if a shared implementation is useful across the
add and edit forms, or inline the markup directly in each page if that's simpler; either is fine.

Wire all three controls into each form's existing `formData` state and into the submitted payload
(`add.js` → `POST /api/reviews/perfumes/add`, `edit/[id].js` → `POST /api/reviews/perfumes/update`).
On the edit form, pre-populate all three controls from the loaded review's existing values,
falling back to sensible defaults (e.g. longevity/projection unset or at their minimum, seasons
empty) when editing an older review that predates these fields.

## Done when

- `/reviews/perfumes/add` renders a longevity slider, a projection slider, and six season/time
  checkboxes. Submitting the form sends `longevity`, `projection`, and `seasons` in the POST body
  to `/api/reviews/perfumes/add`.
- `/reviews/perfumes/edit/[id]` renders the same three controls, pre-filled from the loaded
  review's `longevity`/`projection`/`seasons` (or sensible defaults if absent), and submitting
  sends the updated values to `/api/reviews/perfumes/update`.
- Manually loading `/reviews/perfumes/add` in a running dev server (`npm run dev`) and dragging the
  longevity slider to its maximum shows "8+" (not "8"); dragging the projection slider through its
  full range shows all four Fragrantica labels (Skin scent, Moderate, Strong, Beast mode) in order;
  ticking two season checkboxes (e.g. Winter and Night) and submitting persists both in the saved
  review's `seasons` array.
