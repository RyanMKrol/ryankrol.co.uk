# T024 — Local backlog dashboard (read-only)

## Do

Build a self-contained, **LOCAL-ONLY** backlog dashboard (read-only) under `.harness/`, modelled on
the backlog view of the upstream local-jobs dashboard but standalone. It must NOT become part of the
Next.js site or the Vercel build (it lives under `.harness/`, outside `src/`, so `next build` never
sees it — keep it that way).

- A **pure-Node HTTP server** (NO external npm deps; do NOT import Next/React) at
  `.harness/dashboard.js` that reads `.harness/TASKS.json`, the per-task specs
  `.harness/tasks/TNNN.md`, and the owner overlays `.harness/human-done.json` +
  `.harness/manual-fail.json` + the `.harness/worklog/<id>.md` files, and serves ONE HTML page + a
  small JSON API on a localhost port (print the `http://localhost:PORT` URL on start).
- Add an npm script `"harness:dashboard": "node .harness/dashboard.js"`.
- Put the **pure logic** — parsing TASKS.json + overlays and computing each task's derived status +
  eligibility — in a testable module `.harness/dashboard-lib.js` (with a co-located
  `.harness/dashboard-lib.test.js`). The done-ness logic MUST mirror `loop.sh`/`postflight.sh`
  `task_done`: a task is **done** if `TASKS.json` `status=="done"` **OR** `human-done.json` has
  `<id>.done==true`. Also derive: `needs-human` (gate), `blocked` (its `worklog/<id>.md` contains
  `failed:blocked`), `manual-failed` (`manual-fail.json` has the id), and **ready-now vs
  waiting-deps** (ready = not-done AND `gate==null` AND every `dependsOn` is done — same rule as
  `select_task`).

## Done when

- `npm run harness:dashboard` starts a local server, prints a `localhost` URL, and that page lists
  every task with: id, title, derived status, gate, dependsOn, facets, scope, and its spec text
  (`## Do` / `## Done when`).
- Done-ness reflects BOTH `TASKS.json` status AND `human-done.json` — e.g. **T017** (which is
  overlay-done) renders as **done**, not pending.
- Ready-now vs waiting-deps is shown and matches the loop (with the current backlog: the perfumes
  T004/T005/T006 show as waiting on their deps; the needs-human tasks show as needs-human).
- `.harness/dashboard-lib.js` is covered by `.harness/dashboard-lib.test.js` — **hermetic** (build
  synthetic TASKS.json + overlay objects in-memory; do NOT read the real files): asserts (a) a task
  with `status=="done"` OR a `human-done` entry is reported done; (b) a `gate:"needs-human"` task is
  reported needs-human; (c) ready-now = not-done + ungated + all deps done; (d) a task whose worklog
  has `failed:blocked` is reported blocked. `npm test` passes.
- It is NOT in the Vercel build: `npm run build` still succeeds and NO files were added under `src/`.
- DoD green: `npm run lint`, `npm test`, `npm run build`. (If `eslint .` flags the Node dashboard
  files, either make them lint-clean or add `.harness/` to the ignores in `eslint.config.mjs`.)
- The server is **read-only** in this task — no writes/mutations yet (those are T025).
