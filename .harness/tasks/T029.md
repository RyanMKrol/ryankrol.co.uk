# T029 — Persist Last.fm metadata on album + vinyl add

## Do

Make the album and vinyl **add** APIs accept and store the Last.fm enrichment object chosen in
the add form (see T028 / T030), mirroring how T019 persisted TMDB fields and T022 persisted Open
Library fields. We deliberately OVER-store the metadata.

- `src/pages/api/reviews/albums/add.js` — accept the Last.fm metadata in the POST body and store
  it on the `AlbumRatingsV2` item. **Populate the existing `thumbnail` field** with the Last.fm
  cover image URL (today it's always set to `''`). Also store the richer metadata under a nested
  `lastfm` object: `{ mbid, url, listeners, playcount, tags, trackCount, summary, releaseDate,
  images }` (whatever the form supplies; store what's present, omit/empty what isn't). Existing
  required fields (title, artist, rating, highlights, date) and password-gating are unchanged.
- `src/pages/api/vinyl/add.js` — accept the same Last.fm metadata and store it on the
  `VinylCollection` item: add a `thumbnail` (cover URL) field and the same nested `lastfm`
  object. The vinyl record keeps `title` + `artist` as its key. **Do NOT change the vinyl list
  page** — this task only captures the data on write. Keep the existing password gate.
- **Opportunistic fix (same file):** while editing `src/pages/api/vinyl/add.js`, fix the known
  cache-clear bug documented in `CLAUDE.md` Gotchas — it calls `clearApiCache('vinyl-collection')`
  but the read is cached under `api-vinyl-collection`; change it to
  `clearApiCache('api-vinyl-collection')` so a new vinyl entry appears immediately. Remove that
  Gotcha bullet from `CLAUDE.md` once fixed.
- Both add flows must remain **backward compatible**: if the metadata fields are absent from the
  POST body (e.g. the user skipped the search), the record still saves — `thumbnail` falls back to
  `''` and `lastfm` is omitted or empty. No field becomes newly required.
- Update the **Data model** sections in `CLAUDE.md` AND `README.md` to reflect the new
  `thumbnail`/`lastfm` fields on `AlbumRatingsV2` and `VinylCollection`, in the same commit.

## Done when

- `POST /api/reviews/albums/add` with Last.fm metadata stores `thumbnail` = the cover URL and a
  `lastfm` object on the album item; without that metadata it still saves (thumbnail `''`, no
  `lastfm`). Password gate on `RYANKROL_SITE_KEY` unchanged; non-POST → existing behaviour.
- `POST /api/vinyl/add` with Last.fm metadata stores `thumbnail` + `lastfm` on the vinyl item;
  without it, a plain title+artist record still saves. Password gate unchanged.
- `src/pages/api/vinyl/add.js` now calls `clearApiCache('api-vinyl-collection')` (the correct
  key), and the corresponding Gotcha bullet is removed from `CLAUDE.md`.
- The vinyl list page (`src/pages/vinyl/index.js`) is unchanged (not in scope).
- `CLAUDE.md` + `README.md` data-model sections document the new fields on both tables.
- `npm run lint`, `npm test`, `npm run build` all pass.
