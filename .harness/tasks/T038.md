# T038 — Perfume longevity/projection/seasons fields — backend

## Do

Add three new optional fields to the perfume review record: `longevity` (integer 0-8, where 8
represents "8+" hours — the UI will label the max as "8+", not literally "8"), `projection`
(integer 1-4, mapping to Fragrantica-style labels: 1=Skin scent, 2=Moderate, 3=Strong, 4=Beast
mode — the API only stores the integer, the label mapping is a UI concern), and `seasons` (an
array containing zero or more of the exact strings `"Winter"`, `"Spring"`, `"Summer"`, `"Autumn"`,
`"Day"`, `"Night"`).

Update `src/pages/api/reviews/perfumes/add.js` and `src/pages/api/reviews/perfumes/update.js` to
accept these three fields from the request body, validate them, and persist them on the
`PerfumeRatings` DynamoDB item alongside the existing fields (`title`, `designer`, `type`,
`description`, `rating`, `notes`, `considerTravelSize`, `considerFullBottle`). All three fields
are OPTIONAL — a request that omits them must still succeed (existing reviews created before this
change have none of these fields, and new reviews should be allowed to omit them too).

Add validators to `src/lib/perfumes.js`, mirroring the existing `validatePerfumeRating` function's
style: `validateLongevity(value)`, `validateProjection(value)`, `validateSeasons(value)`. Export
all three alongside the existing exports.

## Done when

- `src/lib/perfumes.js` exports `validateLongevity(value)` (true only for an integer 0-8),
  `validateProjection(value)` (true only for an integer 1-4), and `validateSeasons(value)` (true
  only for an array where every element is one of `"Winter"`, `"Spring"`, `"Summer"`, `"Autumn"`,
  `"Day"`, `"Night"` — an empty array is valid).
- A new test file `src/lib/perfumes.test.js` (or an addition to it if one already exists) asserts
  each validator's boundary behavior: for `validateLongevity`, 0 and 8 pass, -1 and 9 fail; for
  `validateProjection`, 1 and 4 pass, 0 and 5 fail; for `validateSeasons`, `[]` and
  `["Winter","Night"]` pass, `["Winter","NotASeason"]` and a non-array fail. `npm test --
  perfumes.test.js` passes.
- `POST /api/reviews/perfumes/add` accepts `longevity`, `projection`, and `seasons` in the request
  body and persists them on the created `PerfumeRatings` item when provided; a request that omits
  all three still succeeds (they are not in the "Missing required fields" check). A request with
  an out-of-range `longevity`/`projection` or an invalid `seasons` entry is rejected with HTTP 400.
- `POST /api/reviews/perfumes/update` accepts and persists the same three fields the same way,
  with the same validation and the same optionality.
