# T031 — Make the appearance picker reachable on every page (incl. home)

## Do

Today the 🎨 appearance trigger + `AppearancePicker` modal live inside `src/components/Header.js`,
so they're only reachable on pages that render `Header`. The homepage (`src/pages/index.js`) is
the one page that does NOT render `Header`, so the picker is unreachable there. Make the picker
**globally available on every page** by lifting it out of Header into the global layout wrapper.

- In `src/components/MatrixLayout.js` (which `_app.js` wraps around EVERY page): render the 🎨
  trigger button as a **fixed top-right control** plus the `AppearancePicker` modal, and own the
  open/close state here (the `useState` + `onClose` that currently live in Header). Import
  `AppearancePicker` here. The control must render regardless of `active` (matrix) state — it is
  not part of the matrix overlay; it's always present.
- In `src/components/Header.js`: REMOVE the 🎨 button, the `AppearancePicker` import, and the
  `pickerOpen` state. Header keeps its back-link (`← ryankrol.co.uk`) and `NowPlaying`. Pages that
  render Header must no longer show a second/duplicate appearance control.
- In `src/styles/globals.css`: add styling for the new fixed top-right control (theme-driven via
  the CSS custom properties — no hardcoded colours; correct under default AND `matrix-active`
  themes), and update/remove the now-unused `.header-appearance-btn` rules so no dead selector
  references markup that no longer exists. Ensure the fixed control has a high enough `z-index` to
  sit above page content and the CRT overlay, and is clickable everywhere.

Position: **fixed top-right corner** (familiar — it currently sits in `header-right`). Make sure it
doesn't overlap page titles awkwardly and remains tappable on mobile widths.
