## Overview
This is the deliberate, human-triggered destructive half of the BookRatingsV4 old-provider-metadata migration. T365 builds and unit-tests the migration script `src/scripts/stripOldBookProviderMetadata.js` and proves its read-only dry run; THIS task is the owner actually running the `LIVE=1` pass against production DynamoDB and verifying the result. It is `needs-human` because it is an irreversible, no-undo deletion of production data that must be run by a person exercising judgment about timing — never by the autonomous build loop. The precondition (Hardcover is confirmed callable from deployed production, so books won't be left with no rich metadata at all) was established by T364.

## Do
This task is completed interactively by the owner, not built by the loop. Steps:
1. Confirm T365 has merged to `origin/main` (the script + tests exist and CI is green) and that T364's Hardcover-from-production verification still holds.
2. Pull production env locally (`vercel env pull .env.local --environment production`) so the script talks to the real table.
3. Run the default dry run once more and read its output: `node src/scripts/stripOldBookProviderMetadata.js` — confirm the set of rows and per-row fields it reports matches expectations (as of the last dry run this was 4 rows).
4. Run the live pass for real: `LIVE=1 node src/scripts/stripOldBookProviderMetadata.js`.
5. Verify by re-scanning BookRatingsV4 afterward: confirm zero rows retain any of the target fields (`source`, `coverUrl`, `volumeId`, `bookAuthors`, `firstPublishedYear`, `isbn`, `subjects`, `pageCount`, `publisher`, `olid`, `coverId`), and that `needsBookBackfill(item)` (src/lib/backfillEligibility.js) now returns true for every row that had `volumeId`/`olid` stripped (so they're eligible for a clean Hardcover re-backfill).
6. Record what was actually stripped (row count + fields) in this task's worklog, then mark it done with `.harness/scripts/mark-done.sh T370` (per the "completing a needs-human task interactively" convention — do NOT route it through `loop.sh`).

## Done when
`LIVE=1 node src/scripts/stripOldBookProviderMetadata.js` has been run against production by the owner; a post-run re-scan of BookRatingsV4 confirms zero rows retain any of the target old-provider fields and that `needsBookBackfill` returns true for every row that had `volumeId`/`olid` removed; the outcome (rows affected + fields stripped) is recorded in `.harness/worklog/T370.md`; the task is marked done via `mark-done.sh`.
