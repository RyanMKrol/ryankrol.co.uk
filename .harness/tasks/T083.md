# T083 — Run the incomplete-workout-metrics backfill against production

## Do

T082 built `src/scripts/backfillIncompleteWorkoutMetrics.js` (dry-run by default, real writes
behind a `LIVE=1` flag) and its npm script. This step requires the owner's AWS credentials and a
judgement call on the dry-run output, so it cannot be done by the autonomous loop.

1. Run `npm run workout:backfill-metrics` (dry-run) and review the printed output: how many
   workouts would be updated, and a sanity-check of a few of the computed metrics summaries against
   what you'd expect for those workouts.
2. If the dry-run output looks reasonable, re-run with the live flag (e.g. `LIVE=1 npm run
   workout:backfill-metrics`) to actually apply the updates.
3. After running, spot-check `/workouts` in production to confirm the previously-incomplete
   workouts (e.g. the one that was crashing the page, `5c4249f9-bd32-44ff-b7a5-b3edf05303f6` at
   time of writing — though by the time this runs, check whichever workouts the dry-run flagged)
   now render with full exercise/volume/set detail instead of a zero-state.

## Done when

- The dry run has been reviewed by the owner.
- The live backfill has been run against the production `Workouts` table (or the owner has
  explicitly decided not to, e.g. because the dry run found nothing to fix by that point) — either
  way, this task is marked done once the owner has made and acted on that call.
- `/workouts` in production has been spot-checked post-backfill to confirm previously-incomplete
  workouts now show full detail.
