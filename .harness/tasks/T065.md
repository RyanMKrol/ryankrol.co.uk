# T065 — Promote the winning UI rewrite variant, delete the other two

## Do

Read the `## Decision` section T064 appended to the end of
`.harness/designs/T042-ui-rewrite-briefs.md` to find which variant (1, 2, or 3) won.

Promote the winning variant to become the site's actual, primary UI: move its pages/components
from `src/pages/v{winner}/**` and `src/components/v{winner}/**` to replace the current production
equivalents (`src/pages/reviews/**`, `src/pages/vinyl/**`, `src/pages/workouts/**`,
`src/pages/exercises/**`, `src/pages/listening/**`, `src/pages/projects/**`, `src/pages/index.js`,
and any shared layout component), updating imports/paths as needed so the promoted pages work at
their normal top-level routes (not under `/v{winner}`). Delete the two losing variants entirely
(`src/pages/v1/**`/`src/pages/v2/**`/`src/pages/v3/**` and their component directories, minus
whichever one just got promoted and moved out).

Keep the existing theming CSS-custom-property mechanism (`src/styles/globals.css`) working if the
winning variant used it, or replace it consistently if the winning variant's brief specified its
own approach — either way, the site must build and run cleanly afterward with no leftover
references to the discarded variants or the old production pages that were just replaced.

## Done when

- The winning variant's pages/components now live at the site's normal top-level routes (no longer
  under `/v{winner}`), and the two losing variants' directories no longer exist anywhere in
  `src/pages/` or `src/components/`.
- `npm run build` succeeds with no references to `/v1`, `/v2`, or `/v3` remaining anywhere in the
  codebase (`grep -rE "/v[123]" src/` returns nothing, aside from incidental unrelated matches you
  should sanity-check by hand).
- Loading the site's normal routes (`/`, `/reviews/movies`, `/vinyl`, `/workouts`, `/listening`,
  `/projects`) in a running dev server shows the promoted variant's UI, and every route that
  existed in production before this task still resolves (nothing was accidentally deleted along
  with the losing variants).
