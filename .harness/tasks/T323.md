## Overview
On /workouts, an exercise-preview `<li class="workout-session-exercise-item">` that carries one or more PrBadge pills is taller than a sibling `<li>` with plain text, because the pill (`.collection-badge` padding 0.25rem 0.75rem) is taller than the 0.85rem monospace text -- the list's `gap: 0.2rem` (`.workout-session-exercise-list`) never changes, but per-row height does, making PR-heavy workout cards look more loosely spaced.

## Do
In src/styles/globals.css, normalize `.workout-session-exercise-item` (currently ~lines 1946-1953) so every exercise row occupies the same vertical footprint whether or not it contains a `PrBadge` child -- e.g. a fixed `min-height`/consistent `line-height` matching the badge's rendered height, or `display:flex; align-items:center` with a fixed row height. The row's internal text/badge alignment can differ, but the outer row height must match across badge and non-badge rows.

## Done when
On /workouts, for a workout card whose exercise-preview list mixes PR-bearing and plain rows (e.g. a 'Pull' card with 'Iso-Lateral High Row (Machine)' carrying two PR pills next to a plain row like 'Chest Fly (Machine)'), every `.workout-session-exercise-item` in that list computes to the same rendered height regardless of whether it holds a PrBadge -- verified via computed bounding-box heights in devtools or a screenshot comparison. The list's `gap: 0.2rem` between items is unchanged.
