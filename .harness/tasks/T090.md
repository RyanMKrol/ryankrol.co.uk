# T090 — /projects: fix the wrong GitHub API endpoint, include archived repos, warn on missing token

## Do

Three related fixes to `src/pages/api/github/repos.js`, in order of importance:

1. **Fix the actual root cause: the code calls the wrong GitHub API endpoint.**
   `GET /users/{username}/repos` (what this route currently calls) is a **public-repos-only
   endpoint by design** — GitHub's API returns ONLY public repositories from this endpoint
   regardless of whether the request is authenticated or what scopes the token has. This is why
   the `/projects` page has never shown private repos, even with `GITHUB_TOKEN` set: no token can
   fix this, because the endpoint itself structurally excludes private repos. Confirmed via a live
   test: with a valid classic PAT carrying the `repo` scope, `GET
   /users/RyanMKrol/repos` still returned only 58 public repos (0 private), while `GET
   /user/repos?affiliation=owner` (the AUTHENTICATED-user endpoint — no username in the path,
   operates on whichever account the token belongs to) returned the full 62 repos including all 4
   private ones.

   Switch to `GET /user/repos` (paginated the same way, `per_page=100` looping until a short page)
   **when `GITHUB_TOKEN` is set**, passing `?affiliation=owner` so the result is scoped to repos
   the token's account OWNS (not org repos or repos merely collaborated on — this is a personal
   "my projects" page). **When `GITHUB_TOKEN` is NOT set**, keep the existing fallback to `GET
   /users/${GITHUB_USERNAME}/repos` (the current code path) since `/user/repos` requires
   authentication and has no keyless equivalent — this preserves today's degraded-but-working
   keyless behavior for local dev without a token.

2. **Include archived repos.** The current filter excludes them:
   `return !repo.fork && !repo.archived && repo.pushed_at;` — remove the `!repo.archived`
   condition (keep excluding forks). The owner has decided `/projects` should show the FULL account
   (active + archived, ~62 repos with both fixes applied), not just active ones — this supersedes
   the earlier decision behind T072 (which intentionally excluded archived repos).
3. **Warn loudly when `GITHUB_TOKEN` is absent.** Add a clear `console.warn` (matching this repo's
   convention of generous API-route logging) when `GITHUB_TOKEN` is not set, e.g. `"⚠️ [GitHub]
   GITHUB_TOKEN not set — falling back to public-only /users/{username}/repos, private repos will
   be missing from /projects"` — so this failure mode shows up in Vercel's function logs instead of
   silently under-counting forever. Do not make the token hard-required — this matches this repo's
   existing pattern for other optional API keys (e.g. `GOOGLE_BOOKS_API_KEY`'s
   fallback-with-warning behavior in `src/pages/api/books/search.js`).

## Done when

- With `GITHUB_TOKEN` set (a classic PAT with `repo` scope, or a fine-grained PAT with Contents +
  Metadata read on "All repositories"), `GET /api/github/repos` calls `/user/repos?affiliation=owner`
  (not `/users/{username}/repos`) and the response includes private repos.
- Without `GITHUB_TOKEN` set, the route still works (falls back to the public
  `/users/{username}/repos` call, matching today's behavior) and logs the warning described above.
- `GET /api/github/repos` no longer filters out `repo.archived` repos in either code path — archived
  repos with a `pushed_at` date are included (still excluding forks).
- Manually verify in a running dev server (`npm run dev`) with a valid `GITHUB_TOKEN` set in
  `.env.local`: `/projects` shows significantly more than the previous ~25-29 repos (expect close
  to the account's real total, currently 62 at time of writing — including private and archived
  ones).
- No file outside `src/pages/api/github/repos.js` was modified.
