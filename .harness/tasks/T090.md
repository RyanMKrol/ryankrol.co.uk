# T090 — /projects: show archived repos too, and warn loudly when GITHUB_TOKEN is missing

## Do

Two related fixes to `src/pages/api/github/repos.js`:

1. **Include archived repos.** The current filter excludes them:
   `return !repo.fork && !repo.archived && repo.pushed_at;` — remove the `!repo.archived`
   condition (keep excluding forks; there are none on this account currently, but the exclusion is
   still correct to keep). The owner has decided the `/projects` page should show the FULL account
   (active + archived, ~62 repos), not just active ones (~29) — this supersedes the earlier
   decision behind T072 (which intentionally excluded archived repos; that choice is now reversed).
2. **Warn loudly when `GITHUB_TOKEN` is absent.** Confirmed via a live check: without the token,
   GitHub's API only returns PUBLIC repos to an unauthenticated caller — private repos are
   completely invisible, silently under-counting the list with no indication anything is wrong.
   Add a clear `console.warn`/`console.error` (matching this repo's convention of being generous
   with logging in API routes, per the root `CLAUDE.md`) when `GITHUB_TOKEN` is not set, e.g.
   `"⚠️ [GitHub] GITHUB_TOKEN not set — only public repos are visible, private repos will be
   missing from /projects"` — so this failure mode shows up in Vercel's function logs instead of
   silently under-counting forever. Do not make the token hard-required (the route should still
   work keylessly, just with reduced visibility and a clear warning) — this matches this repo's
   existing pattern for other optional API keys (e.g. `GOOGLE_BOOKS_API_KEY`'s fallback-with-warning
   behavior in `src/pages/api/books/search.js`).

## Done when

- `GET /api/github/repos` no longer filters out `repo.archived` repos — archived repos with a
  `pushed_at` date are included in the response (still excluding forks).
- When `GITHUB_TOKEN` is unset, the route logs a clear warning identifying the cause (missing
  token → public-only visibility) rather than silently returning a shorter list with no
  explanation.
- `/projects` in a running dev server (without `GITHUB_TOKEN` set locally) shows the warning in the
  server console when the page's data is fetched.
- No file outside `src/pages/api/github/repos.js` was modified.
