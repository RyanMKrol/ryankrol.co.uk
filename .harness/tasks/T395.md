## Overview
T390 added pagination to the perfumes review page (src/pages/reviews/perfumes/index.js) matching the other four review types, using PAGE_SIZE = 12. The owner wants that lowered to 4, for perfumes only, because perfume cards (rendered via the separate perfumeVariants/Variant6Hybrid component) carry substantially more per-item detail than the simpler ReviewCard-based movies/tv/books/albums cards — designer, longevity/projection/seasons meters, application spots, Fragrantica link, description, on top of rating and review text — so a 12-item page currently feels overloaded and too dense.

## Do
In src/pages/reviews/perfumes/index.js, change the `PAGE_SIZE = 12` constant (line ~18) to `PAGE_SIZE = 4`. This is the only line that needs to change — it already feeds directly into the existing `paginate(filteredPerfumes, currentPage, PAGE_SIZE)` call (line ~111), and src/lib/pagination.js's paginate()/getPageCount() already support arbitrary page sizes, so no other logic changes are needed. Do not touch PAGE_SIZE (or equivalent) in the movies/tv/books/albums review pages — this is scoped to perfumes only. No changes needed to scripts/_visual-harness.mjs: it already has 8 synthetic perfume fixture items, which is enough to exercise 2 pages at PAGE_SIZE=4, and none of the existing perfumes FLOWS entries assert a specific page count or pagination-control state (they only waitFor '.perfume-v6-card'), so none of them break at the new page size.
