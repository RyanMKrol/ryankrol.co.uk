## Overview
Vinyl (VinylCollectionV2) has entries whose stored thumbnail URL 404s (broken album art), and vinyl today supports only READ + ADD — no edit path at all. Add a per-entry EDIT page (owner-chosen option a, mirroring the reviews edit recipe) that lets the owner re-search Last.fm for any vinyl entry and pull the correct cover / metadata.

## Do
Build option (a), a per-entry edit flow mirroring the albums reviews recipe (src/pages/reviews/albums/edit.js + edit/[id].js). Create src/pages/vinyl/edit.js (a list of vinyl records, each linking to its edit detail) and src/pages/vinyl/edit/[id].js (a detail form that re-runs a Last.fm search via the EXISTING machinery — MetadataBackfillModal or LastfmAlbumSearch, as albums' edit/[id].js does — previews the found cover, and lets the owner fix the entry's thumbnail/lastfm and title/artist). Reuse the existing /api/lastfm/album-search + /api/lastfm/album-info routes and existing components — do NOT build new search infra. Add edit links from src/pages/vinyl/index.js. Create a new POST src/pages/api/vinyl/update.js gated on process.env.RYANKROL_SITE_KEY (password-gate convention): it writes the new thumbnail (via pickLargestFromImageMap(lastfm.images), same as src/pages/api/vinyl/add.js) and the full lastfm object to the VinylCollectionV2 row by id (UpdateCommand, or GetCommand+PutCommand if the key changes), then calls clearApiCache('api-vinyl-collection') — CRITICAL: the vinyl read route caches under generateCacheKey('vinyl-collection') = key 'api-vinyl-collection', NOT 'api-vinyl'; the cleared key MUST match add.js's or writes leave stale data. The UI must tolerate 'no Last.fm match found' without forcing a bad selection (e.g. the 'Elden Ring' game soundtrack may have no good match) — the owner can leave such an entry unchanged. DELETE is OUT OF SCOPE for this task (edit/re-search only). A per-entry edit page fixes both broken-art and missing-art entries the same way, so no server-side broken-URL detection is needed. Update CLAUDE.md's file map (add the vinyl edit page(s) + api/vinyl/update.js; note vinyl now has an edit path) and README.md if the VinylCollectionV2 flow warrants — docs ship in the same change per the repo Definition of Done.

## Done when
From /vinyl the owner can reach a per-entry edit flow, open any record, re-search Last.fm, preview the found cover, and save; the save hits the password-gated src/pages/api/vinyl/update.js which persists the new thumbnail + full lastfm object to the VinylCollectionV2 row and clears the api-vinyl-collection cache (matching add.js); an entry with no Last.fm match can be left as-is without forcing a bad match; and CLAUDE.md's file map (plus README where relevant) reflect the new vinyl edit page(s) and API route. DELETE is intentionally not included.
