# T010 — Date-range filter helper + control

## Do

Add the shared, reusable pieces for a preset date-range filter (**3 months / 1 year /
all time**), used by the exercise page (T011) and the programmes dashboard (T012).

- `src/lib/dateRange.js` (pure, tested):
  - `DATE_RANGES` — the three options, e.g. `[{ key: '3m', label: '3 months', months: 3 },
    { key: '1y', label: '1 year', months: 12 }, { key: 'all', label: 'All time', months: null }]`.
  - `filterByDateRange(items, rangeKey, getDate, now = new Date())` → the items whose
    `getDate(item)` falls within the window ending at `now`; `all` returns everything.
    `now` is a parameter so tests are deterministic.
- `src/components/DateRangeFilter.js` — a small presentational segmented control:
  props `{ value, onChange }`, renders a button per `DATE_RANGES` entry, highlights the
  active one. No data logic in the component.

## Done when

- `src/lib/dateRange.js` exports `DATE_RANGES` + `filterByDateRange`, covered by
  `src/lib/dateRange.test.js` (hermetic, passing a FIXED `now`): `3m`/`1y` include only
  in-window items, `all` returns everything, boundary dates handled sanely. `npm test` passes.
- `DateRangeFilter` renders one button per range and calls `onChange(key)` on click,
  marking the active range.
- `npm run lint`, `npm test`, `npm run build` pass.
