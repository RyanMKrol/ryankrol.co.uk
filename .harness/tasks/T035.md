# T035 — Per-provider search buttons in the book add flow + persist any-provider match

## Do

Give the book add flow **multiple provider search buttons** — one per source (Open Library, Google
Books) — each searching its provider from the form's title+author and serving results into the same
select-a-match flow, then persist a selected match from EITHER provider. Builds on T034 (the
`provider`-aware `/api/books/search`).

- `src/components/BookSearch.js`: replace the single Search button with **two buttons** —
  "Search Open Library" and "Search Google Books" — each calling
  `/api/books/search?provider=<that>&title=…&author=…` using the existing `title`/`author` props
  (both buttons disabled until a title exists). Show which source the current results came from
  (a small label). In the results list, resolve each item's cover from **`coverUrl` if present,
  else `bookCoverUrl(coverId)`** (Open Library), so both providers render thumbnails. On Select,
  pass the common match (including `source`, and whichever of `coverUrl`/`coverId`,
  `volumeId`/`olid` it has) to `onSelect`. Keep the confirmed-state + Clear UI.
- `src/pages/reviews/books/add.js`: include the provider-agnostic fields in the submit body —
  `source`, `coverUrl`, and `volumeId` (in addition to the existing `olid`/`coverId`/`bookAuthors`/
  `firstPublishedYear`/`isbn`/`subjects`/`pageCount`/`publisher`). Search stays OPTIONAL.
- `src/pages/api/reviews/books/add.js`: persist the new fields when present — store `source`,
  `coverUrl`, and `volumeId` alongside the existing book fields, each only when defined (mirror the
  existing `...(field !== undefined && { field })` pattern) so a Google Books selection is saved
  with a usable cover and a Open Library selection is unaffected.
- Update the `BookRatingsV3` record-shape notes in `CLAUDE.md` AND `README.md` to mention the new
  `source` / `coverUrl` / `volumeId` fields, same commit.

Note: `ReviewCard` does NOT currently render book covers, so no card change is needed — this task
only captures + stores the data and adds the provider buttons.

## Done when

- The book add page shows two provider search buttons; clicking each returns that provider's results
  (verifiable: Google Books returns matches Open Library may miss); both read the page's title +
  author; both buttons disabled until a title is entered.
- Results from either provider render a cover thumbnail (Google via `coverUrl`, Open Library via
  `bookCoverUrl(coverId)`) and can be Selected.
- Selecting a **Google Books** result and submitting stores a book review with `source:'googlebooks'`
  + a `coverUrl`; selecting an **Open Library** result still stores as before. Submitting without
  searching still works (no field becomes required).
- `CLAUDE.md` + `README.md` document the new `source`/`coverUrl`/`volumeId` fields on `BookRatingsV3`.
- Manually verify against the running app: `/reviews/books/add` → type title + author → click
  "Search Google Books" → results appear → Select → submit succeeds and the stored item has the
  Google cover/source.
- `npm run lint`, `npm test`, `npm run build` all pass.
