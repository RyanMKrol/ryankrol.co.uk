# T098 — Required Fragrantica URL field on the perfume add + edit forms

## Do

Add a new REQUIRED field `fragranticaUrl` to the perfume review: a plain reference link to that
perfume's page on Fragrantica (a fragrance database/community site) — stored and shown as a
clickable link, not fetched/scraped for any additional data (this is purely a bookmark field, not
an external-metadata-lookup integration like TMDB/Google Books/Last.fm).

- **Backend**: `src/pages/api/reviews/perfumes/add.js` and `update.js` must require
  `fragranticaUrl` in the request body (alongside the existing required fields — title, designer,
  type, rating), validating it's a syntactically valid URL (e.g. starts with `http://` or
  `https://` and parses via the JS `URL` constructor without throwing — no need to restrict it to
  the fragrantica.com domain specifically, just require it to be a well-formed URL). Reject with
  HTTP 400 if missing or malformed. Add a `validateFragranticaUrl(value)`-style validator to
  `src/lib/perfumes.js` mirroring the existing validator style, and persist the field on the
  `PerfumeRatings` item.
- **Frontend**: add a required "Fragrantica URL" text input to both
  `src/pages/reviews/perfumes/add.js` and `src/pages/reviews/perfumes/edit/[id].js`, wired into
  each form's `formData` state and submit payload, with the standard HTML `required` attribute for
  client-side enforcement. On the edit form, pre-populate it from the loaded review's existing
  value.

Since this field is now required, existing perfume reviews created before this task won't have a
`fragranticaUrl` value — that's fine, no backfill/migration needed here; the edit form should just
show an empty field for those until the owner fills it in and re-saves (which the edit form's own
`update` submission will then satisfy the new required check going forward).

## Done when

- `POST /api/reviews/perfumes/add` and `POST /api/reviews/perfumes/update` both reject a request
  missing `fragranticaUrl`, or with a malformed (non-URL) value, with HTTP 400 — alongside the
  existing required-field checks.
- `src/lib/perfumes.js` exports a validator for this field and a test exercises it (valid URL
  passes, missing/malformed value fails). `npm test` passes.
- Both `/reviews/perfumes/add` and `/reviews/perfumes/edit/[id]` show a required "Fragrantica URL"
  input; submitting either form without it is blocked client-side, and submitting with a valid URL
  persists it.
- Manually verify in a running dev server (`npm run dev`): add a new perfume review with a
  Fragrantica URL, then reload its edit page and confirm the URL is shown and editable.
