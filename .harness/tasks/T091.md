# T091 — Fix transparent background on the metadata backfill modal

## Do

`src/components/MetadataBackfillModal.js` sets its results panel's background with
`background: var(--color-background)` — but `--color-background` is not a defined CSS custom
property anywhere in `src/styles/globals.css` (confirmed: the file defines `--color-bg`, not
`--color-background`, and a repo-wide check confirms this is the ONLY CSS variable reference in
this component that doesn't match an actual defined variable — `--color-accent` and `--color-text`,
also used in this file, are both correctly defined). Since the variable doesn't exist and there's
no fallback value, the panel's background resolves to nothing, making it effectively transparent —
this is the modal used for the backfill-from-API feature across all four review types (books,
albums, movies, tv — T068-T071).

Change `var(--color-background)` to `var(--color-bg)` at the relevant line in
`MetadataBackfillModal.js`. Do not touch the modal's separate dimming backdrop overlay
(`.mbm-overlay`, `rgba(0, 0, 0, 0.6)`) — that element already works correctly; this fix is only for
the inner results panel's own background.

## Done when

- `MetadataBackfillModal.js` no longer references `var(--color-background)` anywhere —
  `grep -c "color-background" src/components/MetadataBackfillModal.js` returns 0.
- Manually verify in a running dev server (`npm run dev`): open the backfill modal on any review
  type's edit page (e.g. books), and confirm the results panel has a solid, opaque background
  matching the site's theme — no underlying page content bleeding through behind it.
- No other file was modified.
