# T002 — Register perfumes table + create-table script

## Do

Set up the DynamoDB table definition for a new `perfumes` review type (do NOT run
it here — provisioning is T003).

- Add `PERFUME_RATINGS_TABLE: 'PerfumeRatings'` to `DYNAMO_TABLES` in
  `src/lib/constants.js`.
- Add `src/scripts/createPerfumeTable.js`, mirroring
  `src/scripts/createWorkoutTables.js`: create the `PerfumeRatings` table (region
  `us-east-2`, credentials from env) with a **single String partition key `id`** (no
  sort key, no GSIs), waiting for status ACTIVE and logging the result.
- Add an npm script `"perfume:create-table": "node src/scripts/createPerfumeTable.js"`
  to `package.json`.

The `id` is a synthetic key because a perfume's `title` is not unique on its own —
e.g. "Ombré Leather" by Tom Ford exists as both an EDP and a Parfum. `id` will be
generated from title+designer+type (the generator is built in T004).

## Done when

- `DYNAMO_TABLES.PERFUME_RATINGS_TABLE === 'PerfumeRatings'`.
- `src/scripts/createPerfumeTable.js` exists, uses the AWS SDK v3 client like
  `createWorkoutTables.js`, and defines the table with partition key `id` (String).
- `npm run perfume:create-table` is wired in `package.json`.
- `npm run lint` and `npm run build` pass. (The script is NOT executed in this task.)
