# T068 — Backfill-from-API button on the books edit page

## Do

On `src/pages/reviews/books/edit/[id].js`, add a "Backfill from Google Books" button using the
shared `MetadataBackfillModal` component from T066. Wire it so:
- The search is seeded from this form's current `title` (and `author` if present) fields, calling
  `GET /api/books/search?title=...&author=...&provider=googlebooks` (the existing search endpoint
  — see `src/pages/api/books/search.js`; reuse `src/components/BookSearch.js`'s calling pattern as
  a reference for how to call it, though you don't have to reuse that component directly).
- On confirm, the selected result's metadata fields (`source, coverUrl, volumeId, bookAuthors,
  firstPublishedYear, isbn, subjects, pageCount, publisher` — see `mapGoogleBooksResult` in
  `src/lib/googlebooks.js` for the exact shape a result has) overwrite this form's corresponding
  fields, per the "overwrite all metadata fields" behavior T067 now supports on
  `POST /api/reviews/books/update`.
- **NEVER touch `rating` or the `overview` field** (this form's `overview` field is the user's OWN
  written review text, not sourced from Google Books — Google Books results don't even provide a
  synopsis field per `mapGoogleBooksResult`'s shape, so there is no ambiguity here, just don't wire
  anything into `overview` from the modal).
- The existing Save/Delete flow for this edit page is otherwise unchanged — the backfill button
  just pre-fills form fields; the user still clicks the page's normal Save button afterward to
  persist (do not auto-submit on confirm).

## Done when

- Loading `/reviews/books/edit/[id]` for an existing book review, clicking "Backfill from Google
  Books", picking a search result, and confirming updates the form's title/author/cover/etc.
  fields visibly on the page, without touching the rating or the review-text field.
- Clicking the page's existing Save button afterward persists the backfilled fields via
  `POST /api/reviews/books/update` (confirm by reloading the edit page and seeing the new values
  still present).
- No file outside `src/pages/reviews/books/edit/[id].js` and `src/components/**` was modified.
