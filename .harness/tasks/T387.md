## Overview
The home-page 'This year at the gym' heading (src/pages/index.js:236) is a <Link href="/workouts" className="home-gym-panel-title home-gym-panel-link">. It sits inside .home-gym-panel, which has a dark background (var(--color-ink)) and white text (color: #FFFFFF). .home-gym-panel-title itself sets no color (font/letter-spacing/uppercase/opacity/margin only), so a plain heading would inherit the panel's white. But .home-gym-panel-title.home-gym-panel-link (globals.css ~line 3432) sets only margin/text-decoration/cursor and never sets color — so as an anchor element it falls back to the browser/global default link colour, including the :visited purple once clicked. There is no markup/JS change needed; this is CSS-only.

## Do
In src/styles/globals.css, add an explicit `color` declaration to `.home-gym-panel-title.home-gym-panel-link` that pins it to the panel's normal white heading colour (matching what .home-gym-panel-title/.home-gym-panel would render with no anchor override — i.e. inherit or explicitly #FFFFFF, whichever matches the existing panel text colour convention used elsewhere in this stylesheet). Apply that same color across all link pseudo-states so none of them can diverge: the base rule plus explicit `:link`, `:visited`, `:active` selectors (all chained off `.home-gym-panel-title.home-gym-panel-link`). Leave the existing `:hover` rule's `text-decoration: underline` behaviour as-is (do not remove the hover underline) — just ensure `:hover` also keeps the same pinned color if it doesn't already inherit it. Do not touch src/pages/index.js; no className or markup change is required.
