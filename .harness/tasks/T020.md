# T020 — TMDB search + confirm step in the movie/TV add flow

## Do

Add a TMDB search-and-confirm step to the movies and TV **add** pages so the owner can
attach the correct TMDB match before saving.

- Build a reusable `src/components/TmdbSearch.js`: given a title + media type, it calls
  `GET /api/tmdb/search` (T018), lists ALL results showing each result's **title, short
  overview, and poster thumbnail** (via `tmdbPosterUrl`), and lets the owner **select one**
  to confirm the match (with a clear "this is the one" selection + the ability to clear /
  re-search). Surfacing zero results is handled gracefully.
- Wire it into `src/pages/reviews/movies/add.js` (type `movie`) and
  `src/pages/reviews/tv/add.js` (type `tv`): the chosen match's
  `{ tmdbId, mediaType, posterPath, overview, date }` is included in the POST to the add
  API (T019). Saving WITHOUT picking a match must still work (TMDB fields optional).

## Done when

- On both add pages, entering a title and searching shows the TMDB results (title +
  overview + poster thumbnail); selecting one marks it as the confirmed match.
- Saving includes the selected TMDB metadata in the POST; saving without a selection still
  succeeds.
- `npm run lint`, `npm test`, `npm run build` pass; manually search on the movies add page,
  pick a match, save, and confirm the metadata is stored (e.g. via the API/DynamoDB).
