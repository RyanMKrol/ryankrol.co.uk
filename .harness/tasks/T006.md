# T006 — Perfume seed-import script (run later, in T007)

## Do

Add a one-off import script `src/scripts/importPerfumeSeed.js` that loads the owner's
existing fragrance spreadsheet into the `PerfumeRatings` table. (Do NOT run it here —
running it is T007, and it needs AWS creds + the local gitignored TSV.)

- Read the tab-separated `.harness/perfume-seed-data.tsv` (columns: Perfume, Designer,
  Type, Description, Rating, Notes, "Consider a Travel Size?", "Consider a Full Bottle?").
- Map each row to a perfume record, reusing helpers from `src/lib/perfumes.js`:
  `perfumeId({title,designer,type})` for the key, and a new pure
  `parsePerfumeSeedRow(row)` that maps a raw row → record, **rounding the 0–10 decimal
  rating to the nearest integer** (e.g. 6.5 → 7; blank → null/omit) and mapping the
  `TRUE`/`FALSE` columns to `considerTravelSize` / `considerFullBottle` booleans.
- Upsert each record via `docClient` `PutCommand` (idempotent — re-running overwrites by `id`).
- Wire an npm script `"perfume:seed": "node src/scripts/importPerfumeSeed.js"`.

## Done when

- `src/scripts/importPerfumeSeed.js` exists and `npm run perfume:seed` is wired.
- `parsePerfumeSeedRow` lives in `src/lib/perfumes.js` and is covered by a test in
  `src/lib/perfumes.test.js` (hermetic — synthetic rows, no real file/DB): a 6.5 rating
  rounds to 7, a blank rating yields no/blank rating, `TRUE`/`FALSE` map to booleans, and
  the record carries a stable `id`. `npm test` passes.
- `npm run lint`, `npm test`, `npm run build` pass. (Not executed against the real DB here.)
