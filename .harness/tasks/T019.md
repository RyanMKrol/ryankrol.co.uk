# T019 — Persist TMDB metadata on movie/TV reviews

## Do

Widen the movies and TV review **add** API routes to accept and store the TMDB metadata
chosen in the add flow (the UI lands in T020).

- `src/pages/api/reviews/movies/add.js` and `src/pages/api/reviews/tv/add.js`: accept the
  optional fields `tmdbId`, `mediaType`, `posterPath`, `tmdbOverview`, `tmdbDate` in the
  request body and include them in the `PutCommand` record (alongside the existing
  `title`, `rating`, `review_text`, `date`).
- Keep the password gate (`RYANKROL_SITE_KEY`), the existing rating validation, and the
  `clearApiCache('api-movies')` / `clearApiCache('api-tv')` calls unchanged.
- The TMDB fields are OPTIONAL — a review saved without a TMDB match must still succeed
  (fields simply absent/null).

## Done when

- Both add routes persist `tmdbId`, `mediaType`, `posterPath`, `tmdbOverview`, `tmdbDate`
  when provided, and still succeed when they're omitted.
- Password gate + rating validation + cache-clear behaviour are unchanged.
- `npm run lint`, `npm test`, `npm run build` pass.
