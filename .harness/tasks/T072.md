# T072 — Show the full repo list on /projects (remove the top-20 cap)

## Do

`src/pages/api/github/repos.js` currently fetches only a single, unpaginated page from
`https://api.github.com/users/${GITHUB_USERNAME}/repos` (GitHub's default page size applies, no
`per_page`/`page` params are set), then hard-caps the result with `.slice(0, 20)` (line ~55,
labeled "Top 20 most active"). Remove this cap, and properly paginate the GitHub API call itself
so every one of the user's repos is fetched, not just the first page: request with
`per_page=100` and follow subsequent pages (`page=2`, `page=3`, ...) until a page returns fewer
than `per_page` results (or an empty array), concatenating all pages before applying the existing
filter/sort logic.

Keep the existing filter (excluding forks and archived repos — `!repo.fork && !repo.archived &&
repo.pushed_at`) and the existing sort (most-recent-push first) exactly as they are; only the
"one page" + ".slice(0, 20)" limits are being removed. The response shape
(`{ repos, total, username }`) and the existing `withApiCache` caching wrapper stay unchanged.

`src/pages/projects/index.js` (the page rendering these repos) already just renders whatever
`data.repos` array it receives with no additional client-side limiting — no changes needed there;
it will automatically show the full list once the API returns it.

## Done when

- `GET /api/github/repos` returns every one of the account's non-fork, non-archived repos with a
  `pushed_at` date (no arbitrary count cap), correctly paginating the underlying GitHub API calls
  rather than only reading GitHub's default first page.
- `/projects` in a running dev server shows repo cards for the full uncapped list (spot-check:
  more than 20 cards render for an account with more than 20 qualifying repos).
- The existing sort order (most-recent-push first) and filter (no forks, no archived) are
  unchanged.
- No file outside `src/pages/api/github/repos.js` was modified (this task's fix is entirely
  backend/pagination — the frontend needs no change).
