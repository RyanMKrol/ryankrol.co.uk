# T021 — Open Library search proxy route + normaliser

## Do

Add a server-side Open Library (openlibrary.org) book-search proxy + a tested normaliser.
Open Library is keyless (no API key / no env var needed).

- `src/pages/api/books/search.js` (GET) — param `query` (title, optionally author); calls
  the Open Library search API server-side; returns a normalised result list. Wrap in
  `withApiCache` (`generateCacheKey('book-search', { query })`).
- Put the pure normalisation in `src/lib/openlibrary.js`:
  - `mapBookResult(doc)` → `{ olid, title, authors, firstPublishedYear, coverId, isbn,
    subjects, pageCount, publisher }` (the **rich set**) from an Open Library search `doc`
    (e.g. `key`/`cover_edition_key` → olid, `author_name` → authors, `first_publish_year`,
    `cover_i` → coverId, `isbn`, `subject`, `number_of_pages_median`, `publisher`).
  - `bookCoverUrl(coverId, size)` → a `covers.openlibrary.org` image URL.
- Document the new `/api/books/search` route in CLAUDE.md's API map (no new env var).

## Done when

- `GET /api/books/search?query=...` returns a normalised array of book matches (the rich
  field set above), cached via `withApiCache`. No API key required.
- `src/lib/openlibrary.js` is covered by `src/lib/openlibrary.test.js` against a synthetic
  Open Library response (hermetic — no live call): maps title/authors/year/coverId/isbn and
  the rich fields; a doc missing a cover yields null `coverId`; `bookCoverUrl` builds the
  expected URL. `npm test` passes.
- CLAUDE.md API map mentions the route.
- `npm run lint`, `npm test`, `npm run build` pass.
