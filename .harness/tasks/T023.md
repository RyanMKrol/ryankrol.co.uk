# T023 — Open Library search + confirm step in the book add flow

## Do

Add an Open Library search-and-confirm step to the books **add** page, mirroring the TMDB
flow on the movie/TV add pages (T020).

- Build `src/components/BookSearch.js`: given a title (and optional author), it calls
  `GET /api/books/search` (T021), lists ALL results showing each result's **title,
  author(s), first-published year, and cover thumbnail** (via `bookCoverUrl`), and lets the
  owner **select one** to confirm the match (clear selection + re-search supported; zero
  results handled gracefully).
- Wire it into `src/pages/reviews/books/add.js`: the chosen match's rich metadata
  (`olid`, `coverId`, `bookAuthors`, `firstPublishedYear`, `isbn`, `subjects`, `pageCount`,
  `publisher`) is included in the POST to the add API (T022). Saving WITHOUT picking a
  match must still work (fields optional).

## Done when

- On the books add page, entering a title and searching shows the Open Library results
  (title + author + year + cover thumbnail); selecting one marks it as the confirmed match.
- Saving includes the selected book metadata in the POST; saving without a selection still
  succeeds.
- `npm run lint`, `npm test`, `npm run build` pass; manually search on the books add page,
  pick a match, save, and confirm the metadata is stored.
