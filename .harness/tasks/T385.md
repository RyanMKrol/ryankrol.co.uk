## Overview
The per-session volume number shown on each workout card on the /workouts page understates the true volume: it is recomputed client-side and explicitly EXCLUDES warm-up sets, while every other volume surface (the stored Workouts.totalVolume, the homepage TOTAL VOLUME / BEST SESSION VOL, and the /workouts programme-stats Total Volume) already INCLUDES warm-up volume. The user wants warm-up volume represented in the card figure so all surfaces agree. This is the direct fix for the user's observation. Investigation notes (verified against current code): the stored totalVolume is computed in src/lib/workoutMetrics.js calculateWorkoutMetrics by summing weight*reps across EVERY set with no warmup filter; getWorkoutStats in src/lib/workoutQueries.js and aggregateProgramme in src/lib/programmeStats.js both read that stored value, so homepage + programme-stats are already warmup-inclusive and need no change. The mismatch lives ONLY in the card's own local getTotalVolume. Note: src/components/WorkoutCard.js has the same warmup-excluding guard but is DEAD CODE (imported nowhere) and does not need touching for this fix.
