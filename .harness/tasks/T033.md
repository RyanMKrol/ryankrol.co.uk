# T033 ‚Äî Drive the book search from the title + author fields (title+author matching)

## Do

Make the book search/confirm step read the **title AND author the user already typed** instead of
having its own editable search box, and use BOTH for matching (Open Library matches far better with
a dedicated author than with a single free-text query). Today `src/components/BookSearch.js` has its
own `<input>` (seeded once from `initialQuery`) and `src/pages/api/books/search.js` only accepts a
single `query` param.

- `src/pages/api/books/search.js`: accept `title` (required) and `author` (optional) query params.
  Build the Open Library request using its dedicated fields ‚Äî `title=<title>` and, when present,
  `author=<author>` (instead of the single `q=`) against `https://openlibrary.org/search.json`.
  Include both in the `generateCacheKey('book-search', { title, author })` key. A missing `title` ‚Üí
  400. Keep `mapBookResult`, the `limit`, logging, and `withApiCache` wrapping. (You may keep `q`
  support as a fallback, but title/author is the path the UI uses.)
- `src/components/BookSearch.js`: REMOVE the standalone search `<input>` and the internal `query`
  state / `initialQuery` prop. Accept `title` and `author` props (the live form fields). The
  **"Search"** button calls `/api/books/search?title=‚Ä¶&author=‚Ä¶` using those props. Keep the
  results list, Select, confirmed-state, and Clear UI. Disable the Search button when `title` is
  empty/whitespace. Drop `handleKeyDown` (no input to press Enter in).
- `src/pages/reviews/books/add.js`: pass `title={formData.title}` and `author={formData.author}` to
  `BookSearch` (NOT `initialQuery`). The `onSelect`/`bookMatch` submit body stays unchanged. Search
  remains OPTIONAL ‚Äî submitting without searching still works.
- Update the API table in `CLAUDE.md` for the new `title`/`author` params on `/api/books/search`.

NOTE for the follow-up: idea #6 may swap/augment the book provider behind this endpoint. Keep the
component's contract provider-agnostic (it sends title+author to `/api/books/search` and renders the
normalised result list), so a later provider change doesn't require touching the component.

## Done when

- `GET /api/books/search?title=Dune&author=Herbert` returns title+author-matched results; a request

- `src/components/BookSearch.js` no longer renders its own text input; it searches using `title` +

- Manually verify against the running app: add-book page ‚Üí type title + author ‚Üí Search ‚Üí results ‚Üí
  Select ‚Üí confirmed chip; submit succeeds.
