## Do

This is the final **cross-cutting visual audit** — after T165-T168 have each polished their own
page cluster in isolation, this task checks the site holds together as ONE coherent design system
rather than 4 independently-tweaked clusters that drifted from each other, plus checks the
add/edit forms restyled in T162-T164 (which had no explicit design mock and were built by inference),
plus a regression check on the Matrix easter egg.

**Process:**
1. `npm run dev`. Open every page in the site (home, all 5 reviews lists + their add/edit forms,
   vinyl + its add form, listening, projects, workouts list + detail, exercise detail) and check for
   CONSISTENCY rather than re-comparing each against its render (that already happened in
   T165-T168):
   - Typography scale: are display headings consistently `Bricolage Grotesque`, body text
     consistently `Nunito`, labels/mono consistently `Space Mono`, across every page — no
     stray leftover fonts from the old theme system?
   - Color tokens: is every accent color pulled from the CSS custom properties defined in T142, with
     no hardcoded hex values reintroduced by any of T149-T164?
   - Spacing/radius/shadow: do cards, pills, and cover tiles use the same `--radius-*`/
     `--shadow-cover` tokens everywhere, or did some page drift to a slightly different radius?
   - The 6 review/content-type add/edit forms (T162-T164) specifically — since they had no explicit
     design mock, confirm they FEEL consistent with their sibling list pages (same card style,
     button treatment, pill styling) rather than looking like a bolted-on legacy form in an otherwise
     redesigned site.
2. **Matrix-mode regression check**: trigger the Konami code (↑↑↓↓←→←→BA) on several different
   rebuilt pages (home, a reviews list, a workout page) and confirm the green code-rain + CRT overlay
   still activates correctly and doesn't visually clash or break on the new markup/CSS structure.
   Reload while active and confirm it persists (sessionStorage restore, verified back in T141, must
   still work after everything built on top of it).
3. Fix any inconsistencies found by updating the SHARED tokens/components (not per-page patches) so
   the fix propagates everywhere automatically.

## Done when

- Typography, color, spacing, radius, and shadow are consistent across every page in the site — no
  page uses a hardcoded value where a shared token exists.
- The 6 restyled forms feel visually consistent with their sibling list pages.
- Matrix mode still activates and persists correctly on every page type after the full redesign.
- `npm run lint`, `npm test`, and `npm run build` all pass.
- Document in your worklog which inconsistencies you found and fixed, and confirm the Matrix
  regression check passed on at least 3 different page types.
