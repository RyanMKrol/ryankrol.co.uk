# T026 — Proportional time x-axis on all date charts

## Do

Make every chart with a date x-axis space its points **proportionally to elapsed time** (a real
chart.js `time` scale) instead of equal-width category slots. Today the charts use category labels
(formatted date strings like "Mon DD"), so a 1-day gap and a 2-month gap look identical — misleading
for reading progress over time.

- Add the **`chartjs-adapter-date-fns`** dependency (plus `date-fns`) to `package.json` and register
  it so chart.js `time` scales work.
- Convert these three components to a **`time`** x-axis with `{ x: <ms-timestamp>, y: <value> }` data
  points (replace the parallel labels-array + category axis): `src/components/ExerciseProgressCharts.js`,
  `src/components/CardioProgressCharts.js`, `src/components/ProgrammeOverviewCharts.js`. Points are
  built from each row's date (`workout_date` for the exercise charts; the programme sessions' dates
  for the programme chart).
- Put the pure date→point mapping in a small shared helper `src/lib/chartTime.js` (e.g.
  `toTimeSeries(rows, getDate, getValue)` → ascending-by-date `{x,y}[]` with numeric `x` timestamps)
  plus a shared time-scale options fragment, and unit-test the mapping.
- Keep theming via `useChartTheme()` intact — colours/fonts unchanged; ONLY the x-axis type + the
  data shape change.

## Done when

- All three charts render with a **time** x-axis where the horizontal distance between two points is
  proportional to the real elapsed time between their dates. Verify on `/exercises/<name>` (strength
  + cardio) and `/programmes`: two far-apart sessions are visibly farther apart than two close ones.
- `chartjs-adapter-date-fns` is added + registered; `npm run build` succeeds.
- `src/lib/chartTime.js` exports the mapping helper, covered by `src/lib/chartTime.test.js`
  (**hermetic** — synthetic rows, no real data): asserts dates map to `{x,y}` points sorted ascending
  by date, `x` is a numeric timestamp, out-of-order input is sorted, and a missing/blank date is
  dropped. `npm test` passes.
- Theming still works: the charts read colours via `useChartTheme()` and recolour on a theme switch.
- DoD green: `npm run lint`, `npm test`, `npm run build`. If you record the "date charts use a chart.js
  time scale" convention, do it in `CLAUDE.md` (in scope).
