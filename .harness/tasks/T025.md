# T025 — Local backlog dashboard: interactive owner actions

## Do

Add interactive owner actions to the local backlog dashboard (`.harness/dashboard.js` + its UI from
T024). Because the dashboard is a **local** server with full filesystem + git access (it is NOT
deployed), its buttons can really mutate local state:

- A **"Mark done"** button on `needs-human` tasks → writes `.harness/human-done.json`
  (`{ "<id>": { "done": true, "at": <iso8601> } }`) and commits + pushes `[skip ci]` under the loop's
  repo lock, using the SAME pattern as `.harness/mark-failed.sh`: either add a small
  `.harness/mark-done.sh` mirroring `mark-failed.sh` (source `loop.sh` with `LOOP_SOURCE_ONLY=1` to
  reuse `acquire_lock`/paths) and have the server shell out to it, OR do the write+commit inline in
  the server under `acquire_lock`. (Marking done applies ONLY to `needs-human` tasks, mirroring the
  upstream rule.)
- A **"Mark failed"** button on `done` tasks → shells out to the existing
  `.harness/mark-failed.sh <id> "<reason>"` (prompt the owner for the reason in the UI).
- After either action, the view refreshes to reflect the new state.
- **Gate the mutating endpoints to localhost only** — refuse any non-loopback request.

## Done when

- Clicking **"Mark done"** on a `needs-human` task writes that id into `.harness/human-done.json` with
  `done:true` and produces a git commit (verify: the overlay file changes + a commit lands); the
  dashboard then shows the task as done and its dependents become ready.
- Clicking **"Mark failed"** on a `done` task runs `mark-failed.sh` (writes `.harness/manual-fail.json`
  + commits); the dashboard then shows the task as manual-failed.
- The mutating endpoints **reject non-localhost requests** (e.g. a request with a non-loopback
  `Host`/remote address gets a 403).
- These are REAL local mutations (files + git) — confirm by running `npm run harness:dashboard`,
  clicking an action, and checking `git log` + the overlay file changed. (Use a throwaway/undo where
  possible, e.g. `mark-failed.sh --undo`, so the smoke test leaves the backlog clean.)
- DoD green: `npm run lint`, `npm test`, `npm run build`. Any pure logic added gets a co-located test.
