# T096 — Remove the "Application notes" field from the perfume schema — backend

## Do

Now that "Application spots" (checkboxes, plus per-spot spray counts from T094/T095) exist, the
separate free-text "Application notes" field (the `notes` field, currently sent/persisted by
`src/pages/api/reviews/perfumes/add.js` and `update.js`) is being retired entirely. Remove `notes`
handling from both API routes: stop reading it from the request body, stop persisting it on the
`PerfumeRatings` item, and remove any validation tied to it.

**Do NOT touch a different, separate field on this same form** — perfumes also have a
`description` field (the general review-text field, "Share your thoughts about this perfume...")
which is completely unrelated to `notes`/"Application notes" and must be left untouched.

Existing reviews that already have a `notes` value stored keep it in DynamoDB (no migration/data
deletion here) — it simply stops being read or written by the app going forward.

## Done when

- `src/pages/api/reviews/perfumes/add.js` no longer reads, validates, or persists a `notes` field.
- `src/pages/api/reviews/perfumes/update.js` no longer reads, validates, or persists a `notes`
  field.
- The `description` field (a different field) is completely unaffected — still required/handled
  exactly as before.
- Any existing test asserting `notes` handling on these routes is removed/updated accordingly;
  `npm test` passes.
