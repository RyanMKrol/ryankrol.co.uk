## Overview
src/components/Markdown.js passes remarkPlugins={[remarkGfm]} to ReactMarkdown in both its inline and block-rendering paths, which enables GFM tables for any Markdown-rendered content site-wide (review write-ups, perfume descriptions, hot takes, the homepage Top-of-mind note). src/styles/globals.css has no table/thead/tbody/tr/th/td selectors anywhere — the only near-miss, `.exercise-sessions-table`, is an unrelated hand-built component table (used by src/pages/exercises/[exerciseName].js), not Markdown output. So a Markdown table typed into any real content field today would render with unstyled browser-default borders/padding instead of the Collection design tokens used everywhere else. This is latent, not actively broken (no UI nudges a user toward typing a table — MarkdownEditor's toolbar only has Bold/Italic/List/Link), but should be fixed proactively rather than discovered later on live content.
