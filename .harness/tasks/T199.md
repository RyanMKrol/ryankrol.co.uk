## Do

In each of the 5 files below, make the page's `searchTerm` state seed itself from an incoming `?q=` URL query param (now emitted by the home page's collection-wall tile links — see the dependency task), instead of always starting as `''`.

Files (all 5 currently follow the identical pattern — `const [searchTerm, setSearchTerm] = useState('')` with no router awareness):
- `src/pages/reviews/movies/index.js`
- `src/pages/reviews/tv/index.js`
- `src/pages/reviews/books/index.js`
- `src/pages/reviews/albums/index.js`
- `src/pages/vinyl/index.js`

In each file:
1. Import `useRouter` from `next/router` (already the established pattern elsewhere in this repo, e.g. `src/pages/reviews/movies/add.js`, `src/pages/vinyl/add.js`).
2. Call `const router = useRouter();` inside the component.
3. Add a `useEffect` that seeds `searchTerm` once the router has hydrated and a `q` param is present:
```js
useEffect(() => {
  if (router.isReady && typeof router.query.q === 'string') {
    setSearchTerm(router.query.q);
  }
}, [router.isReady, router.query.q]);
```
   This is necessary (rather than using `router.query.q` directly as the `useState('')` initializer) because none of these pages use `getServerSideProps`/dynamic route segments, but `router.query` is still empty (`{}`) on the very first client render until `router.isReady` flips true post-hydration — seeding via a `useState` initializer would only ever capture the empty value. Do NOT gate this behind any dynamic-route-specific logic; `router.isReady` works the same way for these static routes.
4. Do not change how `searchTerm`/`setSearchTerm` is otherwise wired to `SearchInput` or used for filtering — this task only changes how the state is *initialized*, not the filtering logic, sort logic, or any other existing behaviour. The existing substring-match filtering (which can surface multiple partial matches, e.g. a franchise with several entries) is left as-is; auto-scroll/highlight-to-exact-match is explicitly OUT of scope for this task (a possible future nicety, not required for 'see the thing you clicked' to be satisfied by a pre-filled, normally-filtering search box).
