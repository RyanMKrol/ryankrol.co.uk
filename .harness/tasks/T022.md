# T022 — Persist Open Library metadata on book reviews

## Do

Widen the books review **add** API route to accept and store the chosen Open Library
metadata (the UI lands in T023).

- `src/pages/api/reviews/books/add.js`: accept the optional fields `olid`, `coverId`,
  `bookAuthors`, `firstPublishedYear`, `isbn`, `subjects`, `pageCount`, `publisher` in the
  request body and include them in the `PutCommand` record (alongside the existing `title`,
  `author`, `rating`, `review_text`, `date`).
- Keep the password gate (`RYANKROL_SITE_KEY`), the existing rating validation, the
  preserved-date behaviour, and `clearApiCache('api-books')` unchanged.
- The Open Library fields are OPTIONAL — a book saved without a match must still succeed.

## Done when

- The books add route persists the Open Library fields when provided, and still succeeds
  when they're omitted.
- Password gate + rating validation + cache-clear behaviour are unchanged.
- `npm run lint`, `npm test`, `npm run build` pass.
