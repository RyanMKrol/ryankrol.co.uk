## Overview
T389's markdown-rendering audit found that every review add/edit form (movies, tv, books, albums, perfumes) and both hot-takes forms render <MarkdownEditor> (raw textarea + toolbar) but never a live <Markdown> preview underneath, unlike src/pages/tom.js which pairs the two. Users can't see how bold/italic/lists/links will render until after saving. This task adds the same preview pane, mechanically, to all 12 forms.

## Do
In each of the 12 files listed in scope, import the Markdown component (`import Markdown from '<relative-path-to>/components/Markdown'`, matching the file's existing relative-import depth for MarkdownEditor) and add a new `collection-form-group` immediately after the existing group that contains `<MarkdownEditor>`, mirroring src/pages/tom.js:126-129 exactly:
```jsx
<div className="collection-form-group">
  <label className="collection-form-label">Preview</label>
  <Markdown>{formData.<field>}</Markdown>
</div>
```
Bind `<field>` to the single markdown-carrying state key already wired to that file's MarkdownEditor -- `gist` for movies/tv, `overview` for books, `highlights` for albums, `description` for perfumes, `text` for hot-takes. Always render the preview (no toggle/show-hide control, matching tom.js). Do not alter MarkdownEditor's own props/behaviour, form submission logic, or any other field. Do not touch src/pages/tom.js itself (already has this). After editing, update scripts/_visual-harness.mjs's manifest coverage only if a new FLOWS-worthy interactive state is introduced (it isn't -- the preview is static, not a new interaction), so no _visual-harness.mjs change is expected; just re-run scripts/visual-check.mjs and inspect the add/edit shots for these 12 pages.

## Done when
All 12 files in scope import Markdown and render a 'Preview' collection-form-group directly after their MarkdownEditor group, bound to that file's existing markdown field, always visible with no toggle. `npm run lint`, `npm test`, and `npm run build` all stay green. `node scripts/visual-check.mjs` is run and the add/edit screenshots for movies, tv, books, albums, perfumes, and hot-takes are visually confirmed to show the rendered preview text beneath the editor toolbar.
