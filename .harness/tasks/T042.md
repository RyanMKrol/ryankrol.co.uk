# T042 — Author three distinct full-site UI/IA rewrite design briefs

## Do

The site's owner wants THREE completely different full-site UI rewrite concepts to compare and
choose between — not incremental theming (the site already has a CSS-custom-property theming
system in `src/styles/globals.css` plus a "Matrix" easter-egg theme; this is explicitly bigger
than that), but three genuinely distinct rewrites of the FORMATTING, STYLING, LAYOUT, and
ORGANISATION of every element, all using the same underlying data the site already has. The owner
has explicitly delegated the creative direction entirely to you — "I'm happy for the loop to
decide everything... as long as three different versions are produced, I don't care how it's
done" — so invent three real, meaningfully different directions yourself; do not ask for
clarification, there is no human review gate on this task.

Research the current site first: read the root `CLAUDE.md` "What this project is" section (five
content areas — reviews across movies/tv/books/albums/perfumes, vinyl, workouts/exercises,
listening, projects), and skim a representative page or two from each area
(`src/pages/index.js`, `src/pages/reviews/movies/index.js`, `src/pages/vinyl/index.js`,
`src/pages/workouts/index.js`) to understand today's actual layout (a fairly conventional
card/list-based personal-site layout) before proposing alternatives.

Write **exactly three** distinct, concrete design briefs to
`.harness/designs/T042-ui-rewrite-briefs.md`, one section per variant (`## Variant 1`,
`## Variant 2`, `## Variant 3`). Each brief must be concrete enough that a fresh builder agent
(with no other context) could implement it consistently across every page without further
creative guessing. For EACH variant's brief, cover at minimum:
- A short name/identity for the direction (e.g. "dense data-table", "editorial card magazine",
  "minimal linear feed" — invent your own three, these are just illustrative).
- The overall navigation paradigm (top nav vs. sidebar vs. something else).
- The information-density approach (compact tabular vs. spacious card-based vs. something else).
- How a list of items (e.g. movie reviews) should look, and how an individual item's detail/form
  should look.
- Typography/visual-hierarchy approach (distinct from just a color palette — this must be a
  structural/layout difference, not merely a new set of CSS custom-property values).
- Any shared layout chrome (header/footer/nav) unique to this variant.

The three variants must be **genuinely, structurally different from each other** — not three
palettes of the same layout. This brief is the sole creative-direction input for 21 downstream
build tasks (T043–T063), so ambiguity here becomes ambiguity multiplied across all of them —
be as concrete and unambiguous as you can.

## Done when

- `.harness/designs/T042-ui-rewrite-briefs.md` exists with exactly three top-level sections
  (`## Variant 1`, `## Variant 2`, `## Variant 3`), each containing all six elements listed above
  (name/identity, navigation paradigm, density approach, list-view description, detail/form-view
  description, typography/hierarchy approach).
- The three briefs describe structurally different layouts from one another — not the same layout
  with three different color schemes (spot-check: each brief's navigation paradigm and
  information-density description must differ from the other two).
- No other file was modified — this task only produces the design doc.
