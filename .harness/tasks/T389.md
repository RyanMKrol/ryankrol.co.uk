## Overview
The user wants 'all markdowns displayed accurately' across the whole site, separate from (and not blocked by) the single-newline decision being handled in its own needs-human task. src/components/Markdown.js is rendered from roughly 25 call sites: ReviewCard.js (movie/TV/book/album review bodies, multiple render paths — list view uses block mode, inline mode strips to a flat run of INLINE_ELEMENTS for album track highlights), perfumeVariants/Variant6Hybrid.js (perfume descriptions), src/pages/hot-takes/{add,edit/[id]}.js, the five review-type add/edit forms under src/pages/reviews/**/{add,edit/[id]}.js (live preview panes), src/pages/tom.js (the /tom preview pane for the homepage Top-of-mind note), and src/pages/index.js (the actual homepage Top-of-mind render). Markdown.test.js already has coverage for bold/italic/list/link/blank-line-gap behaviour in both block and inline mode (7 tests) — this task extends that same rigor to confirm every call site actually renders correctly against its real source content, not just that the shared component's isolated unit tests pass.
