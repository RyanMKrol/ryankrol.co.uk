# T032 — Drive the TMDB movie/TV search from the title field (remove the separate box)

## Do

Make the TMDB search/confirm step read the **title the user already typed** instead of having its
own editable search box. Today `src/components/TmdbSearch.js` renders its own `<input>` (state
`query`, seeded once from the `initialQuery` prop), so the user effectively types the title twice.

- In `src/components/TmdbSearch.js`: REMOVE the standalone search `<input>` and the internal
  `query` state / `initialQuery` prop. Instead accept the live search term as a `query` prop from
  the parent (the form's title). Keep the **"Search"** button — clicking it runs the TMDB search
  against the `query` prop; keep the results list, Select, confirmed-state, and Clear UI exactly as
  they are. The Search button is disabled when the incoming `query` is empty/whitespace. (Enter-to-
  search no longer applies since there's no input — drop `handleKeyDown`.)
- In `src/pages/reviews/movies/add.js` and `src/pages/reviews/tv/add.js`: pass the live title to
  the component as `query={formData.title}` (NOT the one-shot `initialQuery`) so the button always
  searches the current title. The `onSelect` wiring + the `tmdbMatch` submit body stay unchanged.
- Keep everything else about the add flow intact: the search remains OPTIONAL (the user can submit
  a review without ever searching), and the saved record is unaffected when no match is selected.

## Done when

- `src/components/TmdbSearch.js` no longer renders its own text input; it searches the term passed
  in via a `query` prop, driven by the page's title field.
- On `/reviews/movies/add` and `/reviews/tv/add`, typing a title and clicking "Search" returns TMDB
  results for that title with no second search box anywhere on the page; the Search button is
  disabled until a title is entered.
- Selecting a result still attaches the TMDB metadata to the submitted review (unchanged
  `onSelect`/submit body); submitting without searching still works.
- Manually verify against the running app: add-movie page → type a title → Search → results appear
  → Select → confirmed chip shows; submit succeeds.
- `npm run lint`, `npm test`, `npm run build` all pass.
