# T013 — Multi-theme infrastructure (attributes, persistence, hooks)

## Do

Extend the existing CSS-custom-property theming so theme / font / light-dark mode /
reduced-motion are selectable at runtime and persisted. This is the plumbing only —
the 5 actual themes (T014) and the picker UI (T015) come next.

- Drive everything off `<html>` data attributes: `data-theme` (theme family),
  `data-mode` (`light`|`dark`), `data-font`, `data-motion` (`reduced`). CSS rules in
  `globals.css` key off these (base attribute scaffolding here; palettes in T014).
- Add a **pre-paint** inline script in `src/pages/_document.js` that reads the saved
  prefs from `localStorage` and sets the `data-*` attributes BEFORE first paint (no
  flash), falling back to `prefers-color-scheme` for mode and `prefers-reduced-motion`
  for motion when nothing is stored.
- Add a `useTheme` hook (`src/hooks/useTheme.js`) to read/set theme/font/mode/motion,
  persisting to `localStorage` and updating the `data-*` attributes live (mode `System`
  reacts to OS changes while open).
- Put the pure resolution logic in `src/lib/appearance.js` (e.g.
  `resolveMode(storedMode, osPrefersDark)` and a `readStoredAppearance`/defaults helper)
  and unit-test it.
- Update `useChartTheme` (`src/hooks/useChartTheme.js`): it currently re-reads only on
  `class` mutations — add `data-theme`/`data-mode`/`data-font` to its `MutationObserver`
  so charts recolour when the theme switches.
- Reconcile with the Matrix Konami easter egg: `matrix-active` stays a separate overlay
  that works on TOP of whichever theme is active; ensure it and the `data-*` attributes
  don't conflict.

## Done when

- `<html>` carries `data-theme`/`data-mode`/`data-font`/`data-motion`, set pre-paint from
  `localStorage` with OS-preference fallbacks, with NO flash of the wrong theme on load.
- `useTheme` reads, persists, and applies changes live; `useChartTheme` reacts to
  `data-theme`/`data-mode`/`data-font` changes (charts recolour on theme switch).
- `src/lib/appearance.js` is covered by `src/lib/appearance.test.js`: `resolveMode`
  maps stored `system`→OS preference and `dark`/`light`→themselves; defaults are applied
  when storage is empty/garbage. `npm test` passes.
- The Matrix Konami easter egg still works and does not fight the new attributes.
- `npm run lint`, `npm test`, `npm run build` pass.
