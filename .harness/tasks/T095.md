# T095 — Per-spot spray count UI — make it joyful

## Do

T094 changed the perfume review's `applicationSpots` field to an array of `{spot, sprays}` objects
(replacing the old flat array of ticked labels). Update `src/pages/reviews/perfumes/add.js` (and
`src/pages/reviews/perfumes/edit/[id].js`, which mirrors the add form per this repo's review-type
convention) to let the user record a spray count for each of the six application spots (Wrists,
Elbows, Clavicles, Beard, Back of neck, Behind ears), submitting the new `{spot, sprays}[]` shape.

**The owner explicitly asked for a "joyful" UI treatment here, not a plain number input bolted onto
each checkbox** — you have full creative latitude on the actual interaction/visual design. Some
illustrative (non-prescriptive) directions: tap-to-increment counters per spot (e.g. tapping a
spray-bottle icon adds one spray, tapping again adds another, with a running count shown), a small
stepper control per spot, a playful animation or micro-interaction on each tap, or something else
entirely — invent what feels genuinely delightful, appropriate to this site's existing visual style
(check `src/styles/globals.css`'s theming approach and other perfume-form components like
`src/components/PerfumeCharacteristics.js` for the established look/feel to stay consistent with).
A spot with 0 sprays should not appear in the submitted `applicationSpots` array at all (per T094's
validator).
