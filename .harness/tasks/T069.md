# T069 — Backfill-from-API button on the albums edit page

## Do

On `src/pages/reviews/albums/edit/[id].js`, add a "Backfill from Last.fm" button using the shared
`MetadataBackfillModal` component from T066. Wire it so:
- The search is seeded from this form's current `title`/`artist` fields, calling
  `GET /api/lastfm/album-search?query=...` (see `src/pages/api/lastfm/album-search.js`, which
  returns `{ results: [mapAlbumSearchResult, ...] }`).
- When the user selects a candidate from the search results, fetch its full details via
  `GET /api/lastfm/album-info?artist=...&album=...` (or `?mbid=...` if the search result has an
  mbid — see `src/pages/api/lastfm/album-info.js`, which returns `{ info: mapAlbumInfo }`) before
  showing it as the final confirmable candidate, since the search result alone is a summary and
  `album-info` has the full `lastfm` object shape this review type stores.
- On confirm, set this form's `lastfm` field to the full `mapAlbumInfo` result, per the "overwrite
  all metadata fields" behavior T067 now supports on `POST /api/reviews/albums/update` (which
  derives `thumbnail` from `lastfm.images` server-side, same as `add.js` already does — you don't
  need to compute `thumbnail` client-side, just send the `lastfm` object).
- **NEVER touch `rating` or the `highlights` field** (this form's `highlights` field is the user's
  OWN written review text, not sourced from Last.fm).
- The existing Save/Delete flow for this edit page is otherwise unchanged — the backfill button
  just pre-fills form fields; the user still clicks the page's normal Save button afterward to
  persist (do not auto-submit on confirm).

## Done when

- Loading `/reviews/albums/edit/[id]` for an existing album review, clicking "Backfill from
  Last.fm", picking a search result, and confirming updates the form's `lastfm` metadata visibly
  (e.g. a cover thumbnail derived from it appears/updates), without touching the rating or the
  review-text field.
- Clicking the page's existing Save button afterward persists the backfilled `lastfm` data via
  `POST /api/reviews/albums/update` (confirm by reloading the edit page and seeing the new
  thumbnail/metadata still present).
- No file outside `src/pages/reviews/albums/edit/[id].js` and `src/components/**` was modified.
