## Overview
The shared `.collection-review-header` header (title-group + controls, flex `justify-content: space-between`, defined in src/styles/globals.css around line 430-457) renders two visually different layouts purely as a side effect of content width: pages whose `.collection-review-controls` is narrow enough to share the title's row (movies/tv/books/albums: just SearchInput + SortButtons) land right-aligned next to the title, while pages whose controls also render a pill row (perfumes' ownership pills, projects' tag pills, vinyl's view-toggle pills) are wide enough to wrap onto their own line and, being the lone child on that line, default to the left — giving an accidental left-aligned/stacked look. This is one shared CSS rule pair producing two looks, not two authored styles. The fix: make `.collection-review-controls` always wrap onto its own row below the title (matching perfumes/projects' current look) regardless of content width, so every consumer of this header class renders consistently sitewide.

## Do
In `src/styles/globals.css`, change the `.collection-review-controls` rule (around line 452) so it always takes the full row width and sits left-aligned below `.collection-review-title-group`, instead of only doing so incidentally when it's wide enough to wrap under the existing `flex-wrap: wrap` on `.collection-review-header` (line 432). The minimal fix is adding `flex-basis: 100%;` (or `width: 100%;`) to `.collection-review-controls` so it always drops to its own line under `flex-wrap: wrap`, then relies on default flex-start alignment for a single child on that line (the same mechanism perfumes/projects/vinyl already exhibit today, just made deliberate and universal instead of accidental). Do not touch any page-level JSX — this is a CSS-only fix and should apply uniformly to every current consumer of `.collection-review-header`/`.collection-review-controls`: reviews/movies, reviews/tv, reviews/books, reviews/albums, reviews/perfumes, projects, vinyl, and workouts (`src/pages/workouts/index.js`) — confirm with `grep -rn collection-review-controls src/pages` that the fix's scope covers all current usages before finishing. `listening/index.js` and `workouts/[id].js` only render `.collection-review-title-group` with no controls sibling, so they are unaffected and need no verification. Keep the existing `gap`, `align-items`, and internal control spacing (SearchInput/SortButtons/PillGroup gaps) unchanged — only the row-level placement of the controls block relative to the title should change.
