Background: `.projects-grid` (src/styles/globals.css around lines 1750-1760) currently reads:
```
.projects-grid { column-count: 3; column-gap: 1.25rem; margin-bottom: 2rem; }
@media (max-width: 900px) { .projects-grid { column-count: 1; } }
```
Browser-native CSS multi-column layout fills column 1 top-to-bottom completely before starting column 2. So when the owner sorts projects by stars (SortButtons in src/pages/projects/index.js; `sortedRepos` computed around lines 123-135, rendered into `.projects-grid` around lines 210-264), item #1 (highest stars) appears at the top of column 1, item #2 appears BELOW it in column 1 (not beside it in column 2), etc. — reading order is top-to-bottom-then-left-to-right instead of the expected left-to-right-then-top-to-bottom. The masonry PACKING behavior (variable-height cards, no forced uniform rows, no dead gaps) must be preserved — this is purely about reading order.

Build:
1. `src/lib/masonryColumns.js` — export a pure function `distributeRoundRobin(items, columnCount)` returning an array of `columnCount` arrays, assigning `items[i]` to column `i % columnCount` (item 0 -> col 0, item 1 -> col 1, item 2 -> col 2, item 3 -> col 0, ...). Each column keeps its own items in original relative order so it still packs its own variable-height cards top-to-bottom with no gaps — only the column ASSIGNMENT becomes explicit instead of browser auto-fill. Guard `columnCount <= 1` (return `[items]`) and empty `items` (return `columnCount` empty arrays or `[]` — pick one and cover it in the tests).
2. `src/lib/masonryColumns.test.js` — cover: even split across N columns, uneven remainder, columnCount === 1, empty items array, columnCount >= items.length.

4. `src/components/MasonryColumns.js` — presentational component, props `{ items, columnCount, renderItem, className, columnClassName }`. Calls `distributeRoundRobin(items, columnCount)` and renders one outer `<div className={className}>` containing `columnCount` `<div className={columnClassName}>` children, each mapping its bucket through `renderItem(item, originalIndex)` — pass the item's ORIGINAL index within the full `items` array (not the per-column index), so callers can build stable keys/gradients keyed by original position.
