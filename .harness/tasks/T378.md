## Overview
The shared Markdown editor box looks unpolished: its top-right corner does not match the rounded top-left, and the textarea shows the browser's ugly native diagonal resize grip at the bottom-right. Make the editor's corners agree and replace the native resize affordance, scoping the CSS to the MarkdownEditor's own wrapper so the shared `.collection-form-textarea` base class (used by other forms) is not affected.

## Do
CSS-only polish in `src/styles/globals.css`, scoped to the MarkdownEditor's own wrapper (the `.markdown-editor` / `.markdown-editor .collection-form-textarea` selectors already present around lines 2876-2909) — do NOT change the shared `.collection-form-textarea` BASE class, which is used by perfumes/vinyl/hot-takes and other add forms (that would ripple beyond this editor). `src/components/MarkdownEditor.js` is in scope only if a small wrapper/class tweak is genuinely needed to hang the styling off; prefer to achieve everything in CSS via the existing wrapper and leave the component markup untouched if possible.

Two fixes:
- **Corners:** make all four corners of the editor box agree so the top-right visually matches the rounded top-left (the report is that the composite of the toolbar strip above + the textarea reads asymmetric on the right side). Round the editor consistently — e.g. give `.markdown-editor .collection-form-textarea` a symmetric `border-radius` (all four corners equal) using the existing `--radius-*` design tokens, matching the Collection look. Do not introduce hardcoded colours or radii — use the CSS custom properties already in `:root`.
- **Resize grip:** replace the browser's native diagonal resize grip on the textarea with `resize: vertical` (keep drag-to-grow vertically, lose the ugly corner grabber) — scoped to `.markdown-editor .collection-form-textarea`, not the base class.

Do not change the toolbar's behaviour or the buttons (that is a separate task). This is chrome only.
