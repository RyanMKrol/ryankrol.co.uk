# T034 — Google Books search provider (normaliser + route dispatch)

## Do

Add Google Books as a second book-search provider alongside Open Library, because Open Library's
coverage/metadata is sparse. Google Books is **keyless to start** (no API key — works for low
personal volume; results are cached via `withApiCache`). Both providers must return a **common
normalised book-result shape** so the existing select → submit flow works for either source.

- Create `src/lib/googlebooks.js` — a **pure**, unit-tested `mapGoogleBooksResult(volume)` that maps
  a Google Books `volumes` item (`volume.volumeInfo`) to the common shape:
  `{ source: 'googlebooks', volumeId, title, authors (array), firstPublishedYear (number from
  publishedDate's leading year, else null), coverUrl (https thumbnail from imageLinks.thumbnail /
  smallThumbnail, forced to https, else null), isbn (array from industryIdentifiers), subjects
  (array from categories), pageCount (number|null), publisher (array|[]) }`. Be defensive: every
  Google field can be missing — default to null/[] and never throw.
- Extend `src/pages/api/books/search.js` to accept a `provider` param (`openlibrary` | `googlebooks`,
  default `openlibrary`):
  - `googlebooks` → query `https://www.googleapis.com/books/v1/volumes` with
    `q=intitle:<title>` (+ `+inauthor:<author>` when author present), keyless, `maxResults` ~20,
    map each item via `mapGoogleBooksResult`.
  - `openlibrary` → the existing title+author path (from T033). Add `source: 'openlibrary'` to each
    Open Library result (and continue returning `coverId`) so the UI can tell results apart and pick
    a cover. (Open Library results keep `coverId`; Google results carry `coverUrl` — the UI resolves
    whichever is present.)
  - Include `provider` in the `generateCacheKey('book-search', { provider, title, author })` key.
    Missing `title` → 400. Keep `withApiCache`, the 4h cache, and emoji logging.
- Document the new `provider` param + the Google Books integration in `CLAUDE.md` (API table + file
  map for `src/lib/googlebooks.js`) and `README.md` if it lists routes, same commit.

## Done when

- `src/lib/googlebooks.js` exports `mapGoogleBooksResult`, covered by `src/lib/googlebooks.test.js`:
  a full volume maps to the common shape with a https `coverUrl`, ISBNs from industryIdentifiers,
  year parsed from `publishedDate`; a sparse volume (no imageLinks/categories/identifiers) maps
  without throwing, defaulting missing fields to null/[]. `npm test` passes.
- `GET /api/books/search?provider=googlebooks&title=Dune&author=Herbert` returns normalised Google
  results each tagged `source:'googlebooks'` with a usable `coverUrl`; `provider=openlibrary` (or no
  provider) returns Open Library results tagged `source:'openlibrary'`; missing `title` → 400; the
  cache key distinguishes providers.
- `CLAUDE.md` (+ `README.md` if applicable) document the provider param + `googlebooks.js`.
- `npm run lint`, `npm test`, `npm run build` all pass.
