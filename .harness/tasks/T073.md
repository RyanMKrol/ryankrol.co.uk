# T073 — Designer typeahead/autocomplete on the perfume add form

## Do

On `src/pages/reviews/perfumes/add.js`, turn the "Designer" text input (currently a plain
`formData.designer` controlled input around line 101-106) into a typeahead/autocomplete that
suggests designer names already present in the `PerfumeRatings` table as the user types.

Fetch `GET /api/reviews/perfumes` (the existing perfumes list endpoint) — either on page load or
lazily on first focus of the Designer field, your call — and derive the distinct set of
`designer` values from the returned reviews (dedupe, e.g. case-sensitive exact match is fine, no
need for fuzzy normalization). Use a native HTML `<datalist>` bound to the Designer `<input>` (the
simplest, dependency-free way to get typeahead suggestions), or an equivalent lightweight
client-side filtered-suggestion-list if you prefer — no new npm package is needed for this, GitHub
Copilot native `<datalist>` support is sufficient for a plain text suggestion list.

This is the ADD form only — the perfume edit page is out of scope for this task (not requested).

## Done when

- Loading `/reviews/perfumes/add` in a running dev server and typing into the Designer field shows
  suggestions drawn from designer names of existing perfume reviews already in the database.
- Selecting a suggestion (or just typing a new designer name not in the list) both work — the
  field remains a free-text input; the suggestions are optional assistance, not a restrictive
  dropdown that blocks entering a brand-new designer.
- No file outside `src/pages/reviews/perfumes/add.js` was modified (no new API endpoint was
  added — the distinct-designer list is derived client-side from the existing
  `GET /api/reviews/perfumes` response).
