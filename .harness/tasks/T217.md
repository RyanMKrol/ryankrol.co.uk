## Do

Add a new shared date-formatting helper at `src/lib/dateFormat.js` (new file) that will be used to replace every place in this codebase that currently renders a raw DD-MM-YYYY string, or does its own numeric re-formatting, with a written English-language date like '17 May 2026'. This is task 1 of a 4-task cluster; three follow-up tasks depend on this one and import from this exact file — do not skip exporting exactly the two functions named below with these exact signatures/names, since the dependent tasks are written against them.

Export two functions:

1. `formatEnglishDate(date)` — takes EITHER a `Date` object or anything `new Date(...)` can parse (e.g. an ISO-8601 timestamp string such as a workout's `start_time`), and returns a string like `'17 May 2026'` via `.toLocaleDateString('en-GB', { day: 'numeric', month: 'long', year: 'numeric' })`. If given something falsy (`null`/`undefined`/`''`) or a value that produces an Invalid Date, return `''` (empty string) rather than throwing or rendering the literal text 'Invalid Date' — callers will render the result directly in JSX behind a truthy check, so an empty string is the safe 'nothing to show' signal.
2. `formatReviewDate(dateString)` — the review-specific wrapper. This codebase stores review dates (movies/tv/books/albums) as `DD-MM-YYYY` strings (per `CLAUDE.md`'s Data model section, written via `toLocaleDateString('en-GB')` at write time) — NOT a format the built-in `Date` constructor parses reliably across environments. Parse `dateString` by splitting on `-` into day/month/year, construct `new Date(year, month - 1, day)`, then delegate to `formatEnglishDate` for the actual display formatting. Return `''` for a falsy or malformed input (wrong number of `-`-separated segments, or any segment that isn't parseable as a number) rather than throwing.

Also add a co-located Jest test file `src/lib/dateFormat.test.js` covering: `formatEnglishDate` with a plain `Date` object, with an ISO string, and with a falsy input (expect `''`); `formatReviewDate` with a valid `'17-05-2026'`-shaped string (assert it produces exactly `'17 May 2026'`), a single-digit day/month input like `'5-3-2026'`, and falsy/malformed input (expect `''`).

Finally, add one row to root `CLAUDE.md`'s 'File map' table (the `src/lib/*` rows) describing `src/lib/dateFormat.js` — e.g. `formatEnglishDate(date)` / `formatReviewDate(dateString)` — shared English-language date formatting, used by `ReviewCard.js` and the workouts list page — per this repo's Definition of Done requiring docs to move in the same change as a new `src/lib` file.
