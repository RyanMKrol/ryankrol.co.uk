# T009 — Programmes overview page (/programmes)

## Do

Add a new page `src/pages/programmes/index.js` — a per-programme overview with a
**Push / Pull / Legs selector** showing a small dashboard for the selected programme:

- A **volume-over-time** chart (per session), a **working-sets-over-time** chart, and a
  **workout-frequency** view (per month), plus headline totals (total workouts, total
  volume, avg volume per workout).
- Get the full workout list via the all-workouts fetch path introduced in **T001** (do
  not re-add a separate scan), and compute everything client-side via
  `aggregateProgramme` / `classifyProgramme` from `src/lib/programmeStats.js` (T008).
- Put the charts in a new `src/components/ProgrammeOverviewCharts.js`, modelled on
  `src/components/ExerciseProgressCharts.js`, reading theme via `useChartTheme()` (NEVER
  hardcode chart colours — see CLAUDE.md).
- Add a link to `/programmes` from the workouts page (`src/pages/workouts/index.js`).

## Done when

- `/programmes` renders, defaults to one programme, and the selector switches between
  Push / Pull / Legs, updating every chart + the headline totals.
- Charts read colours via `useChartTheme()` and the data is the client-side aggregation
  from `programmeStats` over the all-workouts fetch (no new API endpoint).
- A link to `/programmes` is present on the workouts page.
- `npm run lint`, `npm test`, `npm run build` pass; manually load `/programmes` and
  confirm switching the split updates the dashboard.
