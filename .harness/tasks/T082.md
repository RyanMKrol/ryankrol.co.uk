# T082 — Write a script to backfill missing computed metrics onto incomplete Workouts records

## Do

Confirmed via live production DynamoDB queries: 72 of 367 `Workouts` table items are missing the
denormalized computed-metrics fields (`exercises`, `totalVolume`, `totalWarmupSets`,
`totalWorkingSets`, `uniqueExercises`, `durationMinutes`, `workoutType`) that ~295 other items DO
have. This is NOT missing data — the real per-exercise records exist in the separate `Exercises`
table (verified: workout `5c4249f9-bd32-44ff-b7a5-b3edf05303f6`, which has `exercise_count: 4` on
its incomplete `Workouts` item, has exactly 4 matching rows in `Exercises` queryable via the
`workout_id-index` GSI — see `getExercisesForWorkout`-style query pattern already used in
`src/lib/workoutQueries.js`). The data just was never denormalized onto the `Workouts` item, the
way `src/lib/workoutBackfill.js` does for workouts it fetches directly from the Hevy API (see its
`calculateWorkoutMetrics`/`calculateExerciseMetrics` calls from `src/lib/workoutMetrics.js` — REUSE
these existing functions, don't reimplement the metrics math).

Write a new script `src/scripts/backfillIncompleteWorkoutMetrics.js`, following this repo's
migration-script conventions (see `src/scripts/migrateAlbumsToV2.js`/`src/scripts/auditWorkouts.js`
for style — direct AWS SDK usage, `.env.local` credentials, clear `console.log` progress). For
each `Workouts` item missing an `exercises` field:
1. Query the `Exercises` table via the `workout_id-index` GSI for that workout's id, to get its
   real exercise+set records.
2. Reshape those Exercises-table rows into whatever input shape `calculateWorkoutMetrics`/
   `calculateExerciseMetrics` (`src/lib/workoutMetrics.js`) expect (check their actual parameter
   shapes — they're currently called with `workout.exercises` where each exercise has its own
   `sets` array, matching what `workoutBackfill.js` passes in).
3. Compute the metrics fields and update the `Workouts` item with them (add `exercises`,
   `totalVolume`, `totalWarmupSets`, `totalWorkingSets`, `uniqueExercises`, `durationMinutes`,
   `workoutType` — matching the shape of the ~295 already-complete records).

**This script MUST be dry-run by default** (print what it would change for each incomplete
workout — the id, title, and computed metrics summary — without writing), and only perform real
`UpdateItem`/`PutItem` writes when explicitly invoked with a live flag (e.g. `LIVE=1 node
src/scripts/backfillIncompleteWorkoutMetrics.js`), since this mutates real workout history data.
Add an npm script for it in `package.json` (e.g. `"workout:backfill-metrics": "node
src/scripts/backfillIncompleteWorkoutMetrics.js"`), matching this repo's existing script-naming
convention.

**Verification for THIS task must use mocked/fixture data, never the real production DynamoDB
table** — write a test (or small standalone test harness, matching this repo's existing test
conventions) that feeds the script's core logic a fixture "incomplete workout" + fixture
"Exercises table rows" and asserts the computed metrics match calling
`calculateWorkoutMetrics`/`calculateExerciseMetrics` directly on equivalent input, without ever
touching real AWS credentials. The actual live run against production is a separate, manual,
human-gated step (see T083) — NOT part of this task's "Done when".

## Done when

- `src/scripts/backfillIncompleteWorkoutMetrics.js` exists, defaults to dry-run (no writes without
  `LIVE=1`), and implements the query/compute/update logic described above, reusing
  `calculateWorkoutMetrics`/`calculateExerciseMetrics` from `src/lib/workoutMetrics.js` rather than
  reimplementing metric math.
- A test using mocked/fixture data (not real AWS calls) confirms the script correctly identifies an
  "incomplete" workout (missing `exercises`) vs a "complete" one (has `exercises`, should be
  skipped), and that the computed metrics for a fixture incomplete workout + its fixture Exercises
  rows match what `calculateWorkoutMetrics` would produce directly. `npm test` passes.
- `package.json` has a new npm script to run this backfill, following the existing naming
  convention.
- This task's own verification made zero real AWS calls.
