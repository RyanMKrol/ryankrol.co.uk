# T014 — Five bright/joyful themes (palettes + fonts)

## Do

Design and implement **five distinct themes**, leaning bright, joyful, and pixel-art /
Terraria-flavoured (the owner finds the current look dark and grim). Build on the T013
attribute system.

- Each theme is a COMPLETE set of the app's CSS custom properties (backgrounds, text,
  borders, accents, semantic success/error, chart colours, tooltip, buttons, inputs,
  cards, star ratings, etc.) defined in `src/styles/globals.css`, keyed by
  `html[data-theme="..."]`, with BOTH a light and a dark palette (via `data-mode`).
- Each theme gets its own **font pairing**; load the needed fonts (e.g. in
  `src/pages/_document.js`). Give each a friendly name + emoji (for the picker tiles).
- One BRIGHT theme is the **default** (applied when nothing is stored). Keep the original
  dark look available as one of the five (so the current style isn't lost).
- Ensure NO theme leaves any used CSS var undefined — every surface must be coloured in
  every theme × mode.

## Done when

- Five themes exist, each defining the full CSS-var set for light AND dark, each with a
  distinct font pairing and a name+emoji; a bright theme is the default.
- Switching `data-theme` / `data-mode` (via `useTheme`) recolours the WHOLE app including
  charts; spot-check key surfaces (page bg, text, borders, accents, chart series,
  buttons, cards) in every theme × mode — none undefined/unreadable.
- `npm run lint`, `npm test`, `npm run build` pass; manually cycle all 5 themes in light
  + dark and confirm no broken or illegible surfaces. (Aesthetic sign-off is T016.)
