# T274 — MarkdownEditor component

## Approach taken
Hand-rolled toolbar (fallback path), not `@uiw/react-md-editor`. No new dependency was added —
`package.json` is unchanged. Rationale: the repo already tolerates zero markdown-editing deps, the
task's own fallback (a selection-based textarea toolbar) fully satisfies the "bold/italic/list/link"
requirement with no theming-integration risk, and CLAUDE.md's "no new infra unless explicitly asked"
convention favours the smaller footprint when a library isn't required to hit the bar.

## What was built
- `src/components/MarkdownEditor.js` — controlled component with the exact textarea-compatible prop
  contract (`{ id, name, value, onChange, placeholder, required, className }`). Toolbar buttons
  (Bold/Italic/List/Link) operate on `textarea.selectionStart`/`selectionEnd` and call `onChange`
  with a `{ target: { name, value } }`-shaped event, so it drops into any existing
  `handleInputChange(e)` unchanged. Falls back to `collection-form-textarea` as the default
  className, matching every current call site.
- `src/components/MarkdownEditor.test.js` (RTL) — renders the component (via a small stateful demo
  wrapper plus one direct-props case), selects all text, clicks each toolbar button, and asserts the
  resulting markdown syntax + that `onChange` fires with the new value.
- `src/styles/globals.css` — added `.markdown-editor`, `.markdown-editor-toolbar`, and
  `.markdown-editor-toolbar-button` rules built entirely from existing CSS custom properties
  (`--font-mono`, `--color-hairline-strong`, `--radius-pill`, `--color-bg`, `--color-ink`, etc.), so
  it automatically re-themes under `html.matrix-active` with no separate matrix override needed (same
  mechanism as every other themed element).
