# T168 — Visual audit: workouts list, workout detail, exercise detail

Compared the live pages (dev server, desktop 1400px + mobile 390px via Playwright
screenshots) against `renders/3f`/`4k` (workouts list), `3g`/`4l` (workout detail), and
`2c`/`4d` (exercise detail).

## Gaps found and fixed

1. **Chart cards had no visible border/background in the default theme.**
   `.chart-card` in `globals.css` referenced `--color-surface` / `--color-border` /
   `--radius-large`, which are only ever defined inside the `html.matrix-active` block —
   the design's paper theme had no fallback, so all three progress charts
   (`ExerciseProgressCharts`, `CardioProgressCharts`) rendered as borderless, backgroundless
   panels floating on the page background. Fixed to use the Collection tokens
   (`--color-card`, `--border-width` solid `--color-ink`, `--radius-card`), matching the
   bordered white card in `2c`.

2. **Volume-chart highlighted bar used the wrong purple.** `useChartTheme`'s `chartGrape`
   fallback was `#93328E` (a dark plum), not defined anywhere in the design tokens. The
   design's "grape" is `#7A5AF8` (same value as the listening/perfumes accent). Fixed the
   fallback in both the SSR default and the CSS-var `get()` call, and added matching
   `--color-chart-*` tokens to `:root` in `globals.css` so the default theme has real values
   instead of only inheriting the JS hardcoded fallback.

3. **Chart titles carried emoji + were colored per-chart** (🎯 coral, 📊 indigo, 💪 marigold,
   plus a "📈 Progress Charts" emoji heading above them). The design shows plain black/ink
   titles with no emoji and no wrapper heading — cards flow directly under the stat row.
   Removed the emoji and the wrapper `<h2>` from both `ExerciseProgressCharts.js` and
   `CardioProgressCharts.js`; chart card titles are now `var(--color-ink)`. Removed the
   now-unused `chartCardDefault` destructure from `CardioProgressCharts.js`.

4. **"Progress Insights" panel title was dim white**, but the design's "PROGRESS INSIGHTS"
   label is coral/orange. Changed `.exercise-insights-panel-title` color to
   `var(--accent-movies)` and dropped the `opacity: 0.7`.

5. **Workout list card stats were rendered as heavy bordered boxes** (via `StatBlock`), but
   the design (`3f`/`4k`) shows plain unboxed "4 exercises · 11.2t volume" pairs. Replaced
   the `StatBlock` usage in `workouts/index.js` with plain markup + new
   `.workout-session-stat(-value|-label)` CSS classes.

6. **Workout list volume was shown as raw kg with commas** (e.g. "8,598.32kg"); the design
   abbreviates to tonnes with one decimal ("8.6t"). Added `formatVolumeTonnes()` in
   `workouts/index.js` and switched the list card's volume stat to it. Left the workout
   *detail* page's volume as full comma-separated kg — the design (`3g`) shows the full
   number there, so list vs. detail intentionally differ.

7. **Workout detail's DATE stat chip used a long weekday format** ("Friday, 3 July 2026"),
   which wrapped across 3 lines and visually broke the 4-chip row compared to the compact
   `02·07·26`-style chip in `3g`. Shortened to the same `DD-MM-YYYY` format used elsewhere
   on the site (kept the existing hyphen convention rather than introducing a new
   dot-separated format, to stay consistent with the rest of the site). Also gave the DATE
   chip `accentColor="var(--accent-workouts)"` so it renders ink-filled like the design,
   instead of the same neutral white/bordered treatment as the other three chips.

## Verified no functional regression

Used a headless Playwright script against the local dev server:
- Workouts list "Push" filter pill correctly filters all visible cards to Push-only badges.
- Exercise detail page's "3 months" time-range pill changes the stat block values (sessions
  count, all-time 1RM, etc.) and the chart data, confirming the toggle still works after the
  markup/CSS changes.
