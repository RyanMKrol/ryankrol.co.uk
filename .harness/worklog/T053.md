# T053 — Variant 2 — reviews add forms

## What was built

Added the 5 review-type add forms for Variant 2 (Editorial Card Magazine), under
`/v2/reviews/{movies,tv,books,albums,perfumes}/add`, reusing the existing `V2Layout`
shared chrome from T044 (no second layout wrapper built).

- `src/components/v2/V2ArticleForm.js` — new shared "article shell" form component per
  the Variant 2 brief: centered ~680px column, large serif headline, underline-only
  minimal-chrome inputs (no boxes), byline-style clickable star rating (`V2StarPicker`).
  Both add pages and the future edit pages can reuse this.
- `src/pages/v2/reviews/{movies,tv}/add.js` — title, TMDB match search, rating, review
  textarea, password. Mirrors the production `/reviews/{movies,tv}/add.js` field set and
  submits to the unchanged `POST /api/reviews/{movies,tv}/add`.
- `src/pages/v2/reviews/books/add.js` — title, author, book match search, rating,
  overview, password → unchanged `POST /api/reviews/books/add`.
- `src/pages/v2/reviews/albums/add.js` — title, artist, Last.fm match search (with cover
  preview), rating, highlights, password → unchanged `POST /api/reviews/albums/add`.
- `src/pages/v2/reviews/perfumes/add.js` — title, designer, type, 0–10 rating, description,
  notes, travel/full-bottle checkboxes, longevity/projection sliders, seasons checkboxes,
  password → unchanged `POST /api/reviews/perfumes/add`. Perfume-specific controls
  (sliders/checkboxes) were reimplemented locally with `v2-*` classes rather than reusing
  `components/PerfumeCharacteristics.js`, since that component is styled with the
  production `form-*` global classes and would have broken Variant 2's minimal
  underline-input look.

Search-match widgets (`TmdbSearch`, `BookSearch`, `LastfmAlbumSearch`) are reused as-is
from the production `src/components` (read-only reuse, same pattern T044 used for
`NowPlaying` in `V2Layout`) — no files under `src/components/*` (non-v2) were modified.

## Verify

- `npm run lint` — 0 errors (2 pre-existing warnings, unrelated files).
- `npm test` — 137/137 passing.
- `npm run build` — succeeds; all 5 new routes appear as static pages
  (`/v2/reviews/movies/add`, `/tv/add`, `/books/add`, `/albums/add`, `/perfumes/add`).
- `npm run dev` + `curl` — all 5 routes return `200`.

## Scope check

Only files under `src/pages/v2/**` and `src/components/v2/**` were added/changed. No file
under `src/pages/api/**` or the production `src/pages/reviews/**` was touched.
