# T144 — Re-skin StarRating, build PipMeter

## What changed
- `src/styles/globals.css`: added `--color-star-filled: var(--color-star-gold)` to the
  Collection `:root` token block (the matrix theme already defines its own override at
  line ~102, unaffected). Added `.pip-meter` / `.pip-dot` / `.pip-dot.filled` /
  `.pip-dot.interactive` rules using `var(--pip-filled)` / `var(--pip-empty)`.
- `src/components/StarRating.js`: added a `readOnly` boolean prop (default false). When
  true, renders the same 5 star glyphs with the same fill logic via non-interactive
  `<span>`s, no hover state, and no trailing `rating-text`. Existing interactive behavior
  is unchanged for all current callers (add/edit forms) that don't pass `readOnly`.
- `src/components/PipMeter.js` (new): 10-dot meter for the perfume 0-10 scale. Mirrors
  StarRating's prop shape (`value`/`onChange` instead of `rating`/`onRatingChange`, plus
  `readOnly`) so a later task can drop it into `reviews/perfumes/add.js` in place of the
  current plain range input without changing the `rating` (0-10 integer) contract.
