# T074 — Perfume "Application notes" field — backend

## What changed

- `src/lib/perfumes.js`: added `validateApplicationSpots(value)` — true only for an array whose
  elements are each one of the fixed six strings (`Wrists`, `Elbows`, `Clavicles`, `Beard`,
  `Back of neck`, `Behind ears`); empty array passes; non-array fails. Exported alongside existing
  validators.
- `src/pages/api/reviews/perfumes/add.js` and `.../update.js`: destructure `applicationSpots` from
  the body, validate it with `validateApplicationSpots` when present (400 on invalid), and persist
  it on the item conditionally (`...(applicationSpots !== undefined && { applicationSpots })`) —
  matches the existing `seasons` optional-field pattern exactly. Omitting the field still succeeds.
- Did not touch the existing `notes` field or its handling, per the task note (label/placeholder
  change is T075's concern).
