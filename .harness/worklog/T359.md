# T359 — hot takes edit/delete flow

2026-07-10

## What I did

Re-attempt of T356 (blocked by the loop's own bracket-glob scope_match bug, since
fixed). Same scope, same spec, built fresh:

- `src/pages/api/hot-takes/update.js` — POST, password-gated, validates `text` via
  `validateHotTakeText`, `GetCommand`s the existing item to preserve `date`, `PutCommand`s
  `{ id, text, date }` back, clears `api-hot-takes` cache.
- `src/pages/api/hot-takes/delete.js` — DELETE, password-gated, validates `id`,
  `DeleteCommand`s by `{ id }`, clears cache.
- `src/pages/hot-takes/edit.js` — fetches `/api/hot-takes`, sorts newest-first via
  `sortByDateDesc`, each row links to `/hot-takes/edit/<id>` (mirrors the `review-edit-card`
  pattern used by the perfumes/movies/tv/books/albums edit lists).
- `src/pages/hot-takes/edit/[id].js` — fetches and finds the take by id, pre-fills a
  `MarkdownEditor` + password field, POSTs to `/api/hot-takes/update` on submit, DELETEs to
  `/api/hot-takes/delete` on Delete (with a `confirm()` prompt), redirects to
  `/hot-takes/edit` after ~2s on either success — same shape as
  `src/pages/reviews/perfumes/edit/[id].js`.

No changes to `src/pages/hot-takes/index.js` or `src/lib/hotTakes.js` (reused as-is, per spec).
