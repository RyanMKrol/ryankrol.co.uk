# T150 — poster-banner ReviewCard variant + movies list rebuild

## What changed

- `src/components/ReviewCard.js`: added a `styleVariant === 'poster-banner'` branch. Renders a
  gradient banner (reusing `gradientForKey` exported from `CoverTile.js` for a stable per-item
  hash-based gradient — no image assets, keyed off `item.id` falling back to title) with the title
  overlaid at the bottom, then a body with read-only `StarRating`, a mono right-aligned date, and
  an italic review-text snippet. Built generically (keys off `type`-agnostic helpers already in the
  component: `getTitle()`/`getRating()`/`getThoughts()`) so TV can reuse it unchanged in T151.
- `src/pages/reviews/movies/index.js`: rebuilt around the new variant — big "movies" title +
  meta line (`N rated · avg X.X★ · N this year`, computed from the real fetched array, this-year
  count derived from `editedDate || date` in `DD-MM-YYYY` format), `SearchInput` for title search
  (same filter logic as before), `PillGroup` for date/title/score sort (date default, replacing
  `SortButtons` on this page only — `SortButtons.js` left in place since other review pages aren't
  redesigned yet), and a 3-column `poster-banner-grid` of `ReviewCard`s.
- `src/styles/globals.css`: added `.poster-banner-card/-banner/-title/-body/-meta/-date/-snippet`,
  the `.poster-banner-grid` (3 col → 1 col under 900px), and a `.collection-review-header` group for
  the title+meta / search+sort row layout.

## Verify

- `npm run lint`: 0 errors (5 pre-existing warnings, unchanged).
- `npm test`: 35 suites / 237 tests passing.
- `npm run build`: succeeds.
- `run-app`: `npm run dev`, `GET /reviews/movies` → 200, confirmed `/api/reviews/movies` returns
  real DynamoDB-backed review data (id/title/rating/date/review_text) that the page renders through
  the new poster-banner cards. No headless browser available in this environment to capture a
  screenshot, so pixel-level fidelity against `design_handoff/renders/2a.png` / `4b.png` is left to
  the dedicated visual-audit task (T166) as the spec allows.
- Considered a co-located `index.test.js` for the pure summary-math helper, but Next's page build
  treats any file under `src/pages/**` as a route and `describe`/`it` globals aren't defined at
  build time (`ReferenceError: describe is not defined` collecting page data) — removed it rather
  than fight the framework for an out-of-scope `src/lib` extraction (task's `expectsTest` is
  `false` and scope doesn't include a new `src/lib` file).
