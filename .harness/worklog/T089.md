# T089 — Require a confirmed Last.fm match to add a vinyl entry

## What changed

- `src/pages/vinyl/add.js`:
  - `handleSubmit` now returns early (without saving) and shows an inline error message
    ("Search and select a Last.fm match before saving") if `lastfmMatch` is null.
  - The submit body always includes `lastfm: lastfmMatch.lastfm` (no longer conditional,
    since submission is now blocked without a match).
  - The Last.fm field label changed from "Last.fm Match (optional)" to "Last.fm Match *",
    with an inline hint shown under the search widget whenever no match is selected yet.
  - Submit button is now disabled when `saving` OR `!lastfmMatch`.
- `src/pages/api/vinyl/add.js`: added a required-field check — requests missing `lastfm`
  are now rejected with HTTP 400, in the same style as the existing title/artist check.
- Added `src/pages/api/vinyl/add.test.js` (no prior test file existed for this route),
  covering: reject-without-lastfm (400, no DynamoDB write) and succeed-with-lastfm (201,
  cache cleared) — mirrors the existing `reviews/albums/add.test.js` pattern.

## Verified

- `npm run lint` — 0 errors (pre-existing warnings only, unchanged).
- `npm test` — 21 suites / 161 tests pass, including the 2 new vinyl add tests.
- `npm run build` — succeeds.
- Did not make any live Last.fm/DynamoDB calls; verified by reading code + running the
  local suite per harness rules.
- Edit/update flow (`src/pages/vinyl/edit/[id].js`, `src/pages/api/vinyl/update.js`) was
  not touched, per the task's scope.
