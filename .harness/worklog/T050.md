# T050 — Variant 1 — workouts + exercises pages

## What was built

- `src/pages/v1/workouts/index.js` — dense workout table (Title/Date/Duration/Volume/Sets),
  push/pull/legs filter pills, sortable columns, compact `1–50 of N ‹ ›` pager. Row click
  navigates to `/v1/workouts/[id]` (matches production's separate-page detail model — this
  chunk's routes are individual page navigations, not the review-table slide-in panel).
- `src/pages/v1/workouts/[id].js` — workout stat block (date/time/duration/volume/sets) as a
  dense label:value stack, then an exercise table (name links to `/v1/exercises/[name]`,
  volume/1RM right-aligned, sets rendered inline).
- `src/pages/v1/exercises/[exerciseName].js` — reuses production's stat/date-range logic
  verbatim, renders stats as a dense table, keeps `ExerciseProgressCharts`/
  `CardioProgressCharts` (chart.js, theme-aware via `useChartTheme`), and a session-history
  table linking back to `/v1/workouts/[id]`.

All three fetch from the existing unchanged API routes (`/api/workouts`, `/api/workouts/[id]`,
`/api/workouts/[id]/exercises`, `/api/exercises/history/[exerciseName]`) and reuse
`V1Layout` from T043, plus `filterWorkouts` (`src/lib/workoutPagination.js`) and
`DATE_RANGES`/`filterByDateRange` (`src/lib/dateRange.js`).
