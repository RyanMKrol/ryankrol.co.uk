# T130 — Rich result cards in MetadataBackfillModal

## What changed

Updated the `renderResult` prop passed to `MetadataBackfillModal` on all 4 review
edit pages so results render as thumbnail + title/year + secondary-text cards,
matching the pattern already used by `TmdbSearch.js` / `BookSearch.js` /
`LastfmAlbumSearch.js` in the add flows. `MetadataBackfillModal.js` itself was
not touched.

- `src/pages/reviews/movies/edit/[id].js` — poster via `tmdbPosterUrl(result.posterPath)`
  (60x90, placeholder div if none), title + year from `result.date`, truncated
  `result.overview` (160 chars + ellipsis).
- `src/pages/reviews/tv/edit/[id].js` — same as movies (identical result shape).
- `src/pages/reviews/books/edit/[id].js` — cover via `result.coverUrl` directly
  (40x60, placeholder if none — this route only ever returns Google Books results,
  which never have `coverId`), title + `firstPublishedYear`, `result.authors.join(', ')`.
- `src/pages/reviews/albums/edit/[id].js` — cover via `result.image` (50x50,
  placeholder if none), title, `result.artist`.

Each page kept its own scoped `<style jsx>` block (movies/tv/books gained a new
one; albums appended rules to its existing block) rather than factoring a shared
component, since this task's scope only allowed the 4 edit pages — no new files.
Styling reuses the modal's existing CSS custom properties (`--color-border`,
etc.), matching the repo's theming convention.
