# T211: Client-side search + tag filtering on /projects

## Do

Added purely client-side filtering over the already-fetched `repos` array in
`src/pages/projects/index.js`:

- `searchTerm` state + `SearchInput` (reused component) matching against `repo.name` OR
  `repo.description` (case-insensitive substring, null-safe description).
- `selectedTags` state (array) + a multi-select row of `Pill` components (not `PillGroup`,
  which is single-select-only) rendered in a `.collection-pill-group` wrapper for the
  established pill styling. Distinct topics are derived via `useMemo` from the FULL
  `repo.topics` array (not the per-card `.slice(0, 5)` display cap, which is untouched).
  Clicking a pill toggles it in/out of `selectedTags`.
- Tag filter uses OR/any-of semantics (repo passes if it has ANY selected tag, or no tags
  are selected).
- Search AND tag filter are combined with AND via a `useMemo`'d `filteredRepos`.
- Two distinct empty states: "No active repositories found." when `repos` itself is empty,
  vs. "No projects match your search/filters." when search/tags exclude everything.
- No new API route/network call — everything filters the same single `/api/github/repos`
  fetch already in state.
