# T063 — Variant 3 — listening + projects pages

## What was built

- `src/components/v3/V3ListeningFeed.js` — fetches `GET /api/lastfm/top-albums?period=3month&limit=50`
  (unchanged, same call as production `/listening`), renders each album as a `V3TimelineEntry`
  with a `#rank` gutter, `[album]` type tag, and inline-expandable playcount + Last.fm link —
  matching Variant 3's linear-feed/inline-disclosure pattern (no cards/grid).
- `src/components/v3/V3ProjectsFeed.js` — fetches `GET /api/github/repos` (unchanged), renders
  each repo as a `V3TimelineEntry` with the last-push date in the gutter, the repo's primary
  language as the type tag, and stars/forks/topics/GitHub link in the inline-expanded detail.
- `src/pages/v3/listening/index.js` and `src/pages/v3/projects/index.js` — thin pages that render
  the two feed components inside T045's `V3Layout` (reused, not rebuilt).

Both feeds reuse the same loading/error/empty/`— end of feed —` structure and inline "sorted by …
— change" control already established by `V3VinylFeed.js`, keeping the variant internally
consistent.
