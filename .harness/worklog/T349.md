# T349 — Top of Mind placement comparison (?tom=1..5)

2026-07-10

## What I did

Added a `?tom=1..5` query-param-gated comparison of 5 placement treatments for a future "Top of
Mind" homepage section, built directly into `src/pages/index.js` (real layout, not a standalone
dev page) so placement can be judged in context. `?tom=0` (or any unrecognised value) renders
nothing — identical to the homepage today. Default (`tom` absent) is `1`.

- Added a module-level `DUMMY_TOM_TEXT` markdown constant (`## heading` + paragraph + 3-item
  bullet list), rendered via the existing `Markdown` component in every variant.
- Read `router.query.tom` via `useRouter` (client-side only, matching the page's existing
  client-fetch pattern).
- Variant 1 "Rail panel": new `.tom-variant-rail-panel` appended after `home-hot-takes-panel`
  in `.home-rail`, same panel chrome (title row + body) as its siblings.
- Variant 2 "Hero band": `.tom-variant-hero-band` full-width band inserted between `home-hero`
  and `home-stats`, distinct gold tint (`--color-star-gold`).
- Variant 3 "Collapsible section": native `<details>`/`<summary>` between `home-wall` and
  `home-lower`, collapsed by default with a one-line teaser in the summary.
- Variant 4 "Latest-takes card": `.tom-variant-latest-card` woven into the top of `.home-latest`,
  reusing `.home-latest-card` chrome with a thicker accent-coloured border (`--accent-hot-takes`)
  to read as a non-review item.
- Variant 5 "Hero inline strip": `.tom-variant-hero-strip` compact pill-shaped single-line strip
  inside `home-hero`, under the tagline/meta row — CSS truncates the rendered Markdown to one line
  and hides the heading/list elements (still the same Markdown content/component, just visually
  condensed to fit "minimal height").
- All new CSS is under new `.tom-variant-*` classes (globals.css), no existing `.home-*` rules
  touched.
