# T018 — TMDB search proxy route + normaliser

## What was built

- `src/lib/tmdb.js` — pure normalisation helpers:
  - `mapTmdbResult(raw, type)` → `{ tmdbId, mediaType, title, overview, posterPath, date }`
  - `tmdbPosterUrl(path)` → full `https://image.tmdb.org/t/p/w500<path>` URL or `null`
- `src/lib/tmdb.test.js` — 8 hermetic unit tests covering movie mapping, tv mapping, missing poster, and poster URL edge cases
- `src/pages/api/tmdb/search.js` — GET handler with `?query=` and `?type=movie|tv` params; authenticates via `Authorization: Bearer ${process.env.TMDB_API_TOKEN}` (v4 JWT, never exposed client-side); wraps in `withApiCache(generateCacheKey('tmdb-search', { query, type }))`
- `README.md` — added Required Variables table documenting `TMDB_API_TOKEN`
- `CLAUDE.md` — added `TMDB_API_TOKEN` to env var table, added `tmdb.js` and the search route to the file map and architecture section

## Verification

- `npm run build` → succeeded cleanly; `/api/tmdb/search` appears in route listing as dynamic (ƒ)
