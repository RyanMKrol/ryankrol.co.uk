# T142 — Install Collection design tokens as the site's single palette

## Built interactively (gated task, owner-directed session)

This task is `"gate": "gate"` — never routed through `.harness/loop.sh`. Built and verified directly
per this repo's documented interactive-completion pattern, then marked done by hand.

## A real decision surfaced mid-build

The original spec (as I'd authored it) said to fully replace `:root`'s token vocabulary. Before
touching any code I checked how much of the live site still depends on the OLD token names
(`--color-bg`, `--color-text`, `--radius-base`, `--btn-bg`, etc.) — 71 CSS rules and ~47 page/
component files, none of which get rebuilt until T149-T170. A full replace right now would render
nearly the entire live, auto-deployed site unstyled until the whole redesign backlog finishes.

Surfaced this tradeoff to the owner directly (additive-tokens-then-cleanup vs. replace-now-accept-
breakage). Owner explicitly chose **replace now** — full commit to the new palette immediately,
accepting that unmigrated pages render broken until their own task lands, rather than carry legacy-
token debt (and the risk of a forgotten cleanup task) through the whole migration. Recorded this
decision in `.harness/tasks/T142.md` so it doesn't get re-litigated by a future rebuild.

## What was built

- `src/pages/_document.js`: added Bricolage Grotesque (400/600/700/800), Nunito
  (400/500/600/700/800 + italic 500), Space Mono (400/700 + italic 400) to the Google Fonts link,
  alongside JetBrains Mono (kept for Matrix). Dropped Share Tech Mono (confirmed zero remaining
  references anywhere in `src/` once `:root` no longer needs it). Updated the anti-FOUC
  `<Html style={{backgroundColor}}>` hardcode from `#0d1b2a` to `#FBF7EF`.
- `src/styles/globals.css` `:root`: replaced the entire legacy token block with the Collection set —
  color (bg/ink/ink-soft/ink-mute/card/hairline/hairline-strong/star-gold/star-empty), 8 section
  accents, perfume pip-meter colors, workout warmup/working-row colors, typography
  (`--font-display`/`--font-body`/`--font-mono`), shape (`--radius-card` 18px/`--radius-pill`
  999px/`--radius-cover` 10px/`--radius-stat` 16px/`--border-width` 2px/`--border-color`/
  `--shadow-cover`), spacing (`--gutter-desktop`/`--gutter-mobile`) — exact hex/px values from
  `design_handoff/README.md`'s Design tokens table.
- **Extra fix beyond the original spec**: the Matrix override block (`html.matrix-active`) never
  explicitly declared a font — it always inherited whatever `:root` set, which was previously Share
  Tech Mono (incidentally terminal-appropriate). Since the new base `--font-body` is Nunito (a
  rounded sans, not terminal-appropriate), added an explicit `--font-display`/`--font-body`/
  `--font-mono: 'JetBrains Mono', monospace` override inside `html.matrix-active` so the terminal
  aesthetic survives the base font change. Every other Matrix color/shadow/background value in that
  block was left byte-for-byte untouched.
- `CLAUDE.md`: updated the stale 🎨/`AppearancePicker` line in the Components list (that system no
  longer exists post-T141), updated the `globals.css` file-map row, and rewrote the "Theming"
  convention bullet to describe the single Collection palette (no theme-switching mechanism, prior
  AppearancePicker system explicitly noted as retired so it isn't reintroduced) plus an explicit
  note that the site is mid-redesign and unmigrated pages rendering unstyled is expected, not a bug.
- `README.md`: checked for any theming/design content to update — found none (it's purely
  setup/schema documentation; CLAUDE.md is the architecture doc). No changes needed.
