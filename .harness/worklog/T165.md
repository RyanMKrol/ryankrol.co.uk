# T165: Visual audit & tweak — home page vs design renders

Compared the live home page (desktop ~1400px, mobile ~390px) against
`renders/1a.png` / `renders/4a.png`, plus the shared components in isolation
against the token spec in `design_handoff/README.md`.

Most of the page was already faithful (colors, radii, fonts, stat blocks,
wall grid, latest-takes cards, gym/shelf rail all matched hex-for-hex). Gaps
found and fixed:

1. **Home container was capped at 800px, not the design's 1400px.** The
   shared `.container` class (a pre-redesign token) was forcing the nav pills
   to wrap onto two lines and squeezing the whole home layout into a narrow
   column, even at a 1400px viewport. Added a home-page-only `.home-container`
   override (max-width 1400px, `--gutter-desktop` padding) applied alongside
   `.container` in `index.js`, rather than widening the shared class — other
   pages still using the narrow `.container` are out of this task's scope and
   weren't touched.
2. **Hero title (`Howdy.`) was capped at 96px, spec calls for 132px.** Bumped
   `.home-hero-title`'s `clamp()` ceiling from `6rem` to `8.25rem`.
3. **Mobile now-playing indicator was crammed into the header row**, clipping
   against the hamburger button. The design (`4a.png`) removes it from the
   header entirely on mobile and shows it as its own full-width bordered bar
   below the hero tagline. Added a second `<NowPlaying />` render in the hero
   section, toggled via CSS (`.home-hero-now-playing`/`.header-now-playing`
   `display` swap at the existing 768px breakpoint) so only one is visible at
   a time per breakpoint.
4. **"Latest takes" fallback thumbnails were a flat beige square** for items
   without real cover art, while the design (and the collection-wall tiles
   right above them) use vivid color gradients. Reused `gradientForKey` from
   `CoverTile.js` for the latest-takes thumb fallback instead of
   `var(--color-hairline-strong)`.

## Verified
- `npm run lint` — 0 errors (pre-existing `no-img-element` / `exhaustive-deps`
  warnings only, unchanged).
- `npm test` — 241/241 passing.
- `npm run build` — succeeds (had to work around a `.next` corruption caused
  by a `next build` racing a concurrently-running `next dev` from an earlier
  screenshot pass — unrelated to these changes; `rm -rf .next` + rebuild
  fixed it).
- Manual: screenshotted the live dev server at 1400px and 390px viewports
  before/after each fix and compared side-by-side against `1a.png`/`4a.png`.
  Confirmed nav now sits on one row at desktop width, hero type scale matches,
  and the mobile now-playing bar sits below the tagline as its own element
  instead of overlapping the header. Did not call any live third-party API —
  compared against whatever data the dev server's existing DynamoDB/Last.fm
  cache returned.
- No functional regression: nav links/search/sort/pagination weren't touched;
  the only interactive surface changed is the duplicated `NowPlaying` mount,
  which reuses the existing component unmodified.
