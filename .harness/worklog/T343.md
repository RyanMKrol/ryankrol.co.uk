# T343 — Perfume card rating/best-for layout variants (dev preview)

2026-07-09

## What I did

- `src/components/perfumeVariants/Variant6Hybrid.js`: added a `variant` prop
  (`'baseline' | 'big-rating' | 'header-rating' | 'best-for-bottom'`, default `'baseline'`)
  that toggles the rating number's size/position and the 'Best for' block's position via
  additive JSX branching (no forked component files, no existing CSS rule touched):
  - `baseline` — unchanged, exact current layout.
  - `big-rating` — rating number enlarged (1.35rem, new class `perfume-t343-rating-number-big`)
    in its current position under the description.
  - `header-rating` — rating row rendered in the header, under the title, before the
    description; uses the same enlarged class plus a new `margin-top` rule scoped to
    `.perfume-v1-header .perfume-v1-rating-row` (doesn't touch baseline placement).
  - `best-for-bottom` — 'Best for' block relocated to after the longevity/projection
    scales block, above the application-spots checklist / Fragrantica link / date.
- `src/styles/globals.css`: added two new rules only (`perfume-t343-rating-number-big`,
  `.perfume-v1-header .perfume-v1-rating-row`) — no existing `.perfume-v1-*`/`.perfume-v4-*`/
  `.perfume-v6-*` rule was modified or removed.
- `src/pages/dev/perfume-card-variants.js` (new): fetches real data from
  `GET /api/reviews/perfumes` (same pattern as `reviews/perfumes/index.js`), takes the first
  3 records, and renders each of the 4 variants as a labelled section with the same 3 cards
  side by side. Left ungated (ephemeral dev-only page, deleted once a winner is picked —
  same precedent as T187's `/dev/perfume-variants` cleanup).
- `scripts/_visual-harness.mjs`: added a `PAGES` entry `dev-perfume-card-variants` for
  `/dev/perfume-card-variants`, `waitFor: ['.perfume-v6-card']`.
