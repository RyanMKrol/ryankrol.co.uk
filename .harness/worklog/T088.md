# T088 — Require a confirmed Last.fm match to add an album review

## What changed

- `src/pages/reviews/albums/add.js`: `handleSubmit` now blocks and shows an inline error
  message ("Search and select a Last.fm match before saving") if `lastfmMatch` is null.
  Submit button is disabled until a match is selected. Label changed from "(optional)" to
  required, with an inline hint shown while no match is picked. `lastfm` is now always sent
  in the payload (guaranteed present since submit is blocked otherwise).
- `src/pages/api/reviews/albums/add.js`: added `lastfm` to the required-fields check —
  requests missing it now get HTTP 400, matching the existing error-response style.
- Added `src/pages/api/reviews/albums/add.test.js` (new file, no prior test existed for this
  route): asserts 400 + no DynamoDB write when `lastfm` is missing, and 201 + cache clear when
  `lastfm` is present alongside the other required fields.
