# T324 — 2026-07-07

## What I did
`.collection-review-controls` (src/styles/globals.css, was line 452) only wrapped onto its own
row under `.collection-review-header`'s `flex-wrap: wrap` when it happened to be too wide to
share the title's row (perfumes/projects/vinyl, which also render a pill row). Narrower controls
(movies/tv/books/albums/workouts — just search+sort) instead sat right-aligned next to the title,
an accidental difference driven purely by content width rather than an authored style.

Added `flex-basis: 100%;` to `.collection-review-controls` so it always claims the full row width
and drops to its own line under the existing `flex-wrap: wrap`, then left-aligns by default flex
behaviour as the lone child on that row — the same mechanism perfumes/projects/vinyl already
exhibited, now made universal and deliberate. No JSX touched; single CSS-only change.

Confirmed scope of the fix with `grep -rn collection-review-controls src/pages`: matches
workouts/index.js, projects/index.js, vinyl/index.js, reviews/{books,perfumes,tv,movies,albums}/index.js
— all 8 current consumers, all covered by one shared rule change.
