# T066 — Shared metadata-backfill lookup+confirm modal component

## What was built

`src/components/MetadataBackfillModal.js` — a generic button → search → modal → select →
confirm/cancel component, parameterised entirely by the caller:

- `buttonLabel` — trigger button text (default `'Backfill from API'`)
- `onSearch()` — caller-provided async function returning candidate results
- `renderResult(result)` — caller-provided renderer for one candidate (title/cover/year/etc.)
- `onConfirm(result)` — called with the selected candidate on confirm
- `getResultKey(result, index)` — optional React key extractor

No review-type-specific field names, endpoints, or logic are hardcoded — the component only
knows about a generic `results` array and delegates all rendering to `renderResult`.

## Behavior implemented

- Clicking the trigger button opens the modal and immediately runs `onSearch()`.
- Handles zero results ("No matches found."), a single result, and multiple results (each
  rendered as a radio-selectable row via `renderResult`).
- Confirm is disabled until a candidate is selected; clicking it calls `onConfirm(selected)`
  and closes the modal.
- Cancel/close (✕ button, backdrop click, or Escape key) closes without calling `onConfirm`.
- Styled with `styled-jsx` (self-contained, following `BookSearch.js`'s pattern) since the
  overlay/card look in `AppearancePicker.js` lives in `globals.css`, which is out of this
  task's scope (`src/components/**` only).
