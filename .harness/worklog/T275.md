# T275 — Wire the shared Markdown editor into all 5 review 'add' forms

## What changed

Swapped the plain `<textarea className="collection-form-textarea" .../>` for
`MarkdownEditor` (from `src/components/MarkdownEditor.js`, built in T274) in:

- `src/pages/reviews/movies/add.js` (field `gist`)
- `src/pages/reviews/tv/add.js` (field `gist`)
- `src/pages/reviews/books/add.js` (field `overview`)
- `src/pages/reviews/albums/add.js` (field `highlights`)
- `src/pages/reviews/perfumes/add.js` (field `description`)

Each swap is like-for-like: same `id`/`name`/`value`/`onChange`/`className`/`placeholder`/
`required` props, same `formData` state and `handleInputChange`/submit logic untouched.
Nothing else in the forms (search widgets, star ratings, password gating) was touched.

## Tests

Added one RTL test per page under `src/__tests__/pages/reviews/<type>/add.test.js`
(NOT co-located under `src/pages/` — Next's pages router treats every `.js` file
directly under `src/pages/**` as a route, including `.test.js` files with no
default export, which broke `next build`'s "Collecting page data" step with
`ReferenceError: jest is not defined`; moving them to a sibling `src/__tests__/`
tree keeps them outside the pages router while still matching Jest's
`**/?(*.)+(test|spec).[jt]s?(x)` testMatch). Each test renders the add page
(mocking `next/router` and `Header`, since `Header` needs `ResizeObserver`,
unavailable in jsdom) and asserts the MarkdownEditor toolbar (Bold/Italic/Bullet
list/Link) and the field's placeholder text are present.
