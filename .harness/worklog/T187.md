  container — Variant 6 is a normal card, so the 2-col grid still fits, no layout change needed.
- `src/components/ReviewCard.js`: removed the dead `PerfumeCard` helper and the
  `styleVariant === 'perfume-card'` branch, plus now-unused `PipMeter`/`Badge` imports. Verified
  `src/pages/reviews/perfumes/edit.js` passes `styleVariant={2}` (an integer, not the string
  `'perfume-card'`) so it already falls through to the generic default card — unaffected.
- Deleted `Variant1StatBlock.js`, `Variant2Editorial.js`, `Variant3ShelfTile.js`,
  `Variant4SeasonPlanner.js` — only `Variant6Hybrid.js` remains.
- Deleted `src/pages/dev/perfume-variants.js` preview page.
- `src/styles/globals.css`: removed the dead `.perfume-card`/`.perfume-card-body`/etc. classes
  (kept `.perfume-card-grid`, still used as the container), all `.perfume-v2-*` and
  `.perfume-v3-*` rules, and the `.perfume-v1-*`/`.perfume-v4-*` rules not referenced by
  Variant 6 (e.g. `.perfume-v1-stats`, `.perfume-v1-fragrantica-link`, `.perfume-v4-card`,
  `.perfume-v4-header`). Kept every class Variant 6 actually renders.
- `CLAUDE.md`: updated the `ReviewCard` component description (dropped `perfume-card` from its
  variant list, noted perfumes now render `Variant6Hybrid` directly) and added a file-map row for
