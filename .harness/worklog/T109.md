# T109 — MovieRatingsV4 id-migration script

## What was built
- `src/scripts/migrateMoviesToIdTable.js` — one-shot migration script that creates
  `MovieRatingsV4` (id HASH key, PAY_PER_REQUEST), scans all of `MovieRatingsV3`
  (paginated), copies every field unchanged + appends a generated `randomUUID()` id,
  batch-writes in chunks of 25, guards against double-migration (aborts unless
  `--force`), and runs a field-by-field `--verify` pass automatically after migrating.
  Also supports standalone `node src/scripts/migrateMoviesToIdTable.js --verify`.
- `src/scripts/migrateMoviesToIdTable.test.js` — unit tests for the pure
  `buildMigratedItem` transform (field preservation, id presence, UUID v4 shape) and
  `diffMigratedItem` comparison (clean match, changed field, missing item) and
  `findMatchingNewItem`.
- `MovieRatingsV3` was NOT modified or deleted — only scanned.
