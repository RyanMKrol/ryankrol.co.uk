# T230 — cap + rank the projects tag-filter pill row

## What changed
`src/pages/projects/index.js`:
- `allTags` now counts occurrences of each topic across `repos` (one `Map` pass), then sorts
  descending by count, tie-breaking alphabetically — was a plain alphabetical `Set`.
- Added `TAG_DISPLAY_LIMIT = 12` and `tagsExpanded` state.
- New `visibleTags` memo: while collapsed, shows the top 12 by frequency UNIONED with any
  `selectedTags` that fall outside the top 12 (so an active filter never disappears behind the
  fold); while expanded, shows every tag.
- Pill row renders `visibleTags`; when `allTags.length > TAG_DISPLAY_LIMIT` an extra plain
  (no `active`/`accentColor`) toggle `Pill` is appended reading `+N more` / `show less`.
- `toggleTag`, `selectedTags`, and `filteredRepos`'s OR-matching logic are untouched.

`CLAUDE.md`: updated the `src/pages/projects/index.js` file-map row to mention the
frequency-ranked, capped, show-more/show-less tag row.

  tags; clicking the toggle pill flips `tagsExpanded`; with `<=12` total tags the toggle pill
  is never rendered (`allTags.length > TAG_DISPLAY_LIMIT` guard) so today's small-list behaviour
  is unchanged.
