# T164: restyle perfumes add/edit + vinyl add forms to Collection tokens

## What changed

- `src/pages/reviews/perfumes/add.js`, `edit/[id].js`: swapped legacy
  `form-container`/`form-group`/`form-label`/`form-input`/`form-textarea`/`form-button`/
  `success-message`/`error-message`/`btn-danger` classes for the `collection-form-*` set
  already established by the books/albums/movies/tv forms (T161-T163). Replaced the
  perfume rating range/number input with the interactive `PipMeter` component (0-10 pip
  scale matches `PIP_COUNT = 10` exactly) — the underlying `rating` state stays an
  integer 0-10, only the input widget changed. All perfume-specific fields (designer,
  type, fragranticaUrl, considerTravelSize, considerFullBottle, longevity, projection,
  seasons, applicationSpots) are untouched functionally and still rendered on the form —
  `PerfumeCharacteristics.js` (out of scope) keeps its own legacy classes internally,
  which is expected per T164's scope list.
- `src/pages/vinyl/add.js`: same `collection-form-*` swap; renamed the page-local
  `style jsx` classes (`form-hint`, `lastfm-cover-preview`, `lastfm-preview-img`,
  `lastfm-preview-label`) to `vinyl-`-prefixed names using CSS custom properties
  (`var(--font-body)`, `var(--color-ink)`, `var(--radius-cover)`) instead of the
  hardcoded `4px` border-radius. The Last.fm search flow (`LastfmAlbumSearch`) itself
  was already fully token-based from earlier work — no changes needed there.
- `src/pages/reviews/perfumes/edit.js` (list page): already used token-based classes
  (`reviews-wrapper`, `review-card-with-edit`, `edit-button-overlay`, all `var(--...)`
  in `globals.css`) — no legacy hardcoded colors found, so left unchanged.
- `src/styles/globals.css`: bumped `.pip-dot.interactive` to `1.1rem` (up from the
  `0.6rem` base `.pip-dot` size) for a usable click/tap target in the form context.
  `ReviewCard.js` always passes `readOnly` to `PipMeter`, so this only affects the two
  interactive forms, not the perfume list-card display.
