## What I did

`src/components/BulkBackfillList.js`:
- Capped candidates at 3 at store time in `onItemSettled` (`candidates.slice(0, 3)`), so both the
  radio list and the existing `statusLabel()` count derive from the capped array automatically.
- Added `applyingAll` state + a `confirmAll` handler that sequentially `await`s the existing
  `confirmRow(item)` for every row currently `status: 'found'` with a selection — reusing the
  save/error logic as-is rather than routing through `runBackfillQueue` (that queue's
  rate-limit/retry machinery is built for the search phase, not the site's own unthrottled,
  password-gated update routes).
- Rendered one page-level "Apply all selections" `.bbl-page-actions` button (disabled while
  applying or with nothing selected), removed the per-row `.bbl-row-actions` Confirm button/CSS
  entirely — "Apply all selections" is now the only save path.
- Fixed a latent bug the spec's Done-when assumes is already true: `statusLabel()`'s `'found'`
  case never surfaced `row.error` (only the `'error'` search-failure case did), so a failed save —
  which reverts status to `'found'` with `error` set — showed the stale candidate count instead of
  the failure message. Now `'found'` prefers `row.error` when present.

`scripts/_visual-harness.mjs` (exempt from scope per `SCOPE_EXEMPT_GLOBS=scripts/`):
- Added 2 movies fixtures without `tmdbId` (Blade Runner, Moon) so `/reviews/movies/backfill` has
  rows to show, plus a 5-result `/api/tmdb/search` fixture so the 3-candidate cap is visible.
- New `PAGES` entry `reviews-movies-backfill`.
- New `FLOWS` entry `reviews-movies-backfill-apply-all`: selects the first candidate on both rows
  (waiting through the queue's 1500ms inter-row spacing), clicks "Apply all selections", waits for
  both rows to read "saved".

`src/components/BulkBackfillList.test.js` (new, RTL, matches `PillGroup.test.js` style): 4 tests
covering the cap, button disabled-state (idle + selected + applying), selective apply (only rows
with a selection get `onConfirm`), and partial-failure continuation (failed row keeps its
selection + shows the error, later rows still get confirmed).
