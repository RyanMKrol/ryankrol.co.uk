# T318 — fix invisible Longevity/Projection bars on perfume cards

2026-07-07

## What
`Tooltip`'s root `.tooltip-wrapper` is `display: inline-flex` with no explicit width, so it
shrink-to-fits. The `width:100%` `.perfume-v4-scale-track` child inside it (added in commit
0708eac, T301) resolved its flex-basis against that indefinite width and collapsed to ~0.

Fix: gave both scale-bar `<Tooltip>` usages in `Variant6Hybrid.js` (longevity ~line 83,
projection ~line 99) a `className="perfume-v4-scale-tooltip"` (the component already appends
`className` onto `tooltip-wrapper`, `Tooltip.js:4`). Added a new, narrowly-scoped rule in
`globals.css` right above `.perfume-v4-scale-track`:

```css
.perfume-v4-scale-tooltip {
  display: flex;
  width: 100%;
}
```

This stretches only this usage's wrapper to the full width of `.perfume-v4-scale`, so the
`width:100%` track child resolves against a definite width again. The shared `.tooltip-wrapper`
base rule (globals.css:3192) and `.perfume-v4-scale-track`/`-fill` rules were left untouched, per
the task's constraints.
