## What
Added click-and-drag scrolling to the header's `.collection-nav-pills` row
(`src/components/Header.js`) as a fallback for pointers without native horizontal-scroll
input (confirmed unreachable on a plain Windows mouse ‚Äî no trackpad swipe, no
horizontal/tilt wheel).

- `onMouseDown` on the `<nav>` records the starting pointer X and the nav's current
  `scrollLeft` in a ref (`dragRef`, not state ‚Äî avoids re-render churn on every
  `mousemove`), and attaches `mousemove`/`mouseup` listeners on `window` (so the drag keeps
  tracking even if the pointer leaves the nav's bounds, and always ends cleanly on mouseup
  wherever it fires).
- `mousemove` sets `nav.scrollLeft` directly (`startScrollLeft - deltaX`) ‚Äî this is the same
  property the existing `scroll`/`resize`/`ResizeObserver` effect (`showNavFade` /
  `shouldShowScrollFade`) already listens to, so that effect keeps working completely
  unmodified.
- `mouseup` tears down the window listeners and, if total movement exceeded
  `DRAG_CLICK_THRESHOLD_PX` (5px), sets a `suppressClickRef` flag consumed by a new
  `onClickCapture` on the `<nav>` (`e.preventDefault(); e.stopPropagation()`) ‚Äî stops a
  drag-ending-over-a-pill from also firing that pill's `<Link>` navigation, while a plain
  click (movement under the threshold) still navigates normally.
- `e.preventDefault()` in `onMouseDown` + a `.collection-nav-pills.is-dragging` CSS class
  (`user-select: none`, `cursor: grabbing`) stop native text-selection/link-drag-ghost
  during the gesture; `cursor: grab` at rest (`src/styles/globals.css`).
- Cleanup effect removes any still-attached window listeners on unmount (belt-and-suspenders
  for an unmount mid-drag; the normal path removes them at `mouseup`).
