# T241 — fix masonry reading order on /reviews/movies

## What changed
- `src/styles/globals.css`: `.poster-banner-grid` switched from CSS multi-column
  (`column-count: 3`, mobile override `column-count: 1`) to a flex row
  (`display: flex; gap: 1.5rem`) + new `.poster-banner-grid-col`
  (`display: flex; flex-direction: column; gap: 1.5rem; flex: 1 1 0; min-width: 0`).
  Removed `margin-bottom: 1.5rem` from `.poster-banner-card` (redundant with the
  column's flex `gap`), kept `break-inside: avoid`.
- `src/pages/reviews/movies/index.js`: replaced the raw
  `<div className="poster-banner-grid">{pagedMovies.map(...)}</div>` block with
  `MasonryColumns` (`columnCount` from `useResponsiveColumnCount(3, 900)`),
  passing `columnClassName="poster-banner-grid-col"`. `movieGradients[index]`
  still lines up because `MasonryColumns.renderItem` hands back each item's
  original index in the full `pagedMovies` array. Moved the
  `useResponsiveColumnCount` call above the loading/error early returns to
  satisfy the rules of hooks.

## Verification
- `npm run lint` — 0 errors, 6 pre-existing warnings (img elements, one
  exhaustive-deps warning on `dev/cache.js`), no new warnings.
- `npm test` — 48 suites / 308 tests pass.
- `npm run build` — succeeds, `/reviews/movies` builds statically.
- Manual run-through not performed live (headless session) — relied on reusing
  the already-verified `MasonryColumns`/`useResponsiveColumnCount`/
  `distributeRoundRobin` primitives from the Projects page migration, plus a
  code read confirming `distributeRoundRobin` round-robins items into columns
  in reading order (item 0 → col 0, item 1 → col 1, item 2 → col 2, item 3 →
  col 0 row 2, ...), so sorting `pagedMovies` before it's fed into
  `MasonryColumns` naturally produces left-to-right, top-to-bottom reading
  order regardless of variable card heights.
