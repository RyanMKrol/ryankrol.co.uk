# T015 worklog — Appearance picker modal

## What was built

**AppearancePicker.js** — new component:
- Overlay + card modal with `role="dialog"` and `aria-modal`
- THEME section: 5 tiles (Cyberpunk, Terraria, Cotton Candy, Sunset, Ocean) with emoji + name
- FONT section: 6 tiles (Theme Font, Share Tech Mono, VT323, Nunito, Space Mono, JetBrains Mono) with per-tile `fontFamily` style
- MODE section: 3 tiles (Dark, Light, System) with helper text
- Reduce motion checkbox wired to `setMotion`
- Esc key closes; click outside overlay closes; × button closes
- All controls wired to `useTheme` — changes apply live and persist to localStorage

**Header.js** — added:
- `useState` for `pickerOpen`
- `header-appearance-btn` (🎨) button that opens the picker
- `<AppearancePicker>` rendered inside `<header>`

**globals.css** — added:
- `html[data-font="X"]` override rules for all 5 explicit fonts (share-tech-mono, vt323, nunito, space-mono, jetbrains-mono). Placed after theme blocks so same-specificity cascade wins. `data-font="theme"` has no rule, so the active theme's own `--font-body` wins.
- Full `.ap-*` CSS: overlay, card, header, close button, section labels, tile grid, tiles, active state, helper text, reduce-motion label, mobile responsive breakpoint.
- `.header-appearance-btn` styles.

## Observed

- `npm run build` → clean pass, 28 static pages generated, 0 errors. Pre-existing `react-hooks/exhaustive-deps` warning in `dev/cache.js` unchanged.
- Font override rules correctly placed after theme blocks — equal specificity but later position ensures `data-font` beats `data-theme` for font vars.
- `data-motion="reduced"` was already wired by T013 to stop Matrix rain and kill animations; the checkbox in AppearancePicker surfaces that same toggle.
