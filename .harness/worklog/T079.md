# T079 — migrate Open Library books to Google Books

## What was built

- `src/scripts/migrateOpenLibraryBooksToGoogleBooks.js` — dry-run-by-default migration script.
  - Scans `BookRatingsV3` for `source === 'openlibrary'` reviews.
  - Searches Google Books (same query shape as `src/pages/api/books/search.js`) for each,
    reusing `mapGoogleBooksResult` from `src/lib/googlebooks.js`.
  - `findConfidentMatch`: confident only when the top result's normalised title exactly matches
    the review's normalised title; otherwise logs "no match — left as openlibrary" and skips.
  - `computeUpdateFields`: maps a confident match to
    `{ source, volumeId, coverUrl, bookAuthors, firstPublishedYear, isbn, subjects, pageCount, publisher }`
    — never touches `title`/`author`/`rating`/`overview`.
  - Real writes (via `UpdateCommand`, which also `REMOVE`s `olid`/`coverId`) only happen when
    `LIVE=1` is set; default run only prints what it would change.
- `package.json` — added `"books:migrate-openlibrary": "node src/scripts/migrateOpenLibraryBooksToGoogleBooks.js"`.
- `src/scripts/migrateOpenLibraryBooksToGoogleBooks.test.js` — unit tests against fixture review
  objects and a mocked Google Books search response (no network/AWS calls):
  - confident match computes the correct field mapping and clears none of title/rating/overview.
  - unmatched fixture is left alone and reported as unmatched.

## Verification

- `npm run lint` — 0 errors (4 pre-existing warnings, unrelated to this change).
- `npm test` — 147 passed, 15 suites, including the new test file.
- `npm run build` — succeeded.
- No real AWS or Google Books API calls were made during verification — the test file uses only
  in-process fixture data and the pure exported functions (`normaliseTitle`, `findConfidentMatch`,
  `computeUpdateFields`); it never invokes `migrate()`/`main()` or hits the network/DynamoDB.

## Notes for the human-gated live run (T080)

Running `LIVE=1 npm run books:migrate-openlibrary` against production requires `.env.local` with
real `AWS_ACCESS_KEY_ID`/`AWS_SECRET_ACCESS_KEY` and ideally `GOOGLE_BOOKS_API_KEY` — this was
intentionally never invoked live as part of this task.
