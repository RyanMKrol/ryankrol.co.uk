# T046 — Variant 1 — reviews list/index pages

Built `/v1/reviews/{movies,tv,books,albums,perfumes}` per Variant 1's "Dense Data-Table
Console" brief, reusing T043's `V1Layout`.

## What was built

- `src/components/v1/V1ReviewTable.js` — shared table component: filter input, sortable
  columns (Title/Rating/Reviewed, click header to toggle asc/desc), zebra-striped
  `<table>` rows, 50-row-page compact pager (`n–m of total  ‹ ›`), row click opens a
  420px right-hand slide-in detail panel (Esc-free close via × button) matching the
  brief's spreadsheet-row-inspector description. Table narrows and drops the
  Info/Excerpt columns while the panel is open, per the brief's "table stays visible at
  reduced width" note.
- Five thin page files under `src/pages/v1/reviews/` (`movies.js`, `tv.js`, `books.js`,
  `albums.js`, `perfumes.js`), each just wiring the existing unchanged API route
  (`/api/reviews/<type>`) and per-type field mapping (secondary column, thoughts field,
  max rating) into `V1ReviewTable`.
- Added a `Perfumes` row to `V1_SECTIONS` in `V1Layout.js` (it existed as a real API/page
  but was missing from the variant's sidebar nav) so `/v1/reviews/perfumes` is reachable
  from the shell, consistent with the other four types already listed there.

## Verify
