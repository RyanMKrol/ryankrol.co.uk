# T068 — Backfill-from-API button on the books edit page

## What changed
- `src/pages/reviews/books/edit/[id].js`: added a `MetadataBackfillModal` (from T066) below the
  author field, before rating. `handleBackfillSearch` calls
  `GET /api/books/search?title=...&author=...&provider=googlebooks` using the form's current
  title/author. `handleBackfillConfirm` overwrites `source, coverUrl, volumeId, bookAuthors,
  firstPublishedYear, isbn, subjects, pageCount, publisher` on `formData` — `rating` and
  `overview` are never touched. The existing Save button still submits `formData` (now including
  the extra metadata keys) via `POST /api/reviews/books/update`, which already accepts them (T067).
  No auto-submit on confirm.

- Manual UI click-through not run headless (would require a live Google Books call / real book
  record); reviewed code paths against `mapGoogleBooksResult` shape and `BookSearch.js`'s calling
  convention to confirm field names line up.
