# T134: self-healing storeWorkoutInDynamoDB

## What changed

`src/lib/workoutBackfill.js` — `storeWorkoutInDynamoDB(workout)`:

- On `ConditionalCheckFailedException` from the initial workout `PutCommand`, now does a
  `GetCommand` for the existing item before giving up.
  - If the existing item already has `exercises`, behavior is unchanged: log + `return false`.
  - If the existing item is missing `exercises` (external process's lean-stub shape), it's healed
    in place via `UpdateCommand` (same field list as `backfillIncompleteWorkoutMetrics.js`'s
    `buildMetricsUpdate`/`writeMetricsUpdate`: `exercises`, `totalVolume`, `totalWarmupSets`,
    `totalWorkingSets`, `uniqueExercises`, `durationMinutes`, `workoutType` — computed from the
    already-derived `metrics` object at the top of the function), then falls through into the
    existing per-exercise store loop, and returns `true`.
- Extracted the per-exercise store loop into a `storeExercises(workout)` helper so it's shared
  between the happy path and the healing path, instead of duplicating it. Added a per-exercise
  try/catch that tolerates `ConditionalCheckFailedException` for an individual exercise (skip +
  log) rather than letting it bubble up and be misread as "the whole workout already exists" —
  this matches the per-exercise-already-exists tolerance the task spec describes.

Because the healed path returns `true`, `backfillWorkouts()`'s `newWorkouts` counter increments
and `consecutiveExisting` resets, same as a brand-new workout — so the trailing
`if (newWorkouts > 0) clearApiCache(...)` still fires after a run that only healed stubs.

## Tests

New `src/lib/workoutBackfill.test.js`, mocking `docClient.send` (pattern from
`src/pages/api/vinyl/add.test.js`) and `global.fetch` (one page, one workout), driving the real
`backfillWorkouts()` → `storeWorkoutInDynamoDB()` call path:

1. Brand-new workout id → one `PutCommand` for the workout + one for the exercise, no
   `UpdateCommand`, `newWorkouts === 1`.
2. Existing workout id with `exercises` already present → `GetCommand` returns a complete item,
   no `UpdateCommand`, `newWorkouts === 0`.
3. Existing workout id missing `exercises` → `GetCommand` returns the lean-stub shape, one
   `UpdateCommand` with the computed metrics + `exercises` in `ExpressionAttributeValues`, the
   exercise-store loop still runs (one exercise `PutCommand`), `newWorkouts === 1`.
