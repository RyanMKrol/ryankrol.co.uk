# T296 — Promote Header into _app.js as a persistent layout

## What changed
- `src/pages/_app.js`: imports `Header` and renders it once, inside `MatrixLayout`, above
  `<Component {...pageProps} />` and `<Footer />` — a single Header instance now survives
  client-side navigation instead of unmounting/remounting per page.
- Removed every per-page `<Header />` render (and now-dead `import Header from ...`) from all
  33 in-scope page files, including every conditional branch (loading/error/empty/success) in
  multi-return pages: `workouts/index.js` (3), `workouts/[id].js` (4),
  `exercises/[exerciseName].js` (4), the five `reviews/*/edit.js` (3 each), the five
  `reviews/*/edit/[id].js` (2 each), and the four `reviews/*/backfill.js` (3 each). All other
  in-scope files had 1 render site.
- No changes to `src/components/Header.js` or `src/lib/scrollFade.js`.
