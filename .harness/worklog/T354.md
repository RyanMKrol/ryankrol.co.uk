2026-07-10

## What I did
- `src/pages/listening/index.js`: added a `PillGroup` (imported from `../../components/PillGroup`)
  above the page header with three options whose `value` is Last.fm's own period string and
  doubles as the pill label — `1month`/`3month`/`6month`. Added `useState('1month')` (new default,
  replacing the old hardcoded `period=3month`). The `fetchTopAlbums` effect now depends on
  `[period]`, passes it straight through as the `period` query param to
  `/api/lastfm/top-albums`, sets `loading` back to `true` at the start of each fetch (so the
  loading state re-shows on pill change, not just on first mount), and refetches on every pill
  click. The subtitle (`collection-review-meta`) now reads from a `PERIOD_SUBTITLES` map
  (`'last 1 month'` / `'last 3 months'` / `'last 6 months'`) instead of a fixed `'last 90 days'`.
- Did not touch `src/pages/api/lastfm/top-albums.js` — it already accepted an arbitrary `period`
  query param with no changes needed (confirmed by reading it: `const { period = '3month', limit
  = '50' } = req.query`).
- `scripts/_visual-harness.mjs`: added a `FLOWS` entry `listening-period-3month` (clicks the
  "3 months" pill on `/listening`) alongside the existing `listening` baseline `PAGES` entry.
