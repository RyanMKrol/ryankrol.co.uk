## What was actually done

1. Confirmed `HARDCOVER_API_TOKEN` was present in Vercel's production env vars (added in an
   earlier session).
2. Confirmed T361's Hardcover-backed `/api/books/search.js` was already deployed to production
   (via the harness's auto-deploy-on-drain mechanism, ~11h prior).
3. **First live test** — `curl https://ryankrol.co.uk/api/books/search?title=Dune&author=Herbert`
   returned a `500` with `{"error":"Hardcover GraphQL error"}`. Pulled the real error via
   `vercel logs --json`:
   `"unexpected subselection set for non-object field"` on path
   `$.selectionSet.search.selectionSet.results`.
4. This is NOT the risk T364 exists to test (Hardcover blocking Vercel-origin requests) — it's a
   GraphQL query-shape bug. Confirmed by introspecting Hardcover's schema directly
   (`__type(name: "SearchOutput")`): `results` is typed `jsonb` (a scalar), not a nested object —
   so the deployed query's `results { hits { document {...} } } }` subselection is invalid
   GraphQL. The request DID reach Hardcover and got a normal (if unhelpful) response — proving
   Hardcover is reachable from Vercel's production infrastructure, not blocked by origin.
5. Ran a real search against Hardcover's API directly to see the actual `results` shape:
   confirmed it returns `{ found, hits: [{ document: {...} }], ... }` — exactly what
   `src/pages/api/books/search.js`'s existing JS parsing (`data.data.search.results.hits`)
   already expected. Only the GraphQL query text was wrong.
6. **Fixed** `src/pages/api/books/search.js`: replaced the invalid subselection with `results`
   as a leaf field (commit `dc9e7a6`). Verified locally against the real Hardcover API — 20
   correctly-mapped results for "Dune Herbert". `npm run lint` (0 errors), `npm test` (441/441),
   `npm run build` all clean. Pushed to `main`.
7. Deployed the fix to production: `vercel --prod` → `dpl_4hysur8zcn6wV9UU6iwkX9KoiDot`, READY.
8. **Final live test** — `curl https://ryankrol.co.uk/api/books/search?title=Dune&author=Herbert`
   → 20 real results, correctly shaped (`source: "hardcover"`, real `volumeId`/`title`/`authors`/
   `firstPublishedYear`/`coverUrl`/`isbn`/`pageCount`).
