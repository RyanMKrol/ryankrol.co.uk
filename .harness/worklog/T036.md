# T036 worklog

## What was done

Added worklog and audit display to the harness backlog dashboard.

### Changes to `.harness/dashboard/server.js`

1. Added `readWorklog(id)` helper — reads `.harness/worklog/<id>.md`, returns `''` on missing file.
2. Added `readAudit(id)` helper — reads `.harness/worklog/<id>.audit.md`, returns `''` on missing file.
3. Updated `buildBacklog()` to attach `worklogContent` and `auditContent` to each task object alongside the existing `specContent`.
4. Updated the client-side `task(t)` template to render:
   - A "Worklog" label + `<pre>` block when `t.worklogContent` is non-empty.
   - An "Audit" label + `<pre>` block when `t.auditContent` is non-empty.
   - Both are escaped via the existing `esc()` function.
   - Tasks with no worklog show no worklog/audit block.

### Verification

- `npm run lint`: 0 errors (1 pre-existing warning in cache.js, unchanged).
- `npm test`: 115 tests passed, 12 suites (including `.harness/dashboard/lib.test.js`).
- `npm run build`: clean build.
- Manual verification deferred to post-commit (server runs locally only), but logic is directly readable: `readWorklog`/`readAudit` follow exact same try/catch pattern as `readSpec`; template additions are conditional on non-empty strings.
