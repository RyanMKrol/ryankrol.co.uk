# T113: books id-migration script

Mirrored `migrateTvToIdTable.js` / `migrateMoviesToIdTable.js` for books.

- `src/scripts/migrateBooksToIdTable.js`: scans `BookRatingsV3` (composite key
  `title`+`author`), writes into new `BookRatingsV4` (sole HASH key `id`), copying
  every field unchanged (including `title`/`author`, now plain attributes) plus a
  fresh `randomUUID()` id. Guards against double-migration (aborts if target
  non-empty unless `--force`). Supports `--verify` (re-scan + `diffMigratedItem`
  comparison matched on `title`+`author`) and runs verify automatically after a
  migration. `BookRatingsV3` is read-only throughout.
- `src/scripts/migrateBooksToIdTable.test.js`: covers `buildMigratedItem` (field
  preservation incl. title/author retained as attributes, id presence, UUID v4
  shape, uniqueness), `diffMigratedItem` (clean match / changed field / missing
  item), `findMatchingNewItem` (match/no-match on title+author).
