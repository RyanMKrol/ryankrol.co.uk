# T038 — Perfume longevity/projection/seasons fields — backend

## What was done

- Added `validateLongevity`, `validateProjection`, `validateSeasons` to `src/lib/perfumes.js`,
  mirroring the style of the existing `validatePerfumeRating`. Exported alongside existing exports.
- Updated `src/pages/api/reviews/perfumes/add.js` and `update.js` to destructure `longevity`,
  `projection`, `seasons` from the request body, validate them (only when present — all three
  remain optional and are not part of the "Missing required fields" check), and persist them on
  the `PerfumeRatings` item via conditional spreads (`...(longevity !== undefined && { longevity }`)
  so omitted fields are never written as `undefined` (the DynamoDB doc client throws on `undefined`
  attribute values since `removeUndefinedValues` isn't configured in `lib/dynamo.js`).
- Added test cases to the existing `src/lib/perfumes.test.js` covering the boundary values called
  out in the spec (0/8, -1/9 for longevity; 1/4, 0/5 for projection; `[]`/`["Winter","Night"]` pass,
  `["Winter","NotASeason"]`/non-array fail for seasons).
