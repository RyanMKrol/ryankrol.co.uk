# T377: MarkdownEditor toggle formatting

## Summary
Implemented toggle formatting for Bold, Italic, and List buttons in the shared MarkdownEditor component. Second clicks on already-formatted text now remove the formatting instead of adding another layer.

## Implementation Details

### Core Logic
- Added `toggleWrapSelection()` function to handle Bold and Italic toggle detection:
  - Detects markers both outside selection (e.g., `**[test]**`) and inside (e.g., `[**test**]`)
  - Carefully distinguishes between single `*` (italic) and `**` (bold) to prevent misfires
  - When Bold is applied to `*...*`, it wraps to get `***...***, not mistaking for italic toggle
  - Same logic for Italic on `**...**`
  - Falls back to wrapping if no markers detected (first-click behavior unchanged)

- Added `toggleLinePrefix()` function to handle List toggle:
  - Checks if line already starts with `- ` prefix
  - If present, removes it; otherwise adds it via existing insertLinePrefix logic
  - Preserves cursor positioning and maintains consistent behavior

### Tests Added
Extended `src/components/MarkdownEditor.test.js` with comprehensive toggle test coverage:
1. "toggles Bold off by clicking Bold twice" - verifies bold wrapping and unwrapping
2. "toggles Italic off by clicking Italic twice" - verifies italic wrapping and unwrapping  
3. "toggles List off by clicking List twice" - verifies list prefix add/remove toggle
4. "does not mistake Italic for Bold" - ensures `**text**` + Italic click wraps to `***text***`
5. "does not mistake Bold for Italic" - ensures `*text*` + Bold click wraps to `***text***`
6. "still wraps on first click" - confirms first-click wrapping still works unchanged
7. "Link button remains INSERT-ONLY" - confirms Link doesn't toggle, always wraps

All existing tests continue to pass, confirming backward compatibility.
