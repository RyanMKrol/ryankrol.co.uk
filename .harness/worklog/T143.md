Read `design_handoff/README.md` and skimmed renders `2a.png` (movies list — sort pills,
search box) and `3f.png` (workouts list — filter pills) for reference. Added four shared
presentational components under `src/components/`, all styled via global CSS classes
(matching the plain-CSS convention already used by `SortButtons.js`, not CSS Modules):

- `Pill.js` — single pill; `active` fills ink+white by default, or `accentColor` fills
  with a specific hex when active (e.g. section accent).
- `PillGroup.js` — controlled single-select wrapper rendering a `Pill` per `{value,label}`
  option; not wired into any page yet, per spec.
- `SearchInput.js` — bordered pill-shaped box with a leading `⌕` glyph, controlled
  `value`/`onChange`.
- `Badge.js` — small filled pill; `variant="solid"` (opaque fill, white text) or
  `variant="soft"` (tinted background using the fill color + alpha, colored text).

Added `.collection-pill`, `.collection-pill-group`, `.collection-search-input`,
`.collection-search-icon`, `.collection-badge` rules to `globals.css`, reading colors/
radius/border-width from the Collection tokens landed in T142 (`--color-ink`,
`--radius-pill`, `--border-width`, `--color-card`, `--color-ink-mute`, `--font-body`,
`--font-mono`). No hardcoded hex/px. Used a `collection-` class prefix to avoid colliding
with the pre-existing legacy `.search-input` / `.filter-button` classes (old theme system,
different CSS vars) still present elsewhere in the file.
