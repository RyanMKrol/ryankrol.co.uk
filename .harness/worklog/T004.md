# T004 — Perfumes review API routes

## What was built

- `src/lib/perfumes.js` — `validatePerfumeRating(value)` (integer 0-10 only) and
  `perfumeId({ title, designer, type })` (slugified `title__designer__type` key, stable
  and distinct per type so e.g. Ombré Leather EDP vs Parfum coexist).
- `src/lib/perfumes.test.js` — covers valid ratings (0/5/10), invalid (6.5, -1, 11, "x",
  undefined, null), perfumeId stability, and perfumeId divergence by `type`.
- `src/pages/api/reviews/perfumes/index.js` — GET, `withApiCache(generateCacheKey('perfumes'), ...)`
  reading `DYNAMO_TABLES.PERFUME_RATINGS_TABLE`.
- `src/pages/api/reviews/perfumes/add.js` — POST, password-gated, validates rating via
  `validatePerfumeRating`, builds record with `id` from `perfumeId`, `PutCommand`, then
  `clearApiCache('api-perfumes')`.
- `src/pages/api/reviews/perfumes/update.js` — POST, password-gated, recomputes `id` from
  the new title/designer/type; if it differs from `originalId` (passed in body), deletes the
  old item first, then puts the new one; `clearApiCache('api-perfumes')`.
- `src/pages/api/reviews/perfumes/delete.js` — DELETE, password-gated, deletes by `{ id }`;
  `clearApiCache('api-perfumes')`.
