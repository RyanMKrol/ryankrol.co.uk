# T116: Switch albums to id-keyed AlbumRatingsV3 table

## What was built

- `src/lib/constants.js` — `ALBUM_RATINGS_TABLE`: `'AlbumRatingsV2'` → `'AlbumRatingsV3'`.
- `src/pages/api/reviews/albums/add.js` — generates `id: randomUUID()` on create (alongside
  existing `title`/`artist`/`rating`/`highlights`/`date`/`thumbnail`/`lastfm` fields).
- `src/pages/api/reviews/albums/update.js` — accepts `originalId` (replacing
  `originalTitle`+`originalArtist`); still uses `UpdateCommand`, now keyed `Key: { id: originalId }`.
  Title/artist remain non-editable — no new fields added to the update expression.
- `src/pages/api/reviews/albums/delete.js` — accepts `{ id, password }`, `DeleteCommand` keyed
  `{ id }`.
- `src/pages/api/reviews/albums/index.js` — unchanged; table swap is automatic via constants.js.
- `src/pages/reviews/albums/edit.js` — list key + edit link now use `album.id` instead of the
  `title + '|' + artist` composite string.
- `src/pages/reviews/albums/edit/[id].js` — resolves the album via `a.id === decodeURIComponent(id)`
  instead of splitting on `'|'`; submit payload now sends `originalId: originalData.id`; delete
  payload now sends `id: originalData.id`. Title/artist inputs remain `disabled`, unchanged.
- `src/pages/api/reviews/albums/add.test.js` — asserts the created `Item.id` matches a v4 UUID
  shape.
- Docs: `CLAUDE.md`'s Data model row and `README.md`'s `#### AlbumRatingsV3 Table` section
  updated to describe the single `id` partition key (mirroring the `BookRatingsV4` precedent),
  noting the old `AlbumRatingsV2` is left at rest, unused, for rollback.
