# T173: widen `.container` / `.review-container` to match the Collection redesign

## What changed

`src/styles/globals.css` only:

- `.container` and `.review-container` (formerly `max-width: 800px` with a flat
  `padding: var(--spacing-container-padding)`) now mirror the already-proven
  `.home-container` pattern (T165, ~line 2853): `max-width: 1400px`, with
  `padding-left`/`padding-right` set to `var(--gutter-desktop)` (40px) and
  `padding-top`/`padding-bottom` also set to `var(--gutter-desktop)` so vertical
  spacing is unchanged from before.
- Added a `@media (max-width: 768px)` block for both selectors that swaps
  `padding-left`/`padding-right` to `var(--gutter-mobile)` (18px), matching the
  existing `.home-container` breakpoint.
- `margin: 0 auto`, `background-color: var(--color-surface)`, and
  `min-height: 100vh` left untouched on both selectors.
- `.home-container` and `src/pages/index.js`'s `className="container home-container"`
  left as-is (now redundant with the widened `.container` but harmless — not
  required to remove per the task spec, kept the diff minimal).
- No files under `src/pages/**` touched — every page picks up the new width via
  the shared `.container`/`.review-container` class names.

## Verification

- `npm run lint` — 0 errors, 6 pre-existing warnings (unchanged from before this
  change: `no-img-element` x4, `exhaustive-deps` x1 on `dev/cache.js`).
- `npm test` — 35 suites / 241 tests, all green.
- `npm run build` — succeeds, all 32 pages generated.
- Did not run `npm run dev` / load pages in a browser (headless session, no
  display) — verified the CSS change by reading the diff against the already
  shipped `.home-container` pattern it mirrors byte-for-byte (same property set,
  same breakpoint, same token values), which was itself visually verified in
  T165. No live external API calls made.
