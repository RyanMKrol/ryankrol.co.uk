# T178 — nav pills horizontal scroll instead of wrap

## What changed

`src/styles/globals.css`: `.collection-nav-pills` (~line 939) changed from
`flex-wrap: wrap; justify-content: center;` to a horizontally-scrollable strip:

- `flex-wrap: nowrap`
- `overflow-x: auto; overflow-y: hidden`
- `justify-content: flex-start` (was `center` — centering fights scrolling)
- `scrollbar-width: none` + `.collection-nav-pills::-webkit-scrollbar { display: none; }`
  to hide the native scrollbar
- `mask-image` / `-webkit-mask-image` linear-gradient fade on the trailing (right)
  edge as the "more to scroll" affordance
- `.collection-nav-pills > a { flex-shrink: 0; }` so pills don't compress

Did not touch `src/components/Pill.js` / `PillGroup.js` — confirmed Header.js only
imports `Pill` (`import Pill from './Pill'`), and each `NAV_SECTIONS` entry is
rendered as `<Link href={...}><Pill>...</Pill></Link>`, i.e. `Link` (→ `<a>`) is a
direct child of `nav.collection-nav-pills`, so the `> a` selector is correct.
Did not touch the `@media (max-width: 768px)` block (~line 2306) that hides
`.collection-nav-pills` and shows the `☰` mobile menu — left exactly as-is.
