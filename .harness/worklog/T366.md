# T366 ‚Äî Hardcover enrichment: fetch book details on confirmation

## Summary
Successfully implemented enrichment for Hardcover book entries by fetching detailed book metadata (synopsis, slug, rating, series) from Hardcover's GraphQL API when a book is confirmed in the add/edit flows.

## What was done

### 1. Implementation (src/lib/hardcover.js)
- Added `mapHardcoverDetails(book)` ‚Äî normalizes raw Hardcover GraphQL response into 5 optional fields:
  - `hardcoverSynopsis` ‚Äî book description (only when non-empty)
  - `hardcoverSlug` ‚Äî canonical Hardcover URL slug
  - `hardcoverRating` ‚Äî community rating (number only)
  - `seriesName` ‚Äî first series name (if book is part of a series)
  - `seriesPosition` ‚Äî position in series (number only)
- Added `fetchHardcoverBookDetails(bookId)` ‚Äî GraphQL query handler:
  - Queries Hardcover's `/v1/graphql` endpoint with `BookDetails` query
  - Uses `HARDCOVER_API_TOKEN` from env (server-side only)
  - Returns empty object on any error (API down, missing token, network failure, GraphQL errors)
  - Fields are only included in returned object if data exists upstream (no null placeholders)

### 2. Integration (API routes)
- `src/pages/api/reviews/books/add.js` ‚Äî calls `fetchHardcoverBookDetails(volumeId)` when `source='hardcover'` and spreads enriched data into the written item
- `src/pages/api/reviews/books/update.js` ‚Äî same pattern for updates

### 3. Tests (src/lib/hardcover.test.js)
- 23 new test cases covering `mapHardcoverDetails`:
  - Well-formed response with all 5 fields
  - Omit each field individually when missing/null/invalid/empty
  - Handle edge cases (empty book object, null series, whitespace trimming)
  - Degraded response (only slug present)
- 11 test cases for `fetchHardcoverBookDetails`:
  - Input validation (null/undefined/non-string bookId)
  - Missing env var gracefully returns {}
  - Successful fetch with mocked Hardcover API response
  - Error handling (HTTP errors, GraphQL errors, empty results, network failures)
  - Correct GraphQL query structure and variables

### 4. Documentation
- **README.md**: Updated BookRatingsV4 schema section to document the 5 new optional fields and their purpose
- **CLAUDE.md**: Updated Data model table to reflect `source: 'hardcover'` support and list the 5 enrichment fields

## Field behavior

Per the task spec, fields omit completely (not null or empty string) when upstream data is absent:
- Description with only whitespace ‚Üí omitted
- Slug missing or empty ‚Üí omitted
- Rating not a number or null ‚Üí omitted
- Series not an array or empty array ‚Üí seriesName/seriesPosition both omitted
- Series[0].name null or whitespace ‚Üí omitted
- Series[0].position not a number ‚Üí omitted

This allows records to vary in richness based on what Hardcover has, without silent null values.

## Notes

- The enrichment fetch happens **on confirmation only**, not on every search result (as specified)
- Fetch is gated to `source='hardcover'` to avoid unnecessary API calls
- Empty response defaults to `{}` on any error for safe degradation
