# T094 — Per-spot spray count for Application spots — backend

## What changed
- `src/lib/perfumes.js`: `validateApplicationSpots(value)` now validates an array of
  `{spot, sprays}` objects instead of a flat array of spot strings. `spot` must be one of
  the six valid strings; `sprays` must be a positive integer (0/negative rejected). The old
  flat-string shape now fails validation, as required.
- `src/lib/perfumes.test.js`: replaced the old flat-string-array assertions with tests for
  the new `{spot, sprays}` shape — empty array, valid objects, unrecognised spot, non-array,
  old flat-string shape, and zero/negative sprays.
- `src/pages/api/reviews/perfumes/add.js` / `update.js`: no code changes needed. Both routes
  already just call `validateApplicationSpots(applicationSpots)` and spread the field
  verbatim into the DynamoDB item — they're shape-agnostic, so the validator change alone
  makes them accept/reject the new shape correctly. Verified by reading both files; no
  route logic references the internal structure of `applicationSpots`.
