# T090 — /projects: fix wrong GitHub API endpoint, include archived repos, warn on missing token

## Changes (src/pages/api/github/repos.js only)

1. When `GITHUB_TOKEN` is set, call `GET /user/repos?affiliation=owner` (authenticated-user
   endpoint, returns private + public repos owned by the token's account) instead of the
   public-only `GET /users/{username}/repos`. Without a token, fall back to the previous
   `/users/{username}/repos` call (unauthenticated has no `/user/repos` equivalent).
2. Removed the `!repo.archived` filter condition — archived repos with a `pushed_at` date are
   now included. Forks are still excluded.
3. Added `console.warn` when `GITHUB_TOKEN` is not set, so the public-only fallback (and the
   resulting missing private repos) is visible in Vercel function logs.
