# T202 — Wire every list-rendering page to use list-aware gradient assignment

## What changed

Rewired all seven call sites to compute gradients once per rendered list via
`assignGradients` (from T201) instead of each item independently calling
`gradientForKey`:

1. `src/pages/index.js`
   - `wallItems` `useMemo`: after sampling, filters items lacking `imageUrl`,
     calls `assignGradients` over their keys in render order, zips the result
     back onto the sampled array as `item.gradient`, passed to `<CoverTile>`.
   - `latestTakes` (now wrapped in its own `useMemo`): filters items lacking a
     real thumbnail/poster, assigns gradients in `latestTakes` render order,
     zips back, used directly as the `home-latest-thumb` inline background
     instead of calling `gradientForKey`.
2. `src/pages/listening/index.js`: filters albums lacking `album.image`,
   assigns gradients across the rendered `albums` order, zips into a parallel
   `albumGradients` array indexed alongside the `.map()`, used for the
   `listening-cover-thumb` inline background.
3. `src/pages/vinyl/index.js`: concatenates `sortedLetters` into one ordered
   list (`orderedVinyl`) before splitting into per-letter groups, so the
   no-repeat window spans letter-group boundaries; filters out items with a
   real `thumbnail`, assigns once across the full ordered list, and looks the
   result up per-record (by object identity, via a `Map`) when rendering each
   letter group's `<CoverTile>`.
4. `src/pages/reviews/movies/index.js` / `tv/index.js`: `poster-banner`
   variant always renders a gradient, so `assignGradients` runs across the
   full current page (`pagedMovies`/`pagedTvShows`) with no filtering, zipped
   back by index into the `gradient` prop on `<ReviewCard>`.
5. `src/pages/reviews/books/index.js`: same unconditional-gradient pattern for
   `pagedBooks` (`spine-cover` variant).
6. `src/pages/reviews/albums/index.js`: `square-cover` variant supports real
   art, so filters the current page to items lacking `thumbnail`/`coverUrl`
   before assigning, then zips back (`null` for items with real art) into the
   `gradient` prop.

`src/pages/reviews/perfumes/index.js` was left untouched, per the spec —
perfumes render `Variant6Hybrid` directly and never touch the gradient
mechanism.
