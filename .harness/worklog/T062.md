# T062 — Variant 3 — workouts + exercises pages

## What was built

- `src/components/v3/V3WorkoutFeed.js` — `/v3/workouts` list, styled per Variant 3's linear
  feed brief: one `V3TimelineEntry` per workout in the date-gutter timeline, an inline
  text-link filter control (`filtered by all, push, pull, legs`) instead of a button group,
  and each entry expands in place to show working-set count with a link through to the full
  detail page.
- `src/components/v3/V3WorkoutDetail.js` — `/v3/workouts/[id]`. Kept as a dedicated route (per
  the task spec, which explicitly calls for `/v3/workouts/[id]` rather than an inline
  disclosure) but rendered in the same gutter/rule idiom as the rest of variant 3 — no cards,
  no grid, just stacked gutter rows for the summary and each exercise/set.
- `src/components/v3/V3ExerciseDetail.js` — `/v3/exercises/[exerciseName]`. Reuses the existing
  `ExerciseProgressCharts` / `CardioProgressCharts` components and `dateRange` lib unchanged,
  with the date-range switcher rendered as the same inline text-link control used for the
  workout filter, and session history as gutter rows.
- Thin page wrappers: `src/pages/v3/workouts/index.js`, `src/pages/v3/workouts/[id].js`,
  `src/pages/v3/exercises/[exerciseName].js`.

All three fetch from the existing, unchanged API routes (`/api/workouts`, `/api/workouts/[id]`,
`/api/workouts/[id]/exercises`, `/api/exercises/history/[exerciseName]`) and reuse T045's
`V3Layout` shared shell — no second layout wrapper was built.
