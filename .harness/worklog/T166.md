# T166 — Visual audit: reviews list pages vs design renders

No browser-screenshot tool was available in this environment, so the audit was done by
reading the paired renders (`renders/2a.png`, `3a.png`, `3b.png`, `3c.png`, `3d.png`) and
the token spec (`design_handoff/README.md`) side-by-side against the actual CSS rules and
component markup for the 5 list pages, `ReviewCard.js`, `Pill.js`, and `Pagination.js`.

## Gaps found and fixed

1. **`.page-title` used an undefined CSS custom property (`--font-heading`), so the giant
   "movies"/"books"/"tv"/"albums"/"perfumes" headline was silently falling back to the
   inherited body font (Nunito) instead of the display font (Bricolage Grotesque), and was
   far smaller/lighter (2.5rem / bold) than the renders' oversized 88px-class display
   headline.** Fixed in `globals.css`: `font-family: var(--font-display)`, `font-weight: 800`,
   `font-size: 3.5rem` (2.5rem on mobile), `letter-spacing: -0.03em`, `color: var(--color-ink)`.
   Removed the stale `margin-bottom: 2rem` since spacing is already handled by the
   `.collection-review-title-group` flex gap — this also fixes a double-gap under the title.
   (`--font-heading` is still referenced by 3 other selectors outside this task's scope —
   `.review-title`, `.home-section-title`, `.vinyl-section-header` — left alone since those
   files aren't in T166's scope; worth a follow-up task.)

2. **Sort pills and pagination pills were rendering filled with each section's accent color
   when active, instead of ink-black.** Every paired render (movies/tv/books/albums) shows
   the active "date ↓" sort pill and the active page-number pill filled solid ink/black,
   regardless of section — accent color is reserved for the top-nav pill only (per
   `README.md`: "Sort pills / filter pills: single-select, active = ink fill"). The 5 review
   pages were passing `accentColor="var(--accent-<type>)"` into `PillGroup`/`Pagination`,
   which `Pill.js` uses to override the active fill with that color. Removed the
   `accentColor` prop from all `PillGroup`/`Pagination` call sites in
   `movies/tv/books/albums/perfumes/index.js` — `Pill.js` itself was left untouched since
   `Header.js` (out of scope) legitimately uses `accentColor` for the nav's active section
   pill.

## Checked and already correct (no change needed)
- `ReviewCard.js` variants (`poster-banner`, `spine-cover`, `square-cover`, `perfume-card`)
  each have their own CSS block in `globals.css` with no shared/leaked class names —
  spacing, typography, and per-variant colors already matched their renders.
- Perfume pip meter: 10 dots, grape filled (`--pip-filled: #7A5AF8`) / lilac empty
  (`--pip-empty: #E1D9F5`), matches `3d.png` exactly.
- Perfume scale-chip copy and grape fill, and the dashed empty-state "+ your next review
  lands here" card, matched the render as built.
- `.collection-pill.active` base rule (ink fill, white text) was already correct — the bug
  was purely the per-page `accentColor` override described above.
- Poster-banner grid (3-col desktop / 1-col mobile <900px), square-cover grid (2-col /
  1-col <700px), perfume-card grid (2-col / 1-col <700px) breakpoints already match the
  paired mobile renders.
