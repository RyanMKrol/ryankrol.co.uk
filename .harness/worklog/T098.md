# T098 — Required Fragrantica URL field on perfume forms

## What changed
- `src/lib/perfumes.js`: added `validateFragranticaUrl(value)` — requires a non-empty string
  starting with `http://`/`https://` that parses via the `URL` constructor.
- `src/pages/api/reviews/perfumes/add.js` and `update.js`: `fragranticaUrl` is now required
  alongside title/designer/type/rating (400 if missing), and validated via
  `validateFragranticaUrl` (400 if malformed). Persisted on the `PerfumeRatings` item.
- `src/pages/reviews/perfumes/add.js` and `edit/[id].js`: added a required
  `<input type="url" name="fragranticaUrl">` field, wired into `formData` state and the
  submit payload. Edit form pre-populates from `perfume.fragranticaUrl || ''`.
- `src/lib/perfumes.test.js`: added a `validateFragranticaUrl` describe block covering valid
  URLs and missing/malformed/non-http(s) values.

## Verification observed
- `npm run lint` — 0 errors (pre-existing warnings only: dev/cache exhaustive-deps,
  movies/tv edit `<img>`).
- `npm test` — 22 suites / 169 tests passed, including the new validator tests.
- `npm run build` — `next build` succeeded; `/reviews/perfumes/add` and
  `/reviews/perfumes/edit/[id]` compiled as static pages, `/api/reviews/perfumes/{add,update}`
  compiled as dynamic routes.
- Did not run a live dev-server manual check (no DynamoDB write access in this headless run) —
  code paths were verified by reading the diff: add/update reject on missing/malformed
  `fragranticaUrl` before any DynamoDB call, and the edit form reads
  `perfume.fragranticaUrl || ''` the same way it does for other optional-until-now fields
  (e.g. `description`).
