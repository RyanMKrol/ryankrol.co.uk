# T020 worklog — TMDB search + confirm step in the movie/TV add flow

## What was built

Created `src/components/TmdbSearch.js` — a self-contained reusable component that:
- Accepts `mediaType` ('movie'|'tv'), `initialQuery`, and `onSelect` props
- Renders a search input + "Search TMDB" button (fires on Enter or click)
- Calls `GET /api/tmdb/search?query=...&type=...` and renders all results as cards
  (poster thumbnail via `tmdbPosterUrl`, title, year, 160-char overview truncation)
- "Select" button on each result calls `onSelect({ tmdbId, mediaType, posterPath, overview, date })`
- Shows a confirmation banner when a match is selected; "Clear" resets to the results list
- Handles zero-results and error states gracefully
- Styles are scoped with `<style jsx>` so no global CSS impact

Updated `src/pages/reviews/movies/add.js` and `src/pages/reviews/tv/add.js`:
- Added `tmdbMatch` state (null until a TMDB result is selected)
- Inserted `<TmdbSearch mediaType="movie|tv" initialQuery={formData.title} onSelect={setTmdbMatch} />`
  between the Title and Rating fields as an optional "TMDB Match" form group
- On submit, spreads `tmdbId, mediaType, posterPath, tmdbOverview, tmdbDate` into the POST body
  only when `tmdbMatch` is non-null — saving without a match still works (T019 add API treats
  all TMDB fields as optional)
