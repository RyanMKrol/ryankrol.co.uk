# T326 — fix /reviews/albums/edit masonry layout

2026-07-07

## What I did
`src/pages/reviews/albums/edit.js` applied the `square-cover-grid` masonry CSS class directly to
a plain `.map()` div, so every album card rendered into one endless horizontally-scrolling row
instead of the 2-column masonry grid `MasonryColumns` produces (the pattern already used
correctly by `src/pages/reviews/books/edit.js`).

Fixed by:
- Importing `MasonryColumns` and `useResponsiveColumnCount`.
- Adding `const columnCount = useResponsiveColumnCount(2, 700);`.
- Replacing the bare `<div className="square-cover-grid">{albums.map(...)}</div>` with
  `<MasonryColumns items={albums} columnCount={columnCount} className="square-cover-grid"
  columnClassName="square-cover-grid-col" renderItem={(album) => (...)} />`, moving the existing
  per-card JSX (Edit link + `ReviewCard`) into `renderItem` unchanged.

No other files touched — change is scoped entirely to `src/pages/reviews/albums/edit.js`.
