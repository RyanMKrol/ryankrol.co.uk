# T100 — shared in-memory per-instance rate-limit helper

## What was built

- `src/lib/rateLimit.js` — new module exporting:
  - `checkRateLimit(key, { windowMs = 60_000, max = 20 } = {})` — fixed-window counter in a
    module-level `Map`. Returns `{ allowed, retryAfterSeconds }`. Blocked calls do not increment
    the window's count. Logs `🚦 [RateLimit] Blocked key=... retryAfterSeconds=...` on block.
  - `getClientIp(req)` — reads `x-forwarded-for` (first entry, trimmed), falls back to
    `req.socket?.remoteAddress`, falls back to `'unknown'`.
  - Top-of-file comment documents the per-serverless-instance limitation (mirrors apiCache.js's
    gotcha — no cross-instance coordination without a shared store).
- `src/lib/rateLimit.test.js` — covers: first `max` calls allowed; `max+1`th blocked with
  `retryAfterSeconds > 0`; a different key is unaffected; window reset via
  `jest.useFakeTimers()` + `advanceTimersByTime`.
- `CLAUDE.md` — added a row for `src/lib/rateLimit.js` to the `src/lib/*` file map table.
