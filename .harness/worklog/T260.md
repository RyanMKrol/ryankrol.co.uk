# T260: bulk metadata-backfill page for books

Added `src/pages/reviews/books/backfill.js`, mirroring the movies bulk-backfill page
(T258) and reusing `BulkBackfillList`.

- Fetches `/api/reviews/books`, filters with `needsBookBackfill` (missing both `volumeId`
  and `olid`).
- Single page-level password field, reused by every row's confirm.
- `onSearch`: `GET /api/books/search?title=&author=&provider=googlebooks`, mirroring
  `handleBackfillSearch` in `edit/[id].js`; attaches `retryAfterSeconds` on a 429.
- `renderCandidate`/`getCandidateKey`: reused the cover/title/year/authors JSX from
  `edit/[id].js:247-273`.
- `onConfirm`: POSTs to `/api/reviews/books/update` with the full required field set
  (`title`, `author`, `rating`, `overview`, `originalId`, `password`) plus the candidate's
  metadata fields (`source`, `coverUrl`, `volumeId`, `bookAuthors`, `firstPublishedYear`,
  `isbn`, `subjects`, `pageCount`, `publisher`).
- Empty-state "Nothing to backfill" message.
- Added a "Backfill metadata" link from `src/pages/reviews/books/edit.js` to
  `/reviews/books/backfill`, styled identically to the movies/tv equivalents.

## Verified

- `npm test`: 51 suites / 332 tests passed.
- `npm run build`: succeeded; `/reviews/books/backfill` present in the route output.
