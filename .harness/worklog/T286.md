# T286: Merge /programmes into /workouts

## What changed
- `src/pages/workouts/index.js`: added `dateRange` state (default `'1y'`) alongside the existing
  `activeFilter` state. Added a new section between the header and the session grid: a
  `DateRangeFilter` control, the 3 stat blocks (Total Workouts / Total Volume / Avg Volume per
  Session), and `ProgrammeOverviewCharts`, all driven by
  `aggregateProgramme(filterByDateRange(allWorkouts, dateRange, w => w.start_time), activeFilter)`.
  Both the stats block and the paginated session list read from the same `allWorkouts` state
  fetched once in the existing `useEffect` — no second fetch added.
- Removed the "view programme stats →" link (and its preceding middot) from the header meta line.
- Deleted `src/pages/programmes/index.js` entirely (no redirect, per the "keep it simple"
  convention for a low-traffic personal site). `src/lib/programmeStats.js` and
  `src/components/ProgrammeOverviewCharts.js` are unchanged and now imported from
  `workouts/index.js` instead.
- `CLAUDE.md`: updated the file-map row for `src/pages/workouts/index.js` to describe the merged
  stats/charts block and its own date-range control.
