# T372 Worklog

## Summary
Fixed album backfill eligibility gate from `lastfm.mbid` to `lastfm.url` to prevent enriched albums from being re-selected for backfill.

## Changes Made

- Updated `needsAlbumBackfill` function to check `!item.lastfm.url` instead of `!item.lastfm.mbid`
- Updated doc comment from "(no mbid)" to "(no url)"

## Rationale
The original logic gated on `lastfm.mbid` (MusicBrainz ID), but Last.fm frequently returns empty strings for this field even when the album has been fully enriched with other metadata like `url`, `listeners`, `playcount`, etc. By switching to gate on `lastfm.url` — which is always present when an album has been enriched — we avoid the problem of enriched albums perpetually being marked as "needs backfill". This is a pure filter change with no data migration needed.
