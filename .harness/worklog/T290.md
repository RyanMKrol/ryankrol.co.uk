# T290 — wire PR detection into storeExercises/storeWorkoutInDynamoDB

## What changed

- `src/lib/workoutQueries.js`: added `getBestPriorMetrics(exerciseName, beforeDate)`. Queries the
  same `exercise_name-workout_date-index` GSI as `getExerciseHistory`, filtered to
  `workout_date < beforeDate`, paginated the same way. Reduces to
  `{ heaviestWeight, bestEstimated1RM, bestSetVolume }`, each the max across rows that have that
  field (rows without a field don't contribute to its max, never treated as 0). Returns `null` if
  there are no prior rows at all.
- `src/lib/workoutBackfill.js`:
  - `storeExercises(workout)` now computes `priorBest` via `getBestPriorMetrics`, runs
    `detectPersonalBests(exerciseMetrics, priorBest)` (from T289), and sets `isWeightPR` /
    `is1RMPR` / `isVolumePR` directly on the specific `exercise.sets[index]` object(s) *before*
    `exerciseItem` is built — so the flags land in the persisted `Exercises` row.
  - `storeWorkoutInDynamoDB(workout)` reordered: `storeExercises(workout)` (which mutates
    `workout.exercises[].sets` in place) now runs *before* `workoutItem` is constructed, so the
    same mutated sets are embedded in the `Workouts` item too. Removed the redundant downstream
    `storeExercises` calls in the "already exists" and "heal incomplete" branches — the flagging +
    write now happens exactly once per call, before either table write, which is safe against the
    `ConditionalCheckFailedException` healing logic since the two writes target different tables
    and the mutation only needs to land before either one reads from `workout.exercises`.

## Verification

- `migrateLegacyExerciseRows.js` and `backfillMissingExerciseRows.js` both call the shared
  `storeExercises(workout)` (confirmed by inspection, `grep -n "PutCommand|storeExercises"`) —
  neither hand-rolls an Exercises `PutCommand`, so both automatically inherit PR-flag behavior
  with zero code changes. Left untouched (out of scope).
- Confirmed `getBestPriorMetrics` is not referenced from any file under `src/pages/api/` — it's
  only called from `storeExercises`, i.e. write-time only, never from a page-load/read path.
- Added tests:
  - `workoutQueries.test.js`: `getBestPriorMetrics` — null-when-no-prior-rows, max-reduction
    across rows with a `bestSetVolume`-missing row, and GSI pagination via
    `ExclusiveStartKey`/`LastEvaluatedKey`.
  - `workoutBackfill.test.js`: new test stores two sequential synthetic workouts for the same
    exercise (mocked docClient acting as an in-memory Exercises table) and asserts only the
    second workout's weight-PR set is flagged `isWeightPR: true` while its 1RM/volume are
    correctly *not* flagged (workout1's single set was a PR on all three axes, having no prior).
    Also fixed the pre-existing "heals an incomplete workout" test's assertion, which previously
    expected the healed `exercises` field to equal an unflagged fresh workout object — now expects
    the PR flags this task adds.

- `npm run build` — succeeds.
