# T273 — Markdown rendering for review text

## What was built

- `src/components/Markdown.js` — thin wrapper around `react-markdown`. Two modes:
  - Block mode (default): full markdown via `remark-gfm` (paragraphs, lists, bold, italic, links, etc.)
  - Inline mode (`<Markdown inline>`): `allowedElements` restricted to `p/strong/em/a/del/code` with
    `unwrapDisallowed` so block syntax (lists, headings) degrades to plain inline text instead of
    rendering `<ul>/<li>` etc.; the `p` renderer is overridden to a Fragment so it never introduces
    a nested `<p>` when embedded inside an existing `<p>`.
- Wired into all 5 call sites:
  1. `ReviewCard.js` poster-banner snippet (movies/tv) — block mode, wrapped in a `<div>` (was `<p>`,
     since block-mode Markdown renders its own `<p>`s and nesting `<p>` in `<p>` is invalid HTML).
  2. `ReviewCard.js` spine-cover snippet (books) — block mode, same `<div>` treatment.
  3. `ReviewCard.js` default/no-styleVariant branch — block mode, same `<div>` treatment, label
     `<strong>` kept as a sibling before the `Markdown` block.
  4. `ReviewCard.js` square-cover highlights (albums) — `splitHighlights()` fragments are each
     rendered through `<Markdown inline>`, joined by ' · ', preserving the existing chip layout.
  5. `perfumeVariants/Variant6Hybrid.js` description — block mode, `<div>` wrapper.
- `src/components/Markdown.test.js` (RTL) — asserts `**bold**` → `<strong>`, `*italic*` → `<em>`,
  `- list item` → `<ul><li>` in block mode but not in inline mode, `[link](url)` → `<a href>`.

## Dependency version note (deviation from the literal "latest major" suggestion)

The spec suggested installing whatever current major of `react-markdown`/`remark-gfm` installs
cleanly against React 19. `npm install react-markdown remark-gfm` pulled `react-markdown@10` /
`remark-gfm@4`, which installs fine (peer deps are satisfied) but is **ESM-only** — the whole
`unified`/`remark`/`mdast` toolchain moved to ESM-only in the v10-era rewrite. This repo's Jest
setup (`jest.config.js`, out of scope for this task) has no `transformIgnorePatterns` override for
node_modules, so any test importing `Markdown.js` failed with `SyntaxError: Unexpected token
'export'` — both the new `Markdown.test.js` and the pre-existing `Variant6Hybrid.test.js`.

Since `jest.config.js` is outside this task's scope and `npm test` passing is a hard Definition-of-
Done requirement, I pinned `react-markdown@^6.0.3` + `remark-gfm@^1.0.0` instead — the last major
line before the ESM-only rewrite (uses `unified@^9`, CommonJS). It exposes the same
`allowedElements`/`unwrapDisallowed`/`components` API used here, installs cleanly against React 19
(peer range is `react: >=16`), and runs under Jest with zero config changes. Flagging this in case a
future task wants to upgrade to v10+ alongside a `jest.config.js` `transformIgnorePatterns` change
(out of scope here).
