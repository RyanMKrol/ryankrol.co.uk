# T013 worklog — Multi-theme infrastructure (attributes, persistence, hooks)

## What was built

### src/lib/appearance.js
Pure logic module with no DOM/browser dependencies at the module level:
- `DEFAULTS` — canonical defaults (`theme: 'cyberpunk'`, `mode: 'system'`, `font: 'share-tech-mono'`, `motion: 'full'`)
- `STORAGE_KEYS` — `localStorage` key names (`appearance-theme`, `appearance-mode`, `appearance-font`, `appearance-motion`)
- `resolveMode(storedMode, osPrefersDark)` — maps `'system'`/invalid → OS pref; `'light'`/`'dark'` → themselves
- `applyDefaults(raw, osPrefersDark, osReducedMotion)` — validates raw storage values; respects OS motion preference when nothing stored
- `readStoredAppearance()` — browser-only, reads localStorage + matchMedia, delegates to `applyDefaults`

### src/lib/appearance.test.js
14 unit tests covering:
- `resolveMode`: dark/light pass-through, system→OS, null/empty/garbage → system
- `applyDefaults`: valid values used, null/empty → defaults, resolvedMode follows OS pref when system, OS reduced-motion fallback, explicit motion=full overrides OS pref, invalid mode → default

### src/hooks/useTheme.js
React hook that:
- SSR-safe (returns DEFAULTS on server)
- On mount: reads stored appearance, sets appearance state
- Wires `matchMedia('prefers-color-scheme: dark')` listener; when OS changes and mode is `'system'`, updates `data-mode` attribute live
- Exposes `setTheme`, `setMode`, `setFont`, `setMotion` — each writes to `localStorage` and updates `data-*` attribute on `<html>`

### src/hooks/useChartTheme.js (updated)
`MutationObserver` now watches `attributeFilter: ['class', 'data-theme', 'data-mode', 'data-font']` so charts recolour on theme switches, not just `matrix-active` class changes.

### src/pages/_document.js (updated)
Inline pre-paint script (runs synchronously before first paint):
- Reads `localStorage` for all four keys
- Resolves mode with OS `prefers-color-scheme` fallback
- Resolves motion with OS `prefers-reduced-motion` fallback
- Sets `data-theme`, `data-mode`, `data-font`, `data-motion` on `<html>` before any React paint
- Also restores `matrix-active` class from `sessionStorage` (moved from a second `<script>` tag into the same script)

### src/pages/_app.js (updated)
Imports and calls `useTheme()` at app root to initialize the OS change listener. Matrix Konami code overlay (`matrix-active` class) is independent of `data-*` attributes — they coexist without conflict.

### src/styles/globals.css (updated)
Added attribute-based scaffolding section:
- `[data-motion="reduced"]` disables CRT flicker animation, sets all animation/transition durations to `0.001ms`, stops typing cursor animation, stops spinner animation
- Placeholder comments for T014 palette overrides (`[data-theme="cyberpunk"]`, `[data-theme="matrix"]`, `[data-mode="light"]`)
