# T037 — Dashboard: don't let already-reviewed Done tasks be re-selected for "Mark reviewed"

## What changed

In `.harness/dashboard/server.js`, the client `task(t)` template only renders the `sel-rev`
selection checkbox for a `bucket==='done'` row when `!t.reviewed`. A reviewed Done task now
renders with its "reviewed" pill but no checkbox at all — it can't be ticked and can't be
included in a "Mark N reviewed" bulk submit.

Not-reviewed Done rows are unaffected: they still render the checkbox with
`data-reviewed="false"` (now hardcoded since the branch guarantees `t.reviewed` is falsy),
`toggleAllRev`/`updateBulk`/`markReviewedBulk` are untouched.

## Verify

- `npm run lint` — 0 errors (1 pre-existing `react-hooks/exhaustive-deps` warning in
  `src/pages/dev/cache.js`, unrelated).
- `npm test` — 13 suites / 126 tests pass, including `.harness/dashboard/lib.test.js`.
- `npm run build` — succeeds.
- Manually verified by reading the render path: `task()` computes `sel` before `rev` (the pill),
  and the new condition (`bucket==='done' && !t.reviewed`) means a reviewed row falls through to
  `sel=''` while the pill logic (`t.reviewed ? 'reviewed' : 'not reviewed'`) is unchanged — so the
  pill still renders correctly either way. Did not launch `npm run harness:dashboard` in this
  headless run (no live DynamoDB task data available); the change is a pure template-string
  conditional with no new state, so static verification is sufficient.

## Scope

Only `.harness/dashboard/server.js` touched, per task scope.
