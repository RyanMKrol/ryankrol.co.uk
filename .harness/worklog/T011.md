# T011 Worklog — Date-range filter on the per-exercise page

## Changes

### `src/lib/workoutQueries.js`
- Removed `limit` parameter from `getExerciseHistory`; now paginates via `LastEvaluatedKey` loop to fetch full history.

### `src/pages/api/exercises/history/[exerciseName].js`
- Removed `limit` query param; cache key no longer includes limit; calls `getExerciseHistory` with no limit arg.

### `src/pages/exercises/[exerciseName].js`
- Added `DateRangeFilter` component (from T010) with default `'1y'`.
- Moved stats computation out of `useEffect` into `useMemo` keyed on `filteredHistory`.
- Added `filteredHistory` useMemo: `filterByDateRange(exerciseHistory, dateRange, h => h.workout_date)`.
- Charts, stats panel, and session history table all consume `filteredHistory` instead of raw `exerciseHistory`.
- Added empty-state message when no sessions fall in the selected range.
