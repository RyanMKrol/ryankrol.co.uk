# T314 — fix archived-tag corner seam on /projects

## Diagnosis (before fix)

Started `npm run dev` (port 3001, port 3000 was already in use by another process),
opened `/projects`, and used a headless-Chrome (Playwright) screenshot at 4x device
scale factor, cropped to a ~24x24 CSS px region around the top-left corner of a card
with `.project-card-archived-tag` injected via devtools-equivalent DOM manipulation
(no repo in the live GitHub API response happened to be archived, so the tag element
was prepended into a real `.project-card` in the page to reproduce the exact CSS in
context).

Observed geometry via `getBoundingClientRect()`:
- `.project-card` origin: `(40, 409)`
- `.project-card-archived-tag` origin: `(42, 411)` — i.e. **already inset by exactly
  `var(--border-width)` (2px) on both axes**, even though the CSS said `top: 0; left: 0;`.
  This is because an absolutely-positioned element's `top`/`left: 0` resolves against
  the *padding box* of its positioned ancestor, which sits inside the ancestor's border.

The zoomed screenshot confirmed the visible defect this geometry produces: the tag
(`border-top-left-radius: var(--radius-card)` = 18px) is drawn as an 18px-radius arc
starting from a point already 2px inside the card's own 18px-radius border arc. Since
an inner shape nested against an outer radius R with an inset I needs a *smaller*
radius (`R - I`) to stay tangent to the outer curve, using the *same* 18px radius for
the inset tag left it visibly under-curved for its position — a thin arc of the black
card border was visible peeking around the outside of the grey tag's curve, i.e. a
sliver of mismatched curvature / a visible seam, rather than one continuous line.

## Fix

`src/styles/globals.css` `.project-card-archived-tag`:
- Made the inset explicit: `top: var(--border-width); left: var(--border-width);`
  (matches the effective behaviour it already had, but no longer relies on an
  implicit absolute-positioning quirk).
- Changed `border-top-left-radius` from `var(--radius-card)` to
  `calc(var(--radius-card) - var(--border-width))` (16px), the standard nested-radius
  formula for a shape inset by `I` inside an outer curve of radius `R`.
- Left `border-bottom-right-radius`, background, padding, font, and the card's own
  CSS untouched, per the task's "reference only" guidance — the visual defect was
  isolated to the top-left corner geometry.

## Confirmation (after fix)

Re-ran the same Playwright screenshot at the same crop/zoom against the reloaded page.
The black border arc and the grey tag's corner now read as a single continuous nested
curve — no gap, no doubled/seamed line. No other card corners are affected (non-archived
cards never render `.project-card-archived-tag`, and `border-bottom-right-radius` was
untouched).
