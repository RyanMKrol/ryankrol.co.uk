# T210 — Projects page masonry grid

## Do

Fixed `.projects-grid` in `src/styles/globals.css` so project cards no longer stretch to
match their row's tallest neighbour and so short cards let following cards flow up into
the freed space (real masonry, not just per-card auto-height).

Implemented CSS multi-column, as decided in the task spec (rejected: `align-items:start`
alone — doesn't reflow; `grid-template-rows: masonry` — Firefox-only; JS-measured masonry —
disproportionate new infra for a personal-site page):

- `.projects-grid`: replaced `display:grid; grid-template-columns: repeat(3, minmax(0,1fr));
  gap:1.25rem;` with `column-count: 3; column-gap: 1.25rem;` (kept `margin-bottom: 2rem`).
- `@media (max-width: 900px) .projects-grid`: `grid-template-columns: 1fr` → `column-count: 1`.
- `.project-card`: added `break-inside: avoid;` (stops a card's content splitting across
  columns) and `margin-bottom: 1.25rem;` (replaces the vertical spacing the old grid `gap`
  provided — multi-column doesn't support `gap` for stacked-item spacing within a column).
- No other selectors changed — `.project-card-*` internals, `.workout-session-grid`, and
  `src/pages/projects/index.js` are untouched.
