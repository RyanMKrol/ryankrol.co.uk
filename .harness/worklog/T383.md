# T383: Make the whole Now Playing pill clickable when a track is playing

## Summary
Refactored the NowPlaying widget to make the entire pill a single clickable link when a track is playing with a Last.fm URL available. Previously, only the marquee text inside the pill was linked; now the entire chip (including the status dot and padding) serves as the clickable target.

## Changes Made

### 1. src/components/NowPlaying.js
- Restructured the component logic to render the outer `.now-playing-chip` as an anchor tag (`<a>`) when `track.track.lastFmUrl` is present
- The anchor now contains both the status dot and the `.now-playing-split` div
- When no URL is available, the chip remains a plain `<div>` (non-clickable)
- The loading and offline states remain unchanged (plain divs)

**Key structural change:**
- Old: `<div class="now-playing-chip"> → <span dot> → <div split> → <a href=...> → <MarqueeText>`
- New: `<a class="now-playing-chip track-link-single" href=...> → <span dot> → <div split> → <MarqueeText>`

### 2. src/styles/globals.css
- Removed `flex: 1` and `min-width: 0` from `.track-link-single` (these were meant for inner text elements, not for use as the outer chip)
- Added a new `.now-playing-chip.track-link-single` rule that explicitly sets all the chip layout properties (display, alignment, border, padding, etc.)
- Added `.now-playing-chip[href]` cursor styling to indicate the chip is clickable
- Kept `.track-link-single:hover { text-decoration: underline }` but added `.now-playing-chip.track-link-single:hover { text-decoration: none }` to prevent underlining the entire chip on hover

**Rationale:**
The anchor element now needs to inherit the chip's full layout rules (inline-flex, border, padding, pill shape) because it IS the chip. The old approach of applying text-sizing rules to the anchor caused layout conflicts; now the anchor properly renders as the pill itself while the internal `.now-playing-split` continues to flex and contain the marquee.

### 3. src/components/NowPlaying.test.js
- Enhanced the existing test assertion to verify the new behavior
- Added checks that the link element has both `now-playing-chip` and `track-link-single` classes
- Added a critical assertion: the `.now-playing-dot` element exists inside the link, proving the entire chip is the clickable unit
- This assertion prevents regressions where the link might accidentally wrap only partial elements again

## Implementation Notes

**Style reconciliation challenge:**
The main challenge was that `.track-link-single` was originally designed for inner text elements (flex: 1, min-width: 0 are text-sizing rules). When the anchor became the outer chip, it needed the chip's layout rules instead. Solution: create a scoped `.now-playing-chip.track-link-single` rule that provides all the display/border/padding, while leaving the original `.track-link-single:hover` for text underlining (removed from the chip variant).

**Marquee compatibility:**
The MarqueeText component still measures and animates correctly when nested inside the full-width flex anchor. The `.now-playing-split` div continues to have `flex: 1` and `min-width: 0`, which allows the marquee to calculate overflow and scroll as before.

**Accessibility:**
The anchor's accessible name comes from the MarqueeText content (`${track.name} by ${track.artist}`), which remains inside it. The link is a single, accessible landmark.
