# T196 — books list: 2-col spine-cover grid

## What changed

- `src/styles/globals.css`: `.spine-cover-list` switched from `display:flex; flex-direction:column`
  (1 card per row) to `display:grid; grid-template-columns:repeat(2,1fr); gap:1.5rem`, matching
  `.square-cover-grid`'s pattern exactly, plus a `@media (max-width:700px)` collapse to
  `grid-template-columns:1fr` (same breakpoint albums uses).
- No changes needed to `ReviewCard.js`'s `SpineCoverCard` or `books/index.js` — the card's
  internal DOM was already a horizontal flex row (image left, body right), so it drops into the
  grid cell unmodified. `.spine-cover-body` already has `min-width:0`, so no overflow issues
  appeared at half-width.

- `npm run build` — succeeded.
- `curl localhost:3000/reviews/books` after `npm run dev` — 200 OK, confirming the page renders
  with the new CSS with no runtime error.
