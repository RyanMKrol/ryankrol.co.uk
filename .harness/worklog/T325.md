# T325 ‚Äî Fix books review card silently regressed off Markdown (re-attempt of T273)

## 2026-07-07

### Root cause (confirmed per task spec)
T273 (d4d5f3b) wired `<Markdown>` into `SpineCoverCard`. The very next same-day commit, T279
(cea04e9, unrelated: added the Read more/Show less truncation toggle), rewrote the same JSX block
and dropped the `<Markdown>` wrapper, reverting books to raw plain-text interpolation. Movies/TV,
albums, and perfumes were untouched by T279 and stayed correct ‚Äî only books regressed.

### What was built
- `src/components/ReviewCard.js`, `SpineCoverCard`: replaced
  `<p className="spine-cover-snippet">{expanded ? fullText : previewText}</p>` with
  `<div className="spine-cover-snippet"><Markdown>{expanded ? fullText : previewText}</Markdown></div>`
  ‚Äî `<div>` instead of `<p>` because block-mode `Markdown` renders its own `<p>` internally (same
  pattern already used for poster-banner/perfume snippets). Truncate/expand logic
  (`truncateReviewText`, `expanded` state, Read more/Show less button) untouched ‚Äî only the render
  target for `expanded ? fullText : previewText` changed.
- `src/components/ReviewCard.test.js` (new): renders `ReviewCard` with `styleVariant="spine-cover"`
  and `review_text` containing `**bold**`, asserts a real `<strong>` renders (not literal `**`).
  Second test uses long text that exceeds the 260-char truncation threshold, clicks "Read more",
  and re-asserts the expanded view also renders `<strong>` rather than raw markdown syntax.
- `scripts/_visual-harness.mjs`: added `src/components/Markdown.js` to the `reviews-books` PAGES
  entry's `covers` array (matching the pattern already used for `reviews-movies`/`reviews-tv`), and
  updated its description to note Markdown bodies render there.
