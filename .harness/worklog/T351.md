# T351 — Hardcode the chosen Top of Mind placement and wire it to real content

Date: 2026-07-10

## What I did

- `src/pages/index.js`: removed the `?tom=`/`tomVariant` query-param branching (and the now-unused
  `useRouter` import) and all 4 rejected variant blocks (rail panel, collapsible, latest-take card,
  hero inline strip). Kept only variant 2's hero-band markup, now unconditional except on real data.
  Removed the `DUMMY_TOM_TEXT` placeholder.
- Added a `topOfMind`/`topOfMindLoading` fetch of `GET /api/top-of-mind` alongside the page's other
  `useEffect` fetches, storing `{ text, updatedAt }` (or `null` if the response has no `text`, which
  is what an unset note returns — `{}`).
- The section (`showTopOfMind`) now renders only when not loading AND a note exists AND
  `!isExpired(topOfMind.updatedAt)` (imported from `src/lib/topOfMind.js`, added in
  top-of-mind-note-storage) — otherwise nothing is rendered for that section at all, per spec (no
  empty/placeholder box). No loading skeleton was added for this section — it's optional per the spec
  and the section is inherently absent-by-default, so skipping it keeps the change minimal.
- `src/styles/globals.css`: removed all CSS for the 4 rejected variants
  (`.tom-variant-rail-panel*`, `.tom-variant-collapsible*`, `.tom-variant-latest-card*`,
  `.tom-variant-hero-strip*`). Renamed the surviving hero-band classes to permanent names:
  `.tom-variant-hero-band` → `.home-top-of-mind-panel`, `-inner` → `-panel-inner`,
  `-label` → `-panel-label`, consistent with this file's other `.home-*` panel classes.
- `scripts/_visual-harness.mjs`:
  - Added a `topOfMindFresh` fixture (`{ text, updatedAt }`, `updatedAt` computed as "5 days ago"
    relative to script run time so it never goes stale as the calendar moves forward) served for
    `GET /api/top-of-mind`, wired into `fixtureFor`.
  - Extended `routeApiWithDelays` so a per-pathname delay entry may also be `{ delayMs?, body }` to
    override the fixture body for one capture, not just delay it (backward compatible — existing
    plain-number delay entries behave exactly as before).
  - Removed the 5 `home-tom-*` `?tom=` PAGES entries and the `home-tom-3-expanded` FLOWS entry
    (all specific to the now-removed rejected variants / query-param switch).
  - Added `home-top-of-mind-absent` (`/`, with `/api/top-of-mind` overridden to `{}`) to prove the
    section renders nothing when no note is saved (also covers the expired case, since both paths
    hit the same "no valid note" render branch).
  - Updated the baseline `home` PAGES entry's `waitFor`/description/covers to reflect the panel now
    being part of the real homepage (using the default fresh fixture).
