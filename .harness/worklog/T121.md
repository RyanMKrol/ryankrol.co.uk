# T121: TV update API — stamp editedDate on every edit

## What changed
- `src/pages/api/reviews/tv/update.js`: kept the existing `originalDate`/`preservedDate`
  logic untouched (original `date` is still preserved across edits). Added a fresh
  `editedDate` computed via `new Date().toLocaleDateString('en-GB').replace(/\//g, '-')`
  (same convention as `date`/`add.js`), unconditionally included in `reviewData` on every
  successful update.
- Added `src/pages/api/reviews/tv/update.test.js` covering:
  - `date` preserved from the existing item while `editedDate` is set to today.
  - Fallback case (no existing item found) where `date` falls back to today, and
    `editedDate` is also today.

## Docs
- Task scope is limited to `src/pages/api/reviews/tv/update.js` (+ tests/worklog only);
  `CLAUDE.md`/`README.md` are not in scope for this task, so they were not edited, matching
  the precedent set by T120 (movies) and T119. Flagging that a follow-up doc task should add
  `editedDate 'DD-MM-YYYY', set on every edit` to the `TelevisionRatingsV4` row in both
  `CLAUDE.md`'s Data model table and `README.md`'s `TelevisionRatingsV4` schema section.
