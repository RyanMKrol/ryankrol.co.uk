# T257 — Build shared bulk metadata-backfill queue + list component

## Built

- `src/lib/backfillQueue.js` — `runBackfillQueue(items, searchFn, options)`. Processes
  items sequentially (never concurrent), enforces `minSpacingMs` (default 1500ms) between
  the START of consecutive `searchFn` calls, retries a rate-limited item exactly once
  (waiting `max(minSpacingMs, retryAfterSeconds*1000)`) via the `err.retryAfterSeconds`
  contract, and returns `{ cancel() }` to stop further items from starting (in-flight call
  is allowed to finish).
- `src/lib/backfillEligibility.js` — `needsMovieBackfill`, `needsTvBackfill`,
  `needsBookBackfill`, `needsAlbumBackfill` pure predicates, one-liners each.
- `src/components/BulkBackfillList.js` — generic paginated list (uses `paginate()` from
  `src/lib/pagination.js` and renders `Pagination.js`). Runs the current page's items
  through `runBackfillQueue`, cancelling the previous run on page change/unmount. Each row
  shows status (queued/searching/found/no matches/saving/saved/error) and, once candidates
  arrive, a radio-style multi-candidate picker (reusing the `.mbm-result-item` class from
  `MetadataBackfillModal` for visual consistency) plus a per-row Confirm button.
