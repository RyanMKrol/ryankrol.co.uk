# T159 — Rebuild the workouts list page

## What changed

- `src/pages/workouts/index.js` rebuilt to the Collection design system:
  - Header uses `collection-review-header` with title + "N sessions logged · via Hevy"
    meta line (kept the "view programme stats" link inline in the meta).
  - Replaced the old custom filter buttons with `PillGroup` (All/Push/Pull/Legs),
    reusing the existing `filterWorkouts` split-matching logic from
    `lib/workoutPagination.js` unchanged.
  - Replaced the hand-rolled pagination block with the shared `Pagination`
    component + `lib/pagination.js`'s `paginate()` (same helper the reviews
    pages use), matching T155.
  - Session cards are a new 2-col grid (`workout-session-grid` /
    `workout-session-card`) built directly in the page rather than reusing
    `WorkoutCard` — `WorkoutCard` renders the full per-exercise set breakdown
    for the workout detail page and isn't in this task's scope to edit, so the
    list card recomputes duration/volume/exercise-count locally with small
    helpers mirrored from it. Each card shows a solid `Badge` colored by split
    and two compact `StatBlock`s (exercise count, volume in kg).
- `src/styles/globals.css`:
  - Added `--split-push` (#FF5C39), `--split-pull` (#F25CA2), `--split-legs`
    (#4B4DED) as their own tokens, distinct from `--accent-movies`/`--accent-tv`
    even though the hex values coincide, per the task's explicit instruction not
    to couple workout split badges to a review section's accent.
  - Added `.workout-session-grid` / `.workout-session-card` / `-card-top` /
    `-title` / `-datetime` / `-stats` (2-col desktop, 1-col under 900px, mirrors
    the existing `.projects-grid` / `.project-card` pattern).
