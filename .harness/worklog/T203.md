# T203 — Fix header nav right-edge fade showing when there's nothing left to scroll to

## What changed
- Added `src/lib/scrollFade.js` — pure `shouldShowScrollFade({ scrollWidth, clientWidth, scrollLeft, epsilon })`
  returning `true` only when the container overflows AND isn't scrolled to its max.
- Added `src/lib/scrollFade.test.js` covering: no overflow, overflow at start, overflow mid-scroll,
  overflow scrolled exactly to max, overflow within epsilon of max (sub-pixel jitter).
- Wired it into `src/components/Header.js`: `navRef` on `.collection-nav-pills`, `showNavFade` state,
  a `useEffect` that computes fade state on mount, on the nav's `scroll` event, on `window resize`, and
  via a `ResizeObserver` on the nav element (covers content-size changes independent of window resize).
  All listeners/observer cleaned up on unmount. Fade is applied via a conditional `has-scroll-fade`
  class instead of unconditionally.
- Restructured `src/styles/globals.css`: removed the unconditional `mask-image`/`-webkit-mask-image`
  from the base `.collection-nav-pills` rule; added `.collection-nav-pills.has-scroll-fade` with the
  same gradient values.
- Updated `CLAUDE.md`'s `src/lib/*` file map with an entry for `scrollFade.js`.
