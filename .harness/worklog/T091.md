# T091 — Fix transparent background on the metadata backfill modal

## What was done
`src/components/MetadataBackfillModal.js` line 143 referenced `var(--color-background)`, a CSS
custom property that is never defined in `src/styles/globals.css` (the file defines `--color-bg`
instead). With no fallback, the property resolved to nothing, leaving the `.mbm-card` results
panel background transparent. Changed the declaration to `background: var(--color-bg);`.

The dimming backdrop (`.mbm-overlay`, `rgba(0, 0, 0, 0.6)`) was left untouched as instructed.

## Verification
- `grep -c "color-background" src/components/MetadataBackfillModal.js` → `0`.
- Confirmed `--color-bg` is defined in `src/styles/globals.css` for all three themes
  (default, matrix, light).
- `npm run lint` — 0 errors (4 pre-existing warnings, unrelated to this change).
- `npm test` — 161/161 tests passed.
- `npm run build` — succeeded.
- Did not start a dev server / manually click through the UI (headless run); the fix is a
  single CSS custom-property name correction verified by grep + the defined-variable check above.
