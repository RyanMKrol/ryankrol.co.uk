# T060 — Variant 3 — reviews edit pages

## What was built

Edit list + edit/[id] detail pages for all 5 review types, under Variant 3's
"Minimal Linear Feed" namespace (`/v3/reviews/<type>/edit`, `/v3/reviews/<type>/edit/[id]`),
reusing the shared `V3Layout` wrapper from T045.

New components (`src/components/v3/`):
- `V3EditFeed.js` — list view: same date-gutter/horizontal-rule timeline rhythm as
  `V3ReviewFeed`, but each row is a plain link to that entry's edit/[id] route (rather than
  an inline expand toggle) since this task's `## Done when` explicitly requires a separate
  edit/[id] route per type, distinct from the inline-disclosure detail pattern described in
  the brief's general timeline entries.
- `V3EditForm.js` — always-expanded form shell in the same visual language as `V3AddEntry`
  (flat, single-typeface, bordered inputs), plus a delete action gated on password like the
  production edit pages.

New pages (`src/pages/v3/reviews/<type>/edit.js` + `edit/[id].js`) for movies, tv, books,
albums, perfumes — each fetches from the unchanged existing API routes
(`GET /api/reviews/<type>`, `POST /api/reviews/<type>/update`,
`DELETE /api/reviews/<type>/delete`) and mirrors the field set of the production
`src/pages/reviews/<type>/edit/[id].js` pages (title/author/artist disabled once matched,
rating, review text field name quirk per type, perfumes' longevity/projection/seasons
controls via the existing shared `PerfumeCharacteristics` component).
