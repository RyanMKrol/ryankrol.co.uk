# T085 — Require a confirmed TMDB match to add a movie review

## What changed

- `src/pages/reviews/movies/add.js`:
  - `handleSubmit` now returns early with an error message if `tmdbMatch` is null, before any
    fetch is made.
  - The submit button is disabled when `!tmdbMatch` (in addition to existing `loading` /
    `rating === 0` conditions).
  - The "TMDB Match (optional)" label was changed to "TMDB Match", and an inline message
    ("Search and select a TMDB match before saving") is shown under the search widget when no
    match is selected.
  - The payload now always spreads the TMDB fields unconditionally (safe, since submission is
    blocked without a match).
- `src/pages/api/reviews/movies/add.js`: added `tmdbId` to the required-fields check, so a
  request missing it now gets HTTP 400 alongside the existing title/rating/gist checks.
- Added `src/pages/api/reviews/movies/add.test.js` (no prior test file existed for this route)
  covering: request missing `tmdbId` → 400, no DynamoDB write; request with `tmdbId` present →
  201 and the item persisted with `tmdbId`.
