# T099 — Fix perfume slider thumbs not sitting flush at track start

## What changed
Added a single shared CSS rule block in `src/styles/globals.css`, scoped to
`input[type='range'].form-input`, targeting the existing `.form-input` class already applied to
all three range sliders (Rating in `add.js`, Longevity/Projection in `PerfumeCharacteristics.js`).
No component files needed changes since all three already share the `form-input` class.

- Removes native appearance (`-webkit-appearance: none` / `appearance: none`) on the input itself.
- Styles `::-webkit-slider-runnable-track` / `::-moz-range-track` and
  `::-webkit-slider-thumb` / `::-moz-range-thumb` explicitly, sizing the thumb (1rem circle) and
  vertically centering it on the 0.5rem track via `margin-top`, which removes the native
  half-thumb-width inset at the minimum value.
- All colors reference existing theme custom properties (`--color-surface-alt`,
  `--color-accent`, `--radius-base`) — no hardcoded hex values, so all four themes (default,
  matrix, and the other palettes) recolor the sliders automatically.
