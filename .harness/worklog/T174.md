# T174 — Projects grid track blow-out guard

## Change
`src/styles/globals.css` only, three edits:
1. `.projects-grid` `grid-template-columns: repeat(3, 1fr)` → `repeat(3, minmax(0, 1fr))`.
   Bare `1fr` tracks have an implicit `auto` minimum (= largest min-content of any item in
   the track, not 0), so a long unbreakable string in one card can force that whole column
   wider and push the grid past its container. `minmax(0, 1fr)` removes that floor.
2. `.project-card` gains `min-width: 0;` — flex items also default to a content-based
   `min-width: auto`, which would otherwise resist shrinking even inside a correctly-sized
   grid track.
3. `.project-card-name` gains `overflow-wrap: anywhere;` so a long unbroken repo name (e.g.
   `ryankrol-co-uk-collection-redesign`) wraps inside the card instead of expanding it.

No changes to `src/pages/projects/index.js` — markup/classNames were already correct per
the task spec.
