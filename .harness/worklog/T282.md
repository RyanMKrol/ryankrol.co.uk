# T282 — Migrate PerfumeRatings rows to ownership field

## What was built

- `src/scripts/migratePerfumeOwnership.js` — dry-run-by-default (gated on `LIVE=1`) migration
  script modelled on `src/scripts/migrateOpenLibraryBooksToGoogleBooks.js`. Reads the table name
  from `DYNAMO_TABLES.PERFUME_RATINGS_TABLE`. Scans for rows with `ownership === undefined` AND
  (`considerTravelSize` or `considerFullBottle` defined), derives `ownership` via the exported
  pure `deriveOwnership(row)` helper (full bottle > travel size > sample), logs the
  before/after for every matching row, and only issues `UpdateCommand`s (`SET ownership REMOVE
  considerTravelSize, considerFullBottle`, keyed on `id`) when `LIVE=1`.
- `src/scripts/migratePerfumeOwnership.test.js` — unit tests for `deriveOwnership` covering all
  three priority cases.
- `package.json` — added `"perfume:migrate-ownership": "node src/scripts/migratePerfumeOwnership.js"`.

## Verification observed

- `npm run lint` — 0 errors, 6 pre-existing warnings (unrelated files).
- `npm test` — 66 suites / 375 tests passed, including the 3 new `deriveOwnership` tests.
- `npm run build` — succeeded.
- Did not run the script live against DynamoDB (no guaranteed AWS credentials in the build
  environment, and per task instructions this should not be run live as part of this task).
  Code was reviewed by hand against the spec instead: dry-run path only logs and never calls
  `UpdateCommand` unless `IS_LIVE` is true.
