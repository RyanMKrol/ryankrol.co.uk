# T183 — Perfume card variant 4: Season & Occasion Planner

## What was built

`src/components/perfumeVariants/Variant4SeasonPlanner.js` — standalone presentational
component, `{ item }` prop, no dependency on ReviewCard or a parent container.

- `seasons` (if present) rendered as large pill chips under a "Best for" label at the top
  of the card. Absent entirely (no label, no chips) when `seasons` is missing.
- `longevity`/`projection` rendered as horizontal scale bars labelled in plain language
  ("Light ↔ All-day", "Skin scent ↔ Room-filling"), fill width computed as
  `longevity/8` and `projection/4` — no raw integers shown.
- `applicationSpots` (if present) rendered as a checklist ("2 sprays — Wrists", ...).
- Title/designer/type/rating shown as a small, de-emphasised header line at the bottom.
  `description` is intentionally not rendered.
- Fragrantica link included when present.

New CSS: `.perfume-v4-*` rules appended to the end of `src/styles/globals.css` — no
existing rule was edited or removed (confirmed via `git diff` showing only additions).
