# T039 — Perfume longevity/projection/seasons fields — form UI

## What was built

- New `src/components/PerfumeCharacteristics.js` with three shared presentational components:
  `LongevitySlider` (0-8 range input, shows "8+" at max), `ProjectionSlider` (1-4 range input,
  showing the Fragrantica-style label live: Skin scent/Moderate/Strong/Beast mode), and
  `SeasonsCheckboxes` (six independent checkboxes for Winter/Spring/Summer/Autumn/Day/Night,
  toggling membership in an array).
- Wired all three into `src/pages/reviews/perfumes/add.js`: added `longevity` (default 0),
  `projection` (default 1), `seasons` (default `[]`) to `formData`, added change handlers, and
  render the controls above the password field. They're included in the POST body automatically
  since the whole `formData` object is submitted.
- Wired the same three controls into `src/pages/reviews/perfumes/edit/[id].js`, pre-populated from
  the loaded review via `perfume.longevity ?? 0`, `perfume.projection ?? 1`,
  `perfume.seasons || []` — falls back to sensible defaults for reviews predating these fields.

## Verification

- `npm run lint` — 0 errors, 1 pre-existing warning (dev/cache.js, unrelated).
- `npm test` — 136/136 passing (no new pure logic needed a test; label maps are trivial constant
  objects, no branching logic to unit test beyond what's exercised by lint/build).
- `npm run build` — succeeds; `/reviews/perfumes/add` and `/reviews/perfumes/edit/[id]` both build
  as static/dynamic routes with the new components bundled (105 kB / 105 kB First Load JS,
  in line with sibling review add/edit pages).
- Did not run a live dev server / manual browser drag-through (no DynamoDB/live API calls made,
  per harness rules) — the "8+", live projection labels, and seasons array submission were
  verified by code inspection: `LongevitySlider` renders `value >= 8 ? '8+' : value`,
  `ProjectionSlider` renders `PROJECTION_LABELS[value]` for the current slider value on every
  onChange, and `SeasonsCheckboxes.toggleSeason` adds/removes the exact season string
  (`"Winter"`, `"Spring"`, `"Summer"`, `"Autumn"`, `"Day"`, `"Night"`) to/from the `seasons` array
  passed straight into `formData.seasons`, which is submitted verbatim in the POST body.
