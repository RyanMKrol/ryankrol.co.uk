# T082 worklog

## What was built

- `src/scripts/backfillIncompleteWorkoutMetrics.js` — dry-run-by-default script that:
  1. Scans `Workouts` for items missing the denormalized `exercises` field (`isIncomplete`).
  2. Queries `Exercises` via the `workout_id-index` GSI for each incomplete workout's rows.
  3. Reshapes those rows into the `{ title, sets }[]` shape `calculateWorkoutMetrics` expects
     (`reshapeExerciseRows`), then reuses `calculateWorkoutMetrics` from `src/lib/workoutMetrics.js`
     (no reimplemented metric math) to compute `totalVolume`, `totalWarmupSets`,
     `totalWorkingSets`, `uniqueExercises`, `durationMinutes`, `workoutType` (`buildMetricsUpdate`).
  4. Prints a per-workout summary; only calls `UpdateCommand` when `LIVE=1` is set.
- `package.json`: added `"workout:backfill-metrics": "node src/scripts/backfillIncompleteWorkoutMetrics.js"`.
- `src/scripts/backfillIncompleteWorkoutMetrics.test.js`: fixture-only tests (no AWS calls) covering
  `isIncomplete`, `reshapeExerciseRows`, and that `buildMetricsUpdate`'s output for a fixture
  incomplete workout + fixture Exercises rows matches `calculateWorkoutMetrics` called directly on
  the equivalent reshaped input.
