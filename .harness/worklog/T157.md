# T157 — Rebuild listening page (design handoff)

## What changed
- `src/pages/listening/index.js`: rebuilt the ranked album list markup — big "listening"
  title + `top 50 albums · last 90 days · via last.fm` meta line (using the existing
  `.collection-review-header` / `.collection-review-title-group` / `.collection-review-meta`
  primitives already used on `vinyl`/`albums`). Each row is rank number + small cover
  thumbnail (plain `<img>`, falling back to `CoverTile`'s `gradientForKey` gradient when
  Last.fm has no cover) + title/artist + proportional playcount bar + raw playcount. Row #1
  gets a `listening-row-top` class for the grape-filled/white-text treatment; rows 2+ render
  as white bordered cards with a grape bar on a light track.
- `src/styles/globals.css`: replaced the old `.listening-item` / `.popularity-bar` /
  `.listening-rank.top-3` rules with a new `.listening-row*` rule set (uses existing
  `--accent-listening` / `--radius-card` / `--border-width` tokens, no new tokens added) plus
  a small mobile stack breakpoint for the bar.
- No data-layer changes — reused the existing `/api/lastfm/top-albums` fetch and playcount
  math (`maxPlaycount`) as-is.
