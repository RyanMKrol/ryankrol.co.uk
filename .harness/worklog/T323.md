# T323 — 2026-07-07

## What
Normalized `.workout-session-exercise-item` row height in `src/styles/globals.css` so
exercise-preview rows on /workouts render at the same height whether or not they carry a
`PrBadge` pill.

## Why
`.collection-badge` (padding 0.25rem 0.75rem, font-size 0.75rem) + the `.pr-badge` 1px border
renders taller (~24.4px) than the row's own 0.85rem monospace text line box, so PR-bearing rows
were taller than plain-text rows in the same `.workout-session-exercise-list`, even though the
list's `gap: 0.2rem` never changed — making PR-heavy cards look more loosely spaced.

## Change
- Set `line-height: 1.6rem` (25.6px) on `.workout-session-exercise-item` — comfortably exceeds
  the badge's rendered box height (~24.4px), so the line box no longer needs to grow to
  accommodate the inline badge, giving every row (badge or plain) the same fixed row height.
- Added `.workout-session-exercise-item .collection-badge { vertical-align: middle; }` so the
  badge centers within that fixed line box instead of aligning to text baseline.
- Deliberately did NOT switch the row to `display: flex` — the `<li>` mixes a raw text node with
  inline badge(s) (JSX in `src/pages/workouts/index.js`, out of scope for this task), and
  `text-overflow: ellipsis` does not reliably apply across flex-item children; keeping the row a
  normal inline-flow block preserves the existing single-line ellipsis truncation for long
  exercise names.
- `.workout-session-exercise-list`'s `gap: 0.2rem` untouched, per the Done-when criteria.
