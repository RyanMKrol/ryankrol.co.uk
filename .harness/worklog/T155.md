# T155 ‚Äî pagination for review list pages

## What was built
- `src/lib/pagination.js` ‚Äî generic `getPageCount(total, pageSize)` and
  `paginate(items, page, pageSize)` (default page size 10), mirroring the shape/signature of the
  existing `src/lib/workoutPagination.js`. `paginate` returns `{ items, page, pageCount, total,
  hasNext, hasPrev }`, clamping out-of-range page numbers into `[1, pageCount]`.
- `src/lib/pagination.test.js` ‚Äî covers exact page-size boundaries, a partial last page,
  out-of-range page clamping (both above range and below 1), and the single-page case.
- `src/components/Pagination.js` ‚Äî presentational control built on the existing `Pill` primitive
  (from T143). Renders numbered pills for page 1, the last page, and a window around the current
  page, with `‚Ä¶` filling gaps (matches the `1 2 3 ‚Ä¶ 32` reference in
  `design_handoff/renders/2a.png`), plus a trailing `next ‚Üí` pill. Controlled via
  `currentPage`/`totalPages`/`onPageChange`; renders nothing when `totalPages <= 1`.
- Added `.collection-pagination` / `.collection-pagination-ellipsis` to `globals.css`, styled with
  the same Collection design tokens as the other T143 primitives (note: this file already had an
  older, unrelated `.pagination-*` block from a pre-redesign pagination UI ‚Äî left untouched, new
  classes are additively named to avoid collision).
- Wired `Pagination` + `paginate()` into all four rebuilt list pages
  (`reviews/{movies,tv,books,albums}/index.js`), replacing render-all-at-once. Page sizes chosen
  per layout: movies/tv (3-col poster grid) = 9, albums (2-col grid) = 8, books (single-column
  list) = 12. Added a `currentPage` state plus a `useEffect` that resets it to 1 whenever
  `searchTerm` or `sortBy` changes, so a shrunk filtered/sorted result set never strands the user
  past its last page.
