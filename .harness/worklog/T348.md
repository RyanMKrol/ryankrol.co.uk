# T348 — Top of Mind note (data + API + edit page)

2026-07-10

## What was built
- `DYNAMO_TABLES.TOP_OF_MIND_TABLE = 'TopOfMind'` in `src/lib/constants.js`.
- Real DynamoDB table `TopOfMind` provisioned out-of-band (partition key `id`, String,
  PAY_PER_REQUEST) via an ad-hoc `node -e` script using the same AWS SDK calls as
  `src/scripts/createHotTakesTable.js` — NOT committed as a script file, since `src/scripts/`
  is outside this task's allowed scope. Table confirmed ACTIVE.
- `src/lib/topOfMind.js` — pure helpers: `TTL_DAYS = 90`, `MAX_TEXT_LENGTH = 20000`,
  `isExpired(updatedAt, now)` (strictly-greater-than-90-days semantics), `daysSinceUpdate`,
  `daysRemaining`, `validateTopOfMindText`.
- `src/pages/api/top-of-mind/index.js` — GET, `withApiCache('api-top-of-mind')` wrapping a
  `GetCommand` for the fixed key `{ id: 'top-of-mind' }`; returns `{}` (never 404) if unsaved.
- `src/pages/api/top-of-mind/update.js` — POST, gated on `RYANKROL_SITE_KEY`, validates text,
  always `PutCommand`s the fixed-key row with a fresh `new Date().toISOString()`, clears
  `api-top-of-mind` cache.
- `src/pages/top-of-mind/edit.js` — fetches the note on mount (prefilling even expired text),
  renders `MarkdownEditor` + live `Markdown` preview, shows a last-updated/expires-in/expired
  indicator computed from `src/lib/topOfMind.js`, and a password-gated save form matching the
  hot-takes add form's fetch/message-state shape.
- `src/lib/topOfMind.test.js` — boundary tests for `isExpired` (90d-1ms not expired, exactly
  90d not expired, 90d+1ms expired), `validateTopOfMindText` (empty/whitespace/over-length
  rejected, normal/at-cap accepted), and `daysSinceUpdate`/`daysRemaining` arithmetic sanity.
