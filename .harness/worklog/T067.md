# T067 — Accept metadata-backfill fields on all four review update APIs

## What changed

- `src/pages/api/reviews/books/update.js` — now destructures and conditionally persists
  `source, olid, coverId, coverUrl, volumeId, bookAuthors, firstPublishedYear, isbn, subjects,
  pageCount, publisher` onto the `PutCommand` item, mirroring `add.js`'s
  `...(field !== undefined && { field })` spread pattern.
- `src/pages/api/reviews/albums/update.js` — now accepts an optional `lastfm` object. Since this
  route uses `UpdateCommand` (not `PutCommand`), the `UpdateExpression`/`ExpressionAttributeValues`
  are now built dynamically: when `lastfm` is present, `thumbnail` (derived the same way as
  `add.js`: `lastfm?.images?.[lastfm.images.length - 1]?.['#text'] || ''`) and `lastfm` are added
  to the SET clause; when absent, the expression is unchanged from before (`rating`, `highlights`
  only).
- `src/pages/api/reviews/movies/update.js` and `src/pages/api/reviews/tv/update.js` — both now
  accept `tmdbId, mediaType, posterPath, tmdbOverview, tmdbDate` and persist them conditionally on
  the `PutCommand` item, matching their respective `add.js` routes exactly.

In all four, required-field validation (`title`, `rating`, review-text field, `originalTitle`/
`originalAuthor`/`originalArtist`) and existing behavior (date preservation, delete-then-recreate
on key change for books/movies/tv) are untouched.

## Tests

No API-route test pattern existed in this repo prior to this task (only pure `src/lib` logic is
tested). Added `src/pages/api/reviews/books/update.test.js` as the first API-route test, mocking
`docClient.send` and `clearApiCache`. It covers:
1. Optional metadata fields (`source`, `volumeId`, `isbn`, `publisher`) are persisted onto the
   `PutCommand` item when present in the request body.
2. Those same fields are cleanly omitted from the persisted item when absent from the request
   body (the pre-existing/no-new-fields case still works).

Chose the books route as the representative case since it uses the same `PutCommand` +
conditional-spread pattern as movies/tv, and is the most field-heavy of the four.
