# T279 — Truncate long book review text with Read more/Show less

## What changed
- Added `src/lib/reviewText.js`: pure `truncateReviewText(text, maxLength = 260)` helper.
  Cuts at the last whitespace at-or-before `maxLength`, falling back to a hard cut when no
  whitespace exists in range. Returns `{ text, truncated }`, appending a single `…` when
  truncated.
- Added `src/lib/reviewText.test.js` covering: short text unchanged, empty string, word-boundary
  truncation, hard-cut fallback (no whitespace), and a custom `maxLength`.
- `src/components/ReviewCard.js`: `SpineCoverCard` (the `/reviews/books` + `/reviews/books/edit`
  variant) now tracks `expanded` state, truncates the review to 260 chars by default, and shows
  a "Read more" / "Show less" toggle button only when the text was actually truncated. No other
  variant (`SquareCoverCard`, poster-banner, default) was touched.
- `src/styles/globals.css`: added `.spine-cover-expand-btn` styled as an underlined text link in
  `--color-accent-secondary` (books' accent), next to the existing `.spine-cover-snippet` rule.

## Verification
- `npm run lint` — 0 errors (6 pre-existing warnings, unrelated to this change).
- `npm test` — 363/363 passed, including the 5 new `reviewText.test.js` cases.
- `npm run build` — succeeded, all pages generated.
- Did not run `npm run dev` / open a browser (headless session) — verified the truncate/expand
  logic is pure and unit-tested, and confirmed via code review that `SpineCoverCard` is the only
  component both books pages render through (`styleVariant="spine-cover"`), so the fix applies to
  both `/reviews/books` and `/reviews/books/edit` without further changes. No live DynamoDB/API
  calls were made per the no-metered-calls rule.

## Note
Per the spec, the previous `<Markdown>` rendering of the snippet was replaced with plain text
(`<p>{...}</p>`) inside `SpineCoverCard`, since the spec explicitly describes plain preview/full
text swapping under `expanded` state, not markdown-rendered truncation.
