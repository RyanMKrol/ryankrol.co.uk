# T047 — Variant 1 — reviews add forms

## What was built

Added `/v1/reviews/{movies,tv,books,albums,perfumes}/add` pages, matching the
production `/reviews/<type>/add` forms field-for-field but restyled per Variant 1's
brief (dense monospace label:value rows, no card/box styling, panel capped at 420px
width like the detail slide-in panel). Reused T043's `V1Layout` shell for the
sidebar/breadcrumb/search chrome.

Added `src/components/v1/V1FormShell.js` (shared `V1FormShell` + `V1FormRow` +
`V1FormSubmit`) so the 5 pages don't each duplicate the dense-form CSS.

Each add page reuses the existing production search/input subcomponents unchanged
(`StarRating`, `TmdbSearch`, `BookSearch`, `LastfmAlbumSearch`,
`PerfumeCharacteristics`), which themselves call the existing, unmodified API routes
(`/api/tmdb/search`, `/api/books/search`, `/api/lastfm/album-search`, etc.) plus the
existing write endpoints (`/api/reviews/<type>/add`) on submit. No file under
`src/pages/api/**` was touched.

## Verification observed

- `npm run lint` — 0 errors (2 pre-existing warnings, unrelated to this change).
- `npm test` — 137/137 passing.
- `npm run build` — succeeded; all 5 new routes built as static pages
  (`/v1/reviews/movies/add`, `.../tv/add`, `.../books/add`, `.../albums/add`,
  `.../perfumes/add`).
- `npm run dev` + `curl` — all 5 routes returned HTTP 200; dev server log had no
  errors.

## Scope

Only touched `src/pages/v1/reviews/{movies,tv,books,albums,perfumes}/add.js` and
`src/components/v1/V1FormShell.js`. No file under `src/pages/api/**` or any
production `src/pages/reviews/**` path was modified.
