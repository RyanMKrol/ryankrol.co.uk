# T110: Switch movies to MovieRatingsV4 (id-keyed)

## What changed
- `src/lib/constants.js`: `MOVIE_RATINGS_TABLE` now `'MovieRatingsV4'`.
- `src/pages/api/reviews/movies/add.js`: assigns `id: randomUUID()` on every new item.
- `src/pages/api/reviews/movies/update.js`: takes `originalId` (was `originalTitle`), `GetCommand`/`PutCommand`
  keyed by `{ id }`; removed the title-changed delete-old/recreate-new branch entirely (id never
  regenerated on update).
- `src/pages/api/reviews/movies/delete.js`: takes `{ id, password }` (was `{ title, password }`),
  `DeleteCommand` keyed by `{ id }`.
- `src/pages/api/reviews/movies/index.js`: unchanged — already scans `DYNAMO_TABLES.MOVIE_RATINGS_TABLE`,
  which now resolves to `MovieRatingsV4` and returns items including `id`.
- `src/pages/reviews/movies/edit.js`: list `key` and edit link now use `movie.id`.
- `src/pages/reviews/movies/edit/[id].js`: looks up by `m.id === decodeURIComponent(id)`; submits
  `originalId`/`id` instead of `originalTitle`/`title` on update/delete.
- `src/pages/api/reviews/movies/add.test.js`: asserts the created item's `id` is UUID-shaped.
- Docs: `CLAUDE.md` data model row updated for `MovieRatingsV4`; `README.md` gets a new
  `#### MovieRatingsV4 Table` section (there was previously no movies/tv section in README at all).
