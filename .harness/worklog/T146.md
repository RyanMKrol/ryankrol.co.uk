# T146 — Collection persistent nav (Header rebuild)

## What changed
- `src/components/Header.js`: rebuilt as the persistent Collection nav — wordmark (circular
  "r" avatar + "ryan krol") on the left, 9 section pills (books/movies/tv/albums/perfumes/
  vinyl/workouts/listening/projects) in the center using `Pill` (T143), each wrapped in a
  `Link`. Active section is derived from `useRouter().pathname` matched against a route-prefix
  table (`/workouts` and `/exercises` both map to the workouts section) and rendered with
  `accentColor="var(--accent-<section>)"` from T142's tokens. `NowPlaying` sits on the right.
  A ☰ button toggles a simple mobile dropdown listing the same 9 links (no design mock for the
  open state, so implementation is free-form per the spec).
- `src/components/NowPlaying.js`: only the wrapper markup/classNames changed (chip with a dot
  indicator) — the fetch/poll/state logic is untouched.
- `src/styles/globals.css`: added wordmark, nav-pill-row, mobile-menu, and now-playing-chip
  styles; updated the `max-width: 768px` block to hide the pill row and show the ☰ toggle.
- `_app.js` was NOT changed — confirmed `Header` is rendered per-page (e.g.
  `src/pages/vinyl/index.js`, `src/pages/workouts/index.js`), not globally, so there was no
  wiring to update.
