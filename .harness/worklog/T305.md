# T305 — masonry columns for perfumes edit page

## Do

`src/pages/reviews/perfumes/edit.js` was rendering perfume cards as direct children of a bare
`<div className="perfume-card-grid">` with no column-wrapping — it just flowed however-many-fit
per row. Fixed to match `src/pages/reviews/perfumes/index.js` (the view page) and the
`src/pages/reviews/books/edit.js` reference pattern:

- Imported `MasonryColumns` from `../../../components/MasonryColumns` and
  `useResponsiveColumnCount` from `../../../hooks/useResponsiveColumnCount`.
- Added `const columnCount = useResponsiveColumnCount(2, 700);` alongside the existing `useState`s.
- Replaced the bare `<div className="perfume-card-grid">{perfumes.map(...)}</div>` with
  `<MasonryColumns items={perfumes} columnCount={columnCount} className="perfume-card-grid"
  columnClassName="perfume-card-grid-col" renderItem={(perfume, index) => (...)} />`, moving the
  existing per-card JSX (the `review-card-with-edit review-card-with-edit--perfume` wrapper +
  `Variant6Hybrid` + Edit `Link`) into `renderItem` unchanged.
- Did not touch `.perfume-card-grid` / `.perfume-card-grid-col` CSS in `globals.css` (already
  correct, shared with the view page).
