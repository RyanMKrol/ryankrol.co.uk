- `src/components/MarkdownEditor.js`: `applyChange` now captures `textareaRef.current.scrollTop`
  before calling `onChange`, and restores it (`el.scrollTop = scrollTop`) inside the same
  `requestAnimationFrame` callback that already restores focus + selection range. Guarded with
  the same `if (el)` style as the existing restores; no second rAF, no new ref.
- `src/components/MarkdownEditor.test.js`: added a 6th test that builds a long (500-line) demo
  textarea, sets a real selection near the end, sets `scrollTop = 300`, spies on the `scrollTop`
  setter (cleared right after the manual assignment), clicks Bold, flushes the scheduled rAF via
  `await new Promise((resolve) => requestAnimationFrame(resolve))`, then asserts the setter was
  called with `300` and the end-state `scrollTop` is `300`.
