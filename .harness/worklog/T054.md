# T054 — Variant 2 — reviews edit pages

## What was built

Added the 5 review-type edit list pages and edit/[id] detail forms for Variant 2
(Editorial Card Magazine), under `/v2/reviews/{movies,tv,books,albums,perfumes}/edit`
and `/v2/reviews/{movies,tv,books,albums,perfumes}/edit/[id]`, reusing the existing
`V2Layout` shared chrome from T044 and the `V2ArticleForm` article-shell form component
from T053 (no second layout or form wrapper built).

- `src/components/v2/V2EditList.js` — new shared "management list" component: same
  masonry-card visual language as `V2ReviewList` (T052) but without the hero-featured
  item (edit lists are for management, not browsing) and with an "Edit →" link on each
  card instead of a sort control.
- `src/components/v2/V2ArticleForm.js` — extended with an optional `secondaryAction`
  prop so the edit forms can render a destructive "Delete review" button alongside the
  "Save changes" submit button, styled as an outlined red button next to the solid
  black submit button (kept both in a shared `.v2-form-actions` row).
- `src/pages/v2/reviews/{movies,tv,books,albums,perfumes}/edit.js` — list pages using
  `V2EditList`, fetching from the unchanged `GET /api/reviews/<type>` routes.
- `src/pages/v2/reviews/{movies,tv,books,albums,perfumes}/edit/[id].js` — detail forms
  using `V2ArticleForm`, mirroring each production edit page's field set, decode/lookup
  logic (movies/tv by title, books by `title|author`, albums by `title|artist`,
  perfumes by `id`), and submit to the unchanged `POST /api/reviews/<type>/update` /
  `DELETE /api/reviews/<type>/delete` routes with the same request bodies as production.

## Verify

- `npm run lint` — 0 errors (2 pre-existing warnings, unrelated files).
- `npm test` — 137/137 passing (no new pure logic added; existing coverage unaffected).
- `npm run build` — succeeds; all 10 new routes appear as static pages.
- `npm run dev` + `curl` — all 10 routes (`/v2/reviews/{type}/edit` and
  `/v2/reviews/{type}/edit/test`) return `200`, no console errors observed.

## Scope check

Only files under `src/pages/v2/**` and `src/components/v2/**` were added/changed. No
file under `src/pages/api/**` or the production `src/pages/reviews/**` was touched.
