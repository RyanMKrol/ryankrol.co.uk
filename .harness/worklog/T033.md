# T033 worklog

## Summary
Drive book search from title + author form fields, removing the standalone search input in BookSearch.

## Changes
- `src/pages/api/books/search.js`: replaced `?query=` param with `?title=` (required) + `?author=` (optional). Open Library URL now uses dedicated `title=` and `author=` search params instead of `q=`. Cache key includes both params. Missing `title` returns 400.
- `src/components/BookSearch.js`: removed the `<input>` element, `query` state, `initialQuery` prop, and `handleKeyDown`. Now accepts `title` and `author` props from the parent. Search button disabled when `title` is empty/whitespace.
- `src/pages/reviews/books/add.js`: updated `<BookSearch>` to pass `title={formData.title}` and `author={formData.author}` instead of `initialQuery`.
- `CLAUDE.md`: updated API table entry for `/api/books/search` to document `title`/`author` params and 400 behaviour.
