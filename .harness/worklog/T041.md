# T041 — Harness dashboard rework — rendering + interactions (server.js)

## What changed

Reworked the client-side `<script>` embedded in `server.js`'s `PAGE` template to match
`local-jobs`'s Backlog page in structure/behavior, on top of T040's four-bucket
`computeBacklog` (`ready` / `waiting` / `needsHuman` / `done`). No npm deps added — still a
plain string-templated vanilla-JS page.

1. `ORDER`/`CHIPS` rewritten for the new bucket names; `ORDER.map(...)` now always renders
   all four sections (each with its own "None." placeholder when empty) — removed the old
   `ORDER.filter(([k])=>groups[k]&&groups[k].length)` that hid empty sections.
2. Replaced the one-shot `fetch('/api/backlog')...` with `refreshNow()` + `setInterval(refreshNow,
   5000)`; `render(d)` re-populates `#main`/`#counts` on every tick. Header copy now says
   "Auto-refreshes."
3. Added `depLinks(ids)` — renders each dependency id as a `.dep-link` span with an
   `onclick="openTask(id)"`. `window.openTask` adds the `.open` class (expanding the target
   row wherever it currently lives, including inside a different section), scrolls it into
   view, and toggles a `.flash` class for 1.5s (CSS transition on `background-color` using the
   existing `--acc` var) for the highlight.
4. Wrapped worklog/audit content in `<details class="buildlog"><summary>Build log</summary>...`
   / `<summary>Audit</summary>` — collapsed by default, matching local-jobs.
5. `pill(t)` now branches on the new bucket names; a `needsHuman` task with `t.blocked` gets a
   distinct "⚠ blocked (loop gave up)" pill (new `.p-blocked` style) instead of the plain
   "🔒 needs human" pill, so loop-gave-up vs. plain gate/needs-human read differently. `done`
   keeps `failed` (red) vs. plain done (green) pills, unchanged.
6. Kept unchanged: the needs-human bulk "select all + mark done", the done-section
   Show All/Reviewed/Not-reviewed filter + bulk "mark reviewed", and all `/api/mark-*`
   endpoints (no server-side mutation changes).

Also added `window.__openIds` (a Set of manually-expanded task ids) so the 5s auto-refresh
re-render doesn't collapse rows the user has open — re-applied after each `render()` call.
