# T258 — Bulk metadata-backfill list page for movies

## What was built
- `src/pages/reviews/movies/backfill.js`: fetches `/api/reviews/movies`, filters with
  `needsMovieBackfill`, renders a single page-level password field, then
  `<BulkBackfillList>` (pageSize 15) reusing the exact TMDB search/candidate-render/confirm
  logic from `src/pages/reviews/movies/edit/[id].js` (copied, not abstracted, per the spec).
  - `onSearch` mirrors `handleBackfillSearch`; on a 429 it reads the `Retry-After` header and
    attaches `error.retryAfterSeconds` so `backfillQueue.js`'s documented retry-once contract
    fires (the JSON body from `/api/tmdb/search` doesn't carry `retryAfterSeconds` itself, only
    the header does — see `src/pages/api/tmdb/search.js:12`).
  - `onConfirm` POSTs to `/api/reviews/movies/update` with the full required body
    (`title`/`rating`/`gist`/`originalId`/`password`) plus the new TMDB fields, matching
    `handleBackfillConfirm`'s field mapping.
  - Empty filtered list renders "Nothing to backfill" instead of an empty list component.
- `src/pages/reviews/movies/edit.js`: added a "Backfill metadata" link to
  `/reviews/movies/backfill` under the page title (small scoped `<style jsx>` block — did not
  touch the global `.edit-button-overlay` class since it's absolutely positioned and not a good
  fit inline in the page flow here, and globals.css is out of this task's scope anyway).
