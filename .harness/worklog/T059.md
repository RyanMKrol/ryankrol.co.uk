# T059 — Variant 3 — reviews add forms

## What was built

Added inline add forms for all 5 review types under Variant 3's minimal-linear-feed
route namespace:

- `src/pages/v3/reviews/movies/add.js`
- `src/pages/v3/reviews/tv/add.js`
- `src/pages/v3/reviews/books/add.js`
- `src/pages/v3/reviews/albums/add.js`
- `src/pages/v3/reviews/perfumes/add.js`
- `src/components/v3/V3AddEntry.js` — shared wrapper matching the brief's "pinned
  add entry that expands into an inline form" pattern: same gutter/rule timeline
  entry visuals as `V3TimelineEntry`, `[type]` label, `+`/`−` disclosure toggle
  (starts expanded on the dedicated `/add` route since there's no live feed to pin
  it to), submit handler, inline save/error message, then collapses after a
  successful save.

Each page reuses V3Layout (T045) for the utility-bar/`Jump to` chrome and posts to
the unchanged production API route (`/api/reviews/<type>/add`) with the same body
shape as the production `src/pages/reviews/<type>/add.js` forms, including the
external-match integrations (TmdbSearch for movies/tv, BookSearch for books,
LastfmAlbumSearch for albums, PerfumeCharacteristics sliders/checkboxes for
perfumes) — those components are imported unmodified from `src/components/*`, not
duplicated.

Field markup (labels/inputs/textarea/select) is restyled minimally to fit
Variant 3's single-size/no-heading typography inside `V3AddEntry`'s scoped CSS,
rather than pulling in the production `Header`/form CSS classes.
