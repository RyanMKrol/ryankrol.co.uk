# T394 Worklog

## Summary
Added CSS styling for GFM table elements (table/thead/tbody/tr/th/td) in Markdown-rendered content across all five Markdown rendering containers. All styling uses existing Collection design tokens (no hardcoded colors/values introduced). Added a synthetic Markdown table fixture to the visual-check harness to enable screenshot verification.

## Work Done

### 1. CSS Table Styling (src/styles/globals.css)
Added scoped table styling for each Markdown-rendering container:

**Containers styled:**
- `.review-text` — Review bodies in movie/TV/book/album reviews
- `.spine-cover-snippet` — Book review card bodies
- `.poster-banner-snippet` — Movie/TV review card bodies
- `.perfume-v1-description` — Perfume review descriptions
- `.home-top-of-mind-panel-inner` — Top-of-mind homepage panel

**Styling applied to each container:**
- `table`: `border-collapse: collapse; width: 100%; display: block; overflow-x: auto;` (horizontal scrolling on narrow viewports)
- `th` (headers): 
  - Font: uppercase, letter-spaced, `var(--font-body)`, `var(--color-ink-mute)`
  - Background: `var(--color-hairline)` (subtle shading)
  - Border: `var(--border-width) solid var(--color-ink)` (heavier bottom border)
  - Padding: `0.5rem 0.65rem`
- `td` (data cells):
  - Font: `var(--font-body)`, context-appropriate color (`var(--color-ink)` for most; `var(--color-text-secondary)` or `var(--color-ink-soft)` where container already uses those)
  - Border: `1px solid var(--color-hairline-strong)` (lighter bottom border)
  - Padding: `0.5rem 0.65rem`

All token values already existed in the design system — no new hardcoded hex/px values were introduced.

### 2. Synthetic Fixture Addition (scripts/_visual-harness.mjs)

- Updated Parasite movie review fixture to use `MD_TABLE` as its `review_text`
- This ensures the `reviews-movies` visual check screenshot captures a rendered table
