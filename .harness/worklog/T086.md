# T086 — Require a confirmed TMDB match to add a TV review

## What changed
- `src/pages/reviews/tv/add.js`: `handleSubmit` now blocks (returns early with an inline
  error message "Search and select a TMDB match before saving") when `tmdbMatch` is null.
  The submit button is disabled until a match is confirmed. TMDB fields are now always
  spread unconditionally into the payload (safe since submission is blocked without a match).
  Label changed from "TMDB Match (optional)" to "TMDB Match" with an inline warning shown
  when no match is selected — mirrors the movies add page (T085).
- `src/pages/api/reviews/tv/add.js`: added `tmdbId` to the required-fields check, returning
  400 if missing, alongside title/rating/gist.
- Added `src/pages/api/reviews/tv/add.test.js` (new test file) asserting: request missing
  `tmdbId` is rejected 400 without hitting DynamoDB; request with `tmdbId` present succeeds
  (201), persists `tmdbId` in the DynamoDB item, and clears the `api-tv` cache key.
