# T378: Fix MarkdownEditor corners + native resize grip

## Summary

CSS-only polish scoped to `.markdown-editor .collection-form-textarea` in `src/styles/globals.css`. The
shared `.collection-form-textarea` base class (`--radius-stat`, `resize: vertical`) was already correct,
but the browser (Chromium) still renders its native diagonal resize-grip icon in the bottom-right corner
even with `resize: vertical` set — that icon was the visible "unpolished" artifact. Added an explicit
symmetric `border-radius: var(--radius-stat)` + `resize: vertical` (redundant with the base class but
explicit per the task's scope, and immune to any future base-class change) plus a
`::-webkit-resizer { display: none; }` override to hide the native grip icon, all scoped under
`.markdown-editor .collection-form-textarea` so perfumes/vinyl/hot-takes (which use the base class
directly, not through MarkdownEditor) are unaffected.

`src/components/MarkdownEditor.js` was not touched — no markup/wrapper change was needed.

## Changes made

### `src/styles/globals.css`

- `.markdown-editor .collection-form-textarea` now also sets `border-radius: var(--radius-stat)` and
  `resize: vertical` (all four corners equal, vertical-only resize).
- Added `.markdown-editor .collection-form-textarea::-webkit-resizer { display: none; }` to suppress
  Chromium's native diagonal resize-grip icon.
