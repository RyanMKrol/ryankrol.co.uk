# T112: Switch TV to TelevisionRatingsV4 (id-keyed)

## What changed
- `src/lib/constants.js`: `TV_RATINGS_TABLE` now `'TelevisionRatingsV4'`.
- `src/pages/api/reviews/tv/add.js`: assigns `id: randomUUID()` on every new item.
- `src/pages/api/reviews/tv/update.js`: takes `originalId` (was `originalTitle`), `GetCommand`/`PutCommand`
  keyed by `{ id }`; removed the title-changed delete-old/recreate-new branch entirely (id never
  regenerated on update, same rule as movies).
- `src/pages/api/reviews/tv/delete.js`: takes `{ id, password }` (was `{ title, password }`),
  `DeleteCommand` keyed by `{ id }`.
- `src/pages/api/reviews/tv/index.js`: unchanged — already scans `DYNAMO_TABLES.TV_RATINGS_TABLE`,
  which now resolves to `TelevisionRatingsV4` and returns items including `id`.
- `src/pages/reviews/tv/edit.js`: list `key` and edit link now use `tvShow.id`.
- `src/pages/reviews/tv/edit/[id].js`: looks up by `t.id === decodeURIComponent(id)`; submits
  `originalId`/`id` instead of `originalTitle`/`title` on update/delete.
- `src/pages/api/reviews/tv/add.test.js`: asserts the created item's `id` is UUID-shaped.
- Docs: `CLAUDE.md` data model row updated for `TelevisionRatingsV4`; `README.md` gets a new
  `#### TelevisionRatingsV4 Table` section (mirroring the movies section).
