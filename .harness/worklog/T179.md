# T179 — Fix now-playing chip dot/label vertical misalignment

## Root cause
`src/styles/globals.css` had a dead, pre-T146 CSS block (`/* Now Playing Component - Last.fm */`,
old `.now-playing`, `.now-playing.loading/.offline`, an old `.now-playing-label` with
`display:block; margin-bottom:0.5rem;`, `.now-playing.loading/.offline .now-playing-label`, an old
`.now-playing-split`, and unused `.now-playing-label-inline`). No component renders bare
`className="now-playing"` any more (confirmed via grep — only `now-playing-chip` is rendered), so
this block was fully dead EXCEPT that its `.now-playing-label` selector shared a class name with
the still-rendered pill-chip label. Because the dead rule was declared before the current pill-chip
`.now-playing-label` rule, the cascade merged both — the current label inherited
`display:block; margin-bottom:0.5rem;` from the dead rule, pushing the idle/loading text upward
off-center relative to the dot (which has no margin).

## Fix
Deleted the entire dead block (was lines ~1554-1600, from the `/* Now Playing Component - Last.fm
*/` comment through the old `.now-playing-label-inline` rule), leaving `.track-link-single`
untouched immediately after. `.now-playing-label` now resolves to exactly one rule (the pill-chip
version at ~1647), which has no display/margin override, so it sits as a normal flex item centered
by `.now-playing-chip`'s `align-items:center`.

## Verification
- `grep -n "now-playing-label\|now-playing\\."` on globals.css now shows only the single current
  `.now-playing-label` rule — no more `.now-playing`, `.now-playing.loading`, `.now-playing.offline`,
  or `.now-playing-label-inline`.
- `.now-playing-chip`, `.now-playing-dot`, `.now-playing-dot.playing`, `.now-playing-split` (chip
  version), `.header-now-playing .single-line-track`, `.header-now-playing .marquee-container` all
  confirmed intact/unchanged after the edit.
- `npm run lint` — 0 errors (6 pre-existing warnings, unrelated to this change).
- `npm test` — 245/245 passing, no new tests needed (CSS-only change, no new pure logic).
- `npm run build` — succeeded.
- Started `npm run dev`, confirmed `/` returns 200 and compiles cleanly with the change applied.
  Did not have a live browser to pixel-verify vertical centering, but the cascade fix is structural:
  removing the sole conflicting rule leaves `.now-playing-label` with no display/margin override,
  which guarantees it behaves as a normal centered flex child like `.now-playing-dot`.
