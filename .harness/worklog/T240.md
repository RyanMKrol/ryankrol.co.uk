# T240 — masonry reading-order fix + shared column technique

## What changed
- `src/lib/masonryColumns.js`: pure `distributeRoundRobin(items, columnCount)` — assigns items to
  columns round-robin (item i -> column i % columnCount), each column keeping relative order.
- `src/lib/masonryColumns.test.js`: even split, uneven remainder, columnCount===1, empty items,
  columnCount >= items.length.
- `src/hooks/useResponsiveColumnCount.js`: SSR-safe hook, returns `desktopColumns` until a
  `matchMedia` listener (mirroring `useMatrixActive.js`'s style) flips it to `1` below the
  breakpoint.
- `src/components/MasonryColumns.js`: generic presentational wrapper — renders `columnCount` flex
  columns from `distributeRoundRobin`, calling `renderItem(item, originalIndex)` with the item's
  original index in the full list (not per-column index), so callers get stable keys.
- `src/pages/projects/index.js`: replaced the `<div className="projects-grid">{sortedRepos.map(...)}</div>`
  block with `<MasonryColumns items={sortedRepos} columnCount={useResponsiveColumnCount(3, 900)} .../>`,
  card JSX unchanged.
- `src/styles/globals.css`: `.projects-grid` changed from CSS multi-column (`column-count: 3`) to a
  flex row; added `.projects-grid-col` (flex column, `flex: 1 1 0`); removed the 900px
  `column-count: 1` media override (column count is now driven by the JS hook at the same
  breakpoint); removed `.project-card`'s `margin-bottom` (redundant with the column's flex `gap`).

## Why
CSS multi-column layout fills column 1 top-to-bottom before starting column 2, so sorting by stars
put item #2 below item #1 in the same column instead of beside it — reading order was
top-to-bottom-then-left-to-right instead of the expected left-to-right-then-top-to-bottom. Explicit
JS column assignment (round-robin) fixes the reading order while each column still packs its own
variable-height cards with no forced row heights or dead gaps.

`masonryColumns.js` / `useResponsiveColumnCount.js` / `MasonryColumns.js` are intentionally generic
(no project-specific logic) so the four sibling sort-order-fix tasks for movies/tv/books/workouts
can reuse them as-is.
