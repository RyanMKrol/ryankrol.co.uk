# T052 — Variant 2 — reviews list/index pages

## What was built

- `src/components/v2/V2ReviewList.js` — shared list-view component implementing Variant 2's
  ("Editorial Card Magazine") list layout: a hero card for the most-recently-reviewed item
  (large serif title, byline-style subtitle, 1.75rem star row, italic pull-quote excerpt),
  followed by the rest of the items in a masonry (CSS multi-column) grid of cards (title,
  subtitle, 1rem stars, clamped 3-line excerpt). A de-emphasized `Sort: ... ▾` `<select>`
  sits top-right of the list, matching the brief ("single small dropdown, not a row of
  buttons"). Reuses `V2Layout` from T044 for the masthead/colophon chrome.
- `src/pages/v2/reviews/{movies,tv,books,albums,perfumes}.js` — five thin pages, each
  configuring `V2ReviewList` with the field-name quirks of its type:
  - movies/tv: `review_text` excerpt, no subtitle.
  - books: `review_text` excerpt, `author` subtitle.
  - albums: `highlights` excerpt, `artist` subtitle.
  - perfumes: `description`/`notes` excerpt, `designer` subtitle, `ratingMax=10` (perfumes
    are rated 0–10, not 0–5 like the other types — `V2ReviewList` halves+rounds the score
    for perfume star rendering so all five pages share one star-rendering code path).
  All fetch from the existing, unmodified API routes (`GET /api/reviews/<type>`).
