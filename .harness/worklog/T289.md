# T289 — Personal bests (weight / 1RM / set-volume) in workoutMetrics.js

## What changed
- `calculateExerciseMetrics`: the working-set forEach now takes `(set, setIndex)` and tracks a
  third running max, `bestSetVolume` (largest single working set's weight×reps), alongside the
  existing `heaviestWeight`/`bestEstimated1RM`. Each axis also tracks the winning set's index
  (`heaviestWeightSetIndex`, `bestEstimated1RMSetIndex`, `bestSetVolumeSetIndex`), initialized `-1`,
  updated only on strict `>` so ties keep the first set reached.
- Return object gains `bestSetVolume` (same null convention as the other weight metrics) and the
  three `*SetIndex` fields.
- New exported pure function `detectPersonalBests(sessionMetrics, priorBest)` — for each of the
  three axes independently, flags a new PR (returns that axis's set index) when the session's value
  is non-null and either there's no prior record, the prior value for that axis is null, or the
  session strictly exceeds it. Ties don't count. First-ever log of an exercise (`priorBest`
  null/undefined) means every non-null axis counts as a PR — deliberate, documented in the spec.
