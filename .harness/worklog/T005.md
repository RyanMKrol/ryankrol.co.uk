# T005 — Perfumes review pages + nav link

## What was built

- `src/pages/reviews/perfumes/index.js` — list view, mirrors movies index (search + sort by
  date/title/score), fetches `/api/reviews/perfumes`, renders via `ReviewCard` (`type="perfume"`).
- `src/pages/reviews/perfumes/add.js` — password-gated add form: title, designer, type
  (EDP/EDT/Parfum select), rating (`<input type="number" min=0 max=10 step=1>`), description,
  notes, considerTravelSize/considerFullBottle checkboxes. POSTs to `/api/reviews/perfumes/add`.
- `src/pages/reviews/perfumes/edit.js` — list with edit links keyed by `perfume.id`
  (`/reviews/perfumes/edit/<encoded id>`), mirrors movies edit list.
- `src/pages/reviews/perfumes/edit/[id].js` — detail form; looks up the perfume by `id` (not
  title, since perfumes key on a derived `id` per `src/lib/perfumes.js`), submits to
  `/api/reviews/perfumes/update` with `originalId`, and deletes via
  `/api/reviews/perfumes/delete` with `{ id, password }`.
- `src/components/ReviewCard.js` — adapted for `type="perfume"`: shows designer as the "by" line
  (with the EDP/EDT/Parfum type appended), a 0-10 max rating (`getMaxRating()`) instead of the
  hardcoded 5-point scale used by other review types, and `description` as the review body.
- `src/pages/index.js` — added a `~/perfumes` link to the home grid, next to `~/albums`.

## Notes

- Perfumes don't use `StarRating` (that component is hardcoded to a 1-5 click scale) — used a
  plain integer `<input type="number">` clamped 0-10 instead, per the spec's "integer 0-10"
  requirement.
- `ReviewCard`'s rating display now scales its star count to `getMaxRating()` per type instead of
  always rendering 5 stars, so perfume cards show a 10-star rating without affecting movies/tv/
  books/albums (still 5).
