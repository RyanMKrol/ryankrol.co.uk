# T140 — Type field label+select same line on perfume forms

## Do

Changed the `Type` `form-group` block in both `src/pages/reviews/perfumes/add.js` and
`src/pages/reviews/perfumes/edit/[id].js` so the label and `<select>` sit on the same line,
sized to content, instead of stacked full-width.

Since the task scope only allows editing these two page files (not `src/styles/globals.css`),
the fix uses **inline styles** rather than a new shared/scoped CSS class:

- The `form-group` div gets `style={{ display: 'flex', alignItems: 'center', gap: '0.75rem' }}`.
- The `label` (still has `className="form-label"`) gets `style={{ display: 'inline', flexShrink: 0 }}`
  to override the shared `.form-label { display: block }` for this instance only.
- The `select` (still has `className="form-input"`) gets `style={{ width: 'auto' }}` to override
  the shared `.form-input { width: 100% }` for this instance only.

The native `<select>` element and its `PERFUME_TYPES` options (EDP/EDT/Parfum) are unchanged.
No other form-group/field markup was touched in either file, and `src/styles/globals.css` was
not modified — `.form-label`/`.form-input` remain untouched and every other field on both pages
keeps its original stacked, full-width layout.
