# T115: AlbumRatingsV3 id-migration script

## What was built

- `src/scripts/migrateAlbumsToIdTable.js` — mirrors `migrateBooksToIdTable.js` exactly in
  shape: scans `AlbumRatingsV2` (composite key `title`+`artist`), creates `AlbumRatingsV3`
  (sole HASH key `id`, String), copies every field unchanged (including the nested `lastfm`
  object) plus a fresh `randomUUID()` id, batch-writes in chunks of 25, guards against
  double-migration (aborts unless `--force` if target is non-empty), and supports `--verify`
  (re-scans both tables, matches on `title`+`artist`, deep-diffs every field via the exported
  `diffMigratedItem`, prints `VERIFY: PASS`/`VERIFY: FAIL (<n> mismatches)`, non-zero exit on
  failure). Running with no flags migrates then auto-verifies.
- `src/scripts/migrateAlbumsToIdTable.test.js` — covers `buildMigratedItem` (field
  preservation including nested `lastfm`, `id` presence, v4 UUID shape, uniqueness),
  `diffMigratedItem` (clean match, changed field, nested `lastfm` mismatch, missing item),
  and `findMatchingNewItem`.
