# T040 — Harness dashboard rework — bucket grouping + sort order (data layer)

## What changed

Reworked `computeBacklog`/`summarize` in `.harness/dashboard/lib.js` to collapse the previous
seven-bucket model down to exactly four: `ready`, `waiting`, `needsHuman`, `done`, matching
`local-jobs`' backlog page as the reference model.

- `ready` — unchanged.
- `waiting` — old `waiting-human` logic unchanged (recursive `blockedByHuman`). Old
  `waiting-loop` tasks are now filtered out of the returned array entirely (never appear in any
  bucket).
- `needsHuman` — union of old `needs-human` (gate/needs-human) and old `blocked`
  (worklog-blocked-by-owner) buckets. `blocked: true` remains on the task object so the UI (T041)
  can distinguish the two sub-states.
- `done` — union of old `done` and old `failed` buckets. `failed`/`reviewed` flags remain on the
  task object. Sorted not-reviewed-first, then ascending numeric task id (`numericId` helper
  parses the `\d+` out of `T123`).
- `summarize()` now reports `{ ready, waiting, needsHuman, done }`.
