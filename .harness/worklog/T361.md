# T361: Switch book search from Google Books to Hardcover

## Completed

Implemented Hardcover GraphQL API as the new backend for book search, replacing Google Books.

### Files Created

1. **`src/lib/hardcover.js`** — `mapHardcoverResult(hit)` normaliser function
   - Converts Hardcover GraphQL search hit.document to the common book-result shape
   - Maps: id→volumeId, title→title, author_names→authors ([] default), release_year→firstPublishedYear, image.url→coverUrl (null if absent), isbns→isbn ([] default), genres→subjects ([] default), pages→pageCount (null if non-numeric), publisher always []
   - Handles all missing/null/sparse cases gracefully, matching mapGoogleBooksResult's degradation behavior

2. **`src/lib/hardcover.test.js`** — 20 comprehensive unit tests
   - Well-formed hit with all fields present
   - Single and multiple authors
   - Missing/null image object and image.url
   - Missing/null author_names, isbns, genres
   - Non-numeric pages (returns null)
   - Sparse document (only id, all other fields null/[])
   - Missing document property
   - Null and undefined inputs

### Files Modified

1. **`src/pages/api/books/search.js`** — Replaced search backend
   - Renamed `searchGoogleBooks()` → `searchHardcover(title, author)`
   - GraphQL POST to `https://api.hardcover.app/v1/graphql` with Bearer token auth
   - Single free-text query combining title+author (no separate params)
   - Per-page: 20 results
   - Extracts results from `data.data.search.results.hits`
   - Maps each hit through `mapHardcoverResult`
   - Identical caching contract: `withApiCache(generateCacheKey('book-search', {title, author}))`
   - Identical rate-limit gate: 20 req/60s per IP via `checkRateLimit`
   - Identical error handling: 429 detection + Retry-After propagation for backfill retry logic
   - Identical logging style: `📚 [Hardcover]` / `❌ [Hardcover]` prefixes
   - Console.warn if `HARDCOVER_API_TOKEN` unset (mirrors `GOOGLE_BOOKS_API_KEY` pattern)
   - Detects GraphQL errors in response and logs them

2. **`CLAUDE.md`** — Updated documentation
   - Added `HARDCOVER_API_TOKEN` env var to "Key env vars" table with description
   - Updated `/api/books/search` file map entry to describe Hardcover instead of Google Books
   - Added `src/lib/hardcover.js` file map entry describing `mapHardcoverResult` normaliser
