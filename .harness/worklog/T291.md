# T291 — backfill personal-best flags across historical exercise rows

## What was built
- `src/scripts/backfillPersonalBests.js`: scans the `Exercises` table, groups rows by
  `exercise_name`, sorts each group chronologically ascending (oldest first — the opposite order
  from the live write path), and walks forward maintaining an in-memory running best
  `{heaviestWeight, bestEstimated1RM, bestSetVolume}` per exercise. Recomputes session metrics
  fresh from each row's own `sets[]` via `calculateExerciseMetrics` (does not trust any
  pre-existing aggregate fields), calls the shared `detectPersonalBests`, and for every row with
  a new PR on any axis, writes the flag(s) onto the specific set index in both the `Exercises`
  row (`sets`) and the matching `Workouts` item's embedded `exercises[j].sets` — both via
  `buildSetUpdateParams`. Dry-run by default, `LIVE=1` to write. Pure grouping/chronological-walk
  logic exported for testing.
- `src/scripts/backfillPersonalBests.test.js`: covers multi-exercise interleaved-input grouping +
  chronological sort, running-best carried forward correctly across a sequence (including that a
  later dip below the running best does not erase it), and per-row PR flags matching what
  `detectPersonalBests` independently produces.
- `package.json`: added `workouts:backfill-personal-bests` npm script.
