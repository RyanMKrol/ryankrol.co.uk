# T167 — Visual audit: vinyl, listening, projects vs design renders

Compared the live pages (desktop 1400px + mobile 390px, via a headless Playwright pass)
against `renders/2b.png`/`4c.png` (vinyl), `2d.png`/`4e.png` (listening), `3e.png`/`4j.png`
(projects) from the design handoff.

## Gaps found + fixed

1. **Vinyl — search/toggle row not aligned with the title row.** The page used a standalone
   `.search-container` block that stacked full-width below the header on desktop, but the
   design puts search + covers/list toggle inline on the right of the `vinyl` title row (same
   pattern already used by the reviews pages via `.collection-review-controls`). Restructured
   `vinyl/index.js` to nest the search input + `PillGroup` inside `.collection-review-controls`
   as a sibling of `.collection-review-title-group` inside `.collection-review-header`, moved
   the "Found N records" count below the header instead of inside the old container, and
   changed the search placeholder to "search collection..." to match the render. No CSS
   changes needed — reused the existing shared header-row class.

2. **Listening — mobile rows showed a wrapped progress bar; the design has no bar on mobile
   at all.** `globals.css` `@media (max-width: 600px)` previously wrapped `.listening-row` and
   floated `.listening-bar-track` onto its own line. Changed to `display: none` for the bar
   track on mobile and dropped the now-unnecessary `flex-wrap`, matching `4e.png` exactly
   (rank + cover + title/artist + playcount only).

3. **Projects — GitHub topic pills rendered as uppercase Space Mono chips; the design shows
   lowercase Nunito pills** (e.g. "nextjs", "aws"). `Badge` only ever rendered the mono/
   uppercase treatment used correctly elsewhere (workout PUSH/PULL/LEGS badges, perfume EDP/EDT
   chip). Added an opt-in `mono` prop to `Badge.js` (defaults to `true`, so every existing
   caller is unaffected) plus a `.collection-badge-plain` CSS class, and set `mono={false}` on
   the projects page's topic-pill `Badge` only.

4. **Projects — repo name link showed a browser-default underline; design has none** (underline
   only appears on hover in spirit of the other card links). Added
   `text-decoration: none` / `:hover { text-decoration: underline }` to `.project-card-name`.
