# T077 — Remove Open Library from book search (API + toggle UI)

## What changed
- `src/pages/api/books/search.js`: dropped the `provider` query param, the `mapBookResult`
  import from `lib/openlibrary`, and the `searchOpenLibrary` function. Every request now always
  calls `searchGoogleBooks`.
- `src/components/BookSearch.js`: removed the `PROVIDERS` array, `currentProvider` state, and the
  per-provider button row. Replaced with a single "Search" button that always searches Google
  Books. `providerLabel` is now a constant `'Google Books'` (still used in the confirmed/results
  labels).

## Out of scope (left untouched, per spec)
- `src/lib/openlibrary.js` and its test still exist on disk — `BookSearch.js` still imports
  `bookCoverUrl` from it as a cover-URL fallback for any result missing `coverUrl`. Deleting the
  lib is T078's job once nothing references it.

## Verify
- `npm run lint` — 0 errors, 4 pre-existing warnings (unrelated files).
- `npm test` — 14 suites / 141 tests passed.
- `npm run build` — succeeded, `/api/books/search` still builds as a dynamic route.
- Did not run `npm run dev` + manually search (would need `GOOGLE_BOOKS_API_KEY` / live network
  call to Google Books) — verified via code reading + build instead, per the "no live paid API
  calls to verify" rule.
