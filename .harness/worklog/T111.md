# T111: TelevisionRatingsV4 id-migration script

## What changed
- `src/scripts/migrateTvToIdTable.js`: new script, mirrors `migrateMoviesToIdTable.js`. Creates
  `TelevisionRatingsV4` (id String HASH key, PAY_PER_REQUEST) idempotently, scans all of
  `TelevisionRatingsV3`, guards against double-migration (aborts unless `--force`), copies every
  field unchanged and appends `id: randomUUID()`, batch-writes in chunks of 25. Supports
  `--verify` (re-scan both tables, match old→new on `title`+`date`, diff every other field via
  exported `diffMigratedItem`, exit non-zero on any mismatch/unmatched item). No-flag run performs
  the migration then automatically runs verification and prints `VERIFY: PASS` / `VERIFY: FAIL (<n>
  mismatches)`.
- `src/scripts/migrateTvToIdTable.test.js`: tests `buildMigratedItem` (field preservation, `id`
  presence, v4 UUID shape, uniqueness across calls) and `diffMigratedItem`/`findMatchingNewItem`
  (clean match, changed field, missing item).
