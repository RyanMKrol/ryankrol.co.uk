### CSS rules updated to use vars

Updated three previously-hardcoded CSS rules in the "Cyberpunk Enhancements" section:
- `::-webkit-scrollbar-track/thumb/thumb:hover` → use `--scrollbar-*` vars
- `::selection` → use `--selection-bg`, `--selection-color`
- `body::before` (top gradient bar) → use `--top-bar-gradient`

### Default theme change

Pre-paint script in `_document.js` changed: `'cyberpunk'` fallback → `'terraria'`.
`appearance.js` still has `DEFAULTS.theme: 'cyberpunk'` (out of scope for T014) — this
means `useTheme()` React state initialises as 'cyberpunk' for new users while the DOM shows
'terraria'. Visuals are DOM-driven (CSS), so the correct theme renders. The T015 picker
should read from `document.documentElement.getAttribute('data-theme')` rather than
`useTheme().theme` to avoid this mismatch, OR `appearance.js` should be in T015 scope
to sync `DEFAULTS.theme` to 'terraria'.

### Matrix-active block

Updated with matching `--scrollbar-*`, `--selection-*`, and `--top-bar-gradient` vars
(green-on-black scrollbar, green selection highlight, green gradient bar).

`npm run build` passed cleanly. One pre-existing ESLint warning (react-hooks/exhaustive-deps
