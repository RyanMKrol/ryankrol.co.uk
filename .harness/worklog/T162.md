# T162 — Restyle movies/TV add+edit forms to Collection tokens

## What changed

- Added a new `.collection-form-*` CSS namespace to `globals.css` (card, group, label, input,
  textarea, button, danger button, actions row, success/error message) instead of touching the
  shared legacy `.form-*` / `.btn-danger` / `.success-message` / `.error-message` classes — those
  are still used by books/albums/perfumes/vinyl/workouts forms, which are out of this task's scope
  and must render unchanged.
- `src/pages/reviews/movies/add.js`, `tv/add.js`, `movies/edit/[id].js`, `tv/edit/[id].js`: swapped
  every legacy form class for the new Collection classes; swapped the danger delete button and
  the backfill-result thumbnail placeholder colors/radii to Collection tokens (`--color-hairline`,
  `--radius-cover`). No `onSubmit`/`onSearch`/API-calling logic touched.
- `src/components/TmdbSearch.js` (shared by movies + tv add flows): search/select buttons now use
  `.collection-form-button`; the confirmed-match banner, clear button, and result-card borders now
  use Collection tokens (`--accent-movies`, `--color-ink`, `--border-width`, `--radius-*`) instead
  of the matrix-only `--color-accent`/`--color-text`/`--color-border` variables, which are only
  defined under `html.matrix-active` and were previously unstyled/invisible in the default theme.
- `movies/edit.js` and `tv/edit.js` (the editable list pages) needed no changes — they only render
  `ReviewCard` (already Collection-styled) plus classes shared with books/albums/perfumes edit
  lists, which are out of scope.

## Verify

- `npm run lint` — 0 errors, 6 pre-existing warnings (unrelated files).
- `npm test` — 35 suites / 241 tests passed.
- `npm run build` — succeeded.
- `npm run dev` + `curl` — `/reviews/movies/add` and `/reviews/tv/add` both return 200 with no
  server error.
- Did NOT exercise the live TMDB search-and-confirm flow or the backfill modal end-to-end, since

  the "never make live calls to metered APIs" rule. The search/confirm/backfill code paths were
  not modified (only their surrounding CSS classes), so behavior is unchanged; a human should do a
  visual/full-flow pass per the task's "Manual verify" note, and full pixel-fidelity tuning is
  deferred to T169 per the spec.
