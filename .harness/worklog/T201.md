# T201: Add list-aware gradient assignment helper to CoverTile/ReviewCard

## What changed

- `src/components/CoverTile.js`: added `export function assignGradients(keys)` — pure sliding-window
  (last 4) no-repeat gradient assignment, reusing `hashString`/`GRADIENT_PAIRS`. Prefers each key's
  own `gradientForKey` colour and only probes forward (`preferred+1, +2, ...` mod length) when that
  collides with the trailing window. `CoverTile` gained an optional `gradient` prop: when set it's
  used verbatim as the tile background; when omitted, falls back to
  `gradientForKey(id || title || '')` exactly as before.
- `src/components/ReviewCard.js`: `ReviewCard` gained an optional `gradient` prop, threaded to
  `SpineCoverCard` (books) and the inline `poster-banner` branch (movies/tv) — both use
  `gradient || gradientForKey(...)` — and to `SquareCoverCard` (albums), which forwards it straight
  through to its internal `<CoverTile gradient={gradient} />` rather than calling `gradientForKey`
  itself.
- No call sites were changed to pass `gradient` — that's the follow-up wiring task (task 2). All
  existing callers keep computing the same colours as before.

## Tests

Added `src/components/CoverTile.test.js`:
- `assignGradients`: no-repeat-within-4 assertion over 30 synthetic keys, determinism (same input →
  same output), and a preferred-colour-preserved check (first item in a list has no trailing window,
  so it always keeps its own `gradientForKey` value).
- CoverTile backward compatibility: jsdom's bundled CSSOM (`cssstyle`) does not parse the
  `background: linear-gradient(...)` shorthand at all — verified directly (a minimal inline
  component's `style={{ background: 'linear-gradient(...)' }}` renders with no `style` attribute in
  this environment). Rendering the real component confirmed the same: no `style` attribute lands in
  the DOM regardless of props, so DOM-based background assertions are unusable here, not a defect in
  the component. Per the task spec's fallback guidance, backward-compatibility is instead verified by
  asserting the same expression `CoverTile` uses (`gradient || gradientForKey(id || title || '')`)
  directly, plus a render-without-crashing smoke test for both the omitted- and provided-gradient
  cases.
