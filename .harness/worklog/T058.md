# T058 — Variant 3 — reviews list/index pages

## Built

- `src/components/v3/V3ReviewFeed.js` — shared generic feed component (reused by all 5
  pages) implementing Variant 3's brief: single vertical timeline, date gutter via
  `V3TimelineEntry`, inline `+`/`−` disclosure per entry (no separate detail route),
  a `sorted by date, newest first — change` inline toggle link (no button group),
  no `<h1>` (page title only appears in the utility bar via `V3Layout`'s `title` prop).
- `src/pages/v3/reviews/{movies,tv,books,albums,perfumes}.js` — thin per-type pages
  configuring the shared feed with each type's real field shapes fetched from the
  unchanged production API routes (`/api/reviews/<type>`):
  - movies/tv: `title`, `rating` (0–5), `review_text`
  - books: `title` + `author`, `rating` (0–5), `review_text`
  - albums: `title` + `artist`, `rating` (0–5), `highlights`
  - perfumes: `title` + `designer`, `rating` (0–10, different scale — star renderer
    normalises via `ratingScale` prop), `description`/`notes`

## Scope notes

- No file under `src/pages/api/**` touched; no file under `src/pages/reviews/**`
  (production) touched.
- Did not build inline add/edit forms (the brief's "detail/form view" also covers
  authenticated editing) — out of scope for this chunk's done-when, which only
  requires list pages rendering real data with no console errors/404s. Read-only
  inline disclosure (the "detail" half of the brief) is implemented; write/edit
  flows would need password-gate wiring and are a natural follow-up task.
